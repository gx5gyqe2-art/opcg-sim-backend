## 1. 原因の確定（全て測定・残存仮説なし）

対象現象: 付与系マークの 0↔1 反転（「シーソー」）と candidate の足踏み。

1. **value は付与/非付与をほぼ区別しない**（適用後 value 差 0.01〜0.05・gen5/r82/r96 の3ネットで測定）
   ＝効率手の決定は policy prior 質量が支配する。
2. **policy 教師は prior のエコー**: 実生成データ 2,191 決定点で「教師（訪問分布）vs 生成ネット prior」の
   相関**中央値 0.934**（付与決定点 0.937・top1一致 69.5%・低相関はわずか 8.6%＝再ラベル5%＋value主導分）。
   教師に「何が正解か」の信号がほぼ入っていない。LR を下げても止まらないのは学習率でなく
   **教師信号の情報量**の問題だから。
3. **prior は独立に酔歩する（重み干渉の反相関は不在）**: r82→r96 の重み線形補間で、
   g3@64/68（同文脈対）は同方向に単調移動する一方、g1@12/@82 はこの方向に**無反応**
   （0.20/0.31 で不動）。「片方が立つと片方が落ちる」ように見えたのは、**独立なドリフトを
   6シードの 0/1 マーク判定で量子化して見た錯視**（転換点近傍では PIMC 世界線の分散も加わる）。
4. 補助測定: 露出不足ではない（付与候補を含む決定点=全体の 50.2%）。静的特徴の限界も測定済み
   （「勝てる攻撃の増分」特徴は判別失敗・L1評価差でも 4/6・@93 の無駄さは先読みでしか見えない）。

**一言で**: value が無差別な決定点では教師が prior の再生産になり、正解を教える信号がループに
存在しない。prior は訓練ノイズで漂流し、マーク評価がその漂流を 0/1 に量子化して見せていた。

## 2. 対策（3レバー・実装済み・全て 0=恒等）

| 案 | 実装 | 機構 | 推奨値 |
|---|---|---|---|
| **D 生成 prior 部分平坦化** | `p3_loop.priors_fn_of(flatten=λ)`／`pd_gen --prior-flatten` | 生成探索の prior を p′=(1−λ)p+λ/K に＝**全教師に prior 非依存の実探索成分**を常時混ぜ、エコーを構造的に断つ。root 限定の Dirichlet と違い**木の深部まで**効く。serve（`cpu_learned._priors_fn`）は不変 | **0.2** |
| **E 教師ラベル平滑化** | `policy.smooth_target`／`train_policy(smooth=α)`／`pd_learn --policy-smooth` | t′=(1−α)t+α/K＝**prior が 0 に沈む盲点の不可逆化を防ぐ床**。データは生のまま学習時に適用（α変更に再生成不要） | **0.03** |
| **F Q補正教師** | `p3_loop.q_reweight(N,Q,β)`／`pd_gen --q-teacher-beta` | t∝N·exp(β(Q−maxQ))＝**探索が感じた良し悪し（value の意見）を教師に注入**。@82 型（value は正着支持なのに prior が誤誘導）の断線を接続。N=0 の手は持ち上げない | **2.0** |

- 役割分担: D=エコー源の遮断（探索側）／F=value 情報の伝達（教師側）／E=最低保証の床（学習側）。
- 深探索再ラベル（v6 柱②）は継続。@82 の初改善（0→0.33・r99）は再ラベル効果と整合＝D/E/F はこれを加速する。

## 3. 棄却・縮小した案（検証で判定）

- **B 再ラベルの手種別重み付け → 棄却**: 前提（露出不足）が実測で崩れた（付与決定点=50.2%）。
- **A 行動特徴の拡張 → 縮小・保留**: 素朴な「勝てる攻撃の増分」は判別失敗。L1評価差ベースの
  交換経済特徴でも 4/6（g1@12 は L1 自身が人間と不一致・@93 は静的特徴に原理的に写らない）。
  D/E/F の run 結果を見てから再検討する。
- **重み干渉対策としての特徴分離（旧柱④文脈）→ 不要**: 干渉（重み結合の反相関）自体が棄却された。
- **G serve prior 床 → 保留**: 学習不要の serve 側実験として独立に A/B 可能（mark_gate で1時間）。
  v7 run と干渉しないため、必要になった時点で実施。
