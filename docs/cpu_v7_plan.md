## 1. 背景: v6 run が残した実測課題（正本: `docs/reports/seesaw_probe_20260716.md`）

1. **シーソー（付与系マークの 0↔1 往復）の機構が確定**: これらの決断では value がほぼ無差別
   （適用後差 0.01〜0.05）のため、探索訪問＝prior＋ノイズとなり、その訪問分布を policy が教師として
   再学習する**自己参照エコー**が prior 質量のランダムウォークを生む。LR 半減では止まらない
   （学習率でなく教師信号の情報量の問題）。
2. **PRIOR_BOUND マーク（@82/@93）の再ラベル露出不足**: 深探索（flat）は解けることが実証済みだが、
   5% 一様サンプリングでは希少な決定点に十分届かず、cum12k 時点で prior を錨止めできていない。
3. gen5+13pt 相当まで前進した一方、candidate は best 近傍で飽和＝天井の原因は時間でなく
   表現力/教師配分にある可能性が高い。

## 2. v7 の追加2本柱

### A. 行動特徴の拡張（policy の文脈盲の解消）

現行の行動特徴（`action.py`・ACTION_DIM＝手種別 one-hot + 関与カード特徴 + 3）には
**「この手が何を可能にするか」が無い**。付与もカウンターも、結果の文脈を持たないため
policy は同種の手を区別できず、prior が漂流する。追加する特徴（すべて盤面から決定的に計算・
カード固有性なし＝汎用実装の方針に整合）:

- **ATTACH_DON 系**: 付与後に「新たに届くようになる攻撃」の有無/数（相手リーダー・キャラへの
  到達しきい値の跨ぎ）、付与後の残ドンで打てる主要行動の有無、このターン攻撃可能なアタッカー数。
- **SELECT_COUNTER/BLOCKER 系**: この防御で攻撃が不成立になるか（カウンター後のパワー差）、
  守らなかった場合のライフ残・リーサル距離、切るカードの手札価値（コスト/カウンター値）。
- **共通**: 手適用後の自分/相手の即時脅威数の差分（1手先の静的差分＝探索不要の軽量計算）。

実装: `action_features` に append-only で追加（ACTION_DIM 拡張）。policy は ctx 末尾でなく
action 側の拡張なので `PolicyScorer.expanded` 相当の温スタート（新次元ゼロ列＝恒等）を用意する。
事前アブレーション（v6 計画 §4-4 の規律）: 追加特徴の分散・既存との相関・単体寄与を計測してから採用。

### B. 再ラベルの行動種別重み付け（教師配分の是正）

`selfplay_game` の relabel 判定を一様確率から**手種別で重み付け**する（例: ATTACH_DON/
SELECT_COUNTER/SELECT_BLOCKER の決定点は base の4倍・その他は base）。正解は引き続き
深探索（prior平坦化）が局面ごとに導出＝ラベルの汎用性は不変で、**配分だけ**を「value が
無差別で prior エコーに陥りやすい決定種」へ寄せる。総コストは relabel 総量で管理
（type別重み × base_frac の期待値を現行 5% 相当に正規化）。

期待効果: 無情報エコーの代わりに深探索の答えが繰り返し注入され、prior 質量が錨止めされる
＝シーソー停止・@82/@93 の改善。
