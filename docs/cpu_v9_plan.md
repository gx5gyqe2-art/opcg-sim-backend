# CPU v9 計画: 外部アンカー学習（レフェリー再ラベル）2026-07-17 確定

## 0. 位置づけ

v7 で確定した「自己対戦ループ内に真実源が無い」に対する学習側の答え。v8（教師CPU）で実証した
真盤面レフェリー（CRN 対照ロールアウト・対判定・捲りモード）を**教師ラベルの生成器**として
学習ループに接続する。教師ネットは gen5 固定（ドリフトしない錨）。

## 1. 何を学ぶか（v8 実証済みの判定を教師信号に変換）

1. **文脈依存の効率判断（policy）**: 同じ「付与」でも負け気味なら付与→攻撃（@64）・
   勝ち気味ならテンポ攻撃（@68）、カウンター温存（@82）、ブロッカー温存（@81）——
   value が ±0.01 で無差別だったため現ループでは学べなかった領域。
2. **負け局面の非一様な価値（value）**: 従来ラベルは負け一律 −1。レフェリーの勝率＋
   ライフ差ラベルで「同じ負けでも良い負け方」に勾配を入れる。
3. **捲り率（新規・ユーザ方針）**: temp0 同士では負け確定局面の捲り率が構造的に全ゼロ＝
   信号なし。捲りモード（世界数×4＋相手温度）のラベルで「アウツを残す手」と
   「無意味に負けを整える手」を初めて区別して教える（@137: ゼウス付与 1/16 vs 他 0/16）。

## 2. フェーズ1: 再ラベルパイプライン（本計画の実装範囲）

**訂正（設計判断）**: v7 データ枝の 15.8k 局は符号化済みテンソルのみで盤面状態を持たず
直接再ラベル不能。ラウンド別ネット重みの考古学より、**gen5 固定で「再生可能な記録つき
自己対戦」を新規生成**する方が安く確実（ラベルの錨も gen5＝整合的）。

`tests/scripts/referee_labeler.py`（新規・1トピック=1ファイル）:

1. **生成**: 両席 learned(gen5) ＋ 録画 observer で対局を記述子（seed/decks/actions）として
   記録＝真盤面をいつでも再生できる形。
2. **採掘**（1パスの観測・軽量）: 各決定点で
   - **効率盲点**: 合法手の 1-ply 後の value 差が ε 未満（無差別＝ループが学べなかった点）
   - **飽和負け**: value < −しきい（捲りラベルの主戦場）
   を検出し、1局あたり上限 K 点を選ぶ（**少数を深く**）。
3. **ラベル**: 選ばれた決定点を真盤面再生→プラン自動列挙→CRN 対照評価（飽和なら捲り
   エスカレーション）。
   - policy 教師 = 同価値バンド上位プランの**初手**の多重 one-hot（band 外は 0）
   - value 教師 = 最良プラン勝率 z=2·wr−1（捲り時はエスカレーション後の値＝捲り率）
4. **出力**: 既存バッチスキーマ v2（scalars/field/card_idx/value/q_root/turns_left＋
   pol_ctx/pol_am/pol_visit/pol_len）＋ meta（worker="ref"）＝ **pd_learn がそのまま消費可能**。

## 3. フェーズ2: 学習（フェーズ1の産出が貯まってから）

- gen5 を温スタートし、レフェリーバッチ＋通常自己対戦バッチ（正則化）を混合して微調整
  （LR 小・pd_learn の既存機構をそのまま使う）。
- 混合比・LR は数千ラベル貯まった時点でスモークして決める（本計画では未確定のまま残す）。

## 4. フェーズ3: ゲート（昇格判定）

従来ゲートに**教師品質ゲート**を追加（総合勝率は効率改善に鈍感＝進歩検出はコーチ指標側）:
1. 昇格ゲート（対best 勝率・stage1/2）＝退行防壁
2. アンカーゲート（対固定 gen5 ≥ 0.5）＝血統過適合防壁
3. **コーチゲート（新設）**: held-out 録画のスイープ損失数が gen5 以下・飽和負け局面の
   捲り率（対不完全相手）が gen5 以上
4. mark_gate（人間マーク回帰）

## 5. スコープ外（記録のみ）

- 新規対局の大量生成はフェーズ1の材料が枯れてから（その際は捲りモード付きロールアウトを
  生成側に入れ、飽和局面の信号を生成時から確保する）。
- 人間対局の録画が増えれば同じパイプラインでそのまま教師化できる（真盤面化は現行ビルド録画で
  全手可能・v8 実証済み）。
