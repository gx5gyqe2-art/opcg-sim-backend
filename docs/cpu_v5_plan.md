## 3. 根因の統合仮説

### R1. v4 の防御データ被覆が過剰で「守りすぎ」を招いた（C1 の直接原因の疑い）

v4 の (c)**防御応答の温度延長**（SELECT_COUNTER/BLOCKER をゲーム全体で温度サンプリング）は、
「守らない」を直すために入れたが、**自己対戦データに防御手を過剰注入**し、value が「守り＝善」へ
振れた可能性が高い。→ **検証**: v4 自己対戦バッチの counter/block 採択率を「良質プレイの目安」と比較する
計器（`clock_error_by_leader` と同型の読み取り専用分析）で確認し、過剰なら温度延長を**条件付き**に
弱める（例: 相手クロックが高い＝守る価値が高い局面のみ延長）。

### R2. 較正の不安定（C2/C3）＝「程度」を教える信号の欠如

付与も攻撃も「打つ/打たない」の二値でなく**どれだけ価値があるか**の連続量が問われるが、勝敗±1＋
q_root ラベルは局面価値であって「この付与で戦闘結果がどれだけ改善するか」の**差分**を直接教えない。
→ v5 は §4-3 マーク局面シード＋§4-5 encoder 拡張（自デッキ残）でデータ・入力の両面から底上げし、
必要なら「付与前後の value 差分」報酬プローブを診断計器化（学習への直接投入は後段で検討）。

### R3. 負けq飽和（C4）＝残りターン補助の活用不足

value が敗勢で −1 に張り付くと手が区別できない。v4 で入れた**残りターン補助ヘッド**は「あと何ターンで
負けるか」を持つので、**敗勢でも「長く粘る手」を選べる材料**になる。現状 aux は表現学習の誘導のみで
手選択に使っていない。→ v5 で aux 重みの調整、または探索の葉評価に「aux 由来の粘り項」を薄く混ぜる
実験（終局深さ減衰 F2 の中盤版）。

## 4. v5 run の構成（v4 配管を流用・差分のみ）

v4 の生成/学習/ゲート/退避/keep-alive 運用は流用。差分:

1. **前提調査（R1）**: counter/block 採択率計器で「守りすぎ」の原因が防御温度延長かを切り分ける
   （学習前・読み取り専用）。
2. **防御温度延長の条件化**（R1 が確なら・C1）: 相手クロックが高い局面のみ温度延長＝守る価値の低い
   局面に防御サンプルを注入しない。
3. **マーク局面シード**（旧計画・未実施＝D1/C2 の最優先対策）: v4 の失敗局面（C1 の @82/@115・
   C2/D1 の @64/@68・C5 の @3・過去 g1@12/@24 の付与正着）を自己対戦の開始局面に混ぜ、
   **観測された失敗モードそのもの**を in-distribution 化。
4. **忘却抑制（D2）**: (a) mark_gate/対v3 の同時後退を検知する**ピーク自動アラート**を監視サイクルへ
   組込む・(b) 学習損失に **v4 を教師にした軽量 KL 蒸留正則化**を加える試験（EWC より軽実装＝
   「v4 の出力から離れすぎない」制約で振り子=C1 と忘却を同時に抑える）。効果が無ければ外す。
5. **encoder 拡張（D3・C5・append-only）**: 自デッキ残の集約特徴（残カウンター密度・防御札残量・
   キーカード残枚数）を scalars 末尾に追加＝時計誤差の対面別偏りと資源評価（ライフvs手札）を底上げ。
   v4 からの温スタートは既存の恒等拡張機構（`expanded`/warm_start）で自動追従。
6. **温スタート**: v4（`gen4_*.npz`）から。α/aux は v4 起点でホールドアウト較正（C4 対策で aux 寄りも検討）。
7. **ゲート拡張**: 既存16マーク＋**本14マーク**を fixtures に追加（`g3_v4_replay_*`）。特に C1 の過補正
   （守りすぎの解消）・C2/D1 の付与較正・C3 を必達に含める。対L1多様97 は v4 の 0.812 非劣。

> 実装順序: 1（調査）→ 5（encoder 拡張＝データ再生成が要るので最初に確定）→ 2/3（生成側）→
> 4b/6（学習側）→ 7（ゲート）→ smoke → 本走。D1（マーク局面シード）と C1（守りすぎ）が二大主眼。

## 5. やらないこと

- カード個別パッチ。汎用実装のみ。
- ピーク凍結運用そのものは継続（v4 で有効）＝忘却の**完全**解消は狙わない。D2 は「振り子を抑えて
  ピークを高く・広くする」ための軽量策に留め、重い継続学習理論（本格 EWC 等）は範囲外。
