## 2. 課題の集約（v4 §3 の引き継ぎ＋新規）

### F5. ドン付与の価値づけが双方向に誤っている【v5 の学習主眼】

v4 で「付与＝悪」に過剰一般化した副作用が残存。**過小評価**（@6・防御条件を開く付与を打たない）と
**過大評価**（@68・無意味な付与を打つ）が同居＝「付与の価値は**後続プランと効果条件**に依存する」
という文脈依存を net がまだ表現できていない。特に @6 は **ナミ【ドン!!×1】【相手のアタック時】+2000**
の**条件を能動的に開くための付与**＝「今のドン数で何の効果条件が解禁されるか」という
効果条件×資源の相互作用（EffFeat の HAS_DON は静的に存在するが、動的な「あと1枚で解禁」は未表現）。

**治し方（学習）**: 付与が報われる系列をデータに増やす。
- **マーク局面シード（旧v4計画 §4-1(e)・未実施）**: 蓄積マークの復元盤面（付与正着の g1@12/@24・
  本@6）を自己対戦の開始局面に混ぜ、in-distribution 化する。
- **付与→効果解禁の合成報酬プローブ**（任意）: 【ドン!!×N】条件を跨ぐ付与の前後で value 差が
  正になるべき局面を診断計器化（`clock_error_by_leader` と同様の読み取り専用分析）。

### S6/S7. 探索・エンジン層の要調査（学習では直らない・v5 の前提整備）

- **S6（@6）**: 復元盤面で p2（ドン2アクティブ・手札6）の合法手が **TURN_END のみ**。真にエンジンの
  合法手列挙欠落なら重大バグ。まず「復元アーティファクト（passive 状態の近似）か実バグか」を
  `replay_reeval` の recorded_is_legal と実ゲーム進行で切り分ける。
- **S7（@14）**: CONFIRM_TRIGGER 対話で `merged_search_actions` が候補を1つしか出さない＝探索が
  「トリガーを撃つ/撃たない・置き先」を比較できない。`adapter.legal_actions`／`merged_search_actions`
  のトリガー確認展開を点検（up-to/CONFIRM_OPTIONAL は展開済み・CONFIRM_TRIGGER が漏れている疑い）。

### S8. 読み出しと記録の不整合（確認事項・おそらく既に解消）

@68 の記録 chosen=ATTACH_DON に対し、同じ統計を現行 `_select_root_group`（二重ゲート）に通すと
**TURN_END を返す**（q −0.428 > −0.481・訪問25.9≥ゲート22.4・q差0.053≥0.05）。このリプレイが
二重ゲート採用（PR #181）**前**のビルドで採られた可能性が高い＝現行 v4 本番では既に TURN_END を
選ぶ見込み。**v5 着手時に現行 gen4 で @68 を再現し、読み出しが正しく TURN_END を返すことを確認**して
本項をクローズする（新規実装不要なら S8 は「確認のみ」）。

## 3. v5 run の構成（v4 の配管を再利用・差分のみ）

v4 で整備済みの生成/学習パイプライン（sticky・防御温度・L1混合・スキーマv2・混合ラベル・
残りターン補助・mark_gate・checkpoint退避・keep-alive運用）は**そのまま流用**する。v5 の差分:

1. **前提整備（学習前）**: S6/S7 を調査し、配管バグなら先に修正（学習で覆い隠さない）。S8 は確認のみ。
2. **マーク局面シード（§F5）**を生成経路に実装（`selfplay_game(start_boards=…)`＝`replay_reeval`の
   復元機構を流用して開始局面を差し込む・配合比 10〜20%）。
3. **温スタート**: v4（`gen4_*.npz`）から。α・aux 重みは v4 と同じ（0.5 / 0.25）を起点にホールドアウト較正。
4. **ゲート**: 16マーク（既存）＋**本6マークを fixtures に追加**して22マークへ拡張。特に付与正着
   （@6・g1@12/@24）の回復と、@68 の過大評価解消を必達に含める。対L1多様97 は v4 の 0.812 非劣。

## 4. やらないこと

- カード個別パッチ（ナミ専用ロジック等）。汎用実装のみ。
- 忘却対策の本格導入（EWC/蒸留正則化）は v5 の主眼から外す（ピーク凍結運用を継続）。ただし
  v4 で確認したピーク一過性は残るため、マイルストーン評価でのピーク検出は v4 同様に行う。
