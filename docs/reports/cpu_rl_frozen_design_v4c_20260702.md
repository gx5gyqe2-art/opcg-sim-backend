## 1. 探索・データ生成: Playout Cap Randomization（PCR・KataGo流）
- 「自己対戦80固定/評価160」案は**棄却**（80の訪問分布は浅いヒューリスティクスの劣化再生産・
  訓練/評価の探索深さ不一致で value が狂う）。
- **採用**: 自己対戦の各手を確率的に2種へ振り分け:
  - **fast 手（~80-85%）**: 低 sims（25）で高速に打つ＝データ生成速度を稼ぐ。
  - **full 手（~15-20%）**: フル sims（160・評価と同一）で読む。
- **full/fast の選択は手ごとに一様ランダム（Bernoulli・盤面非依存）**。盤面の複雑さ等での
  トリガーは**禁止**（policy 教師の分布が「複雑な局面」へ条件付けされて偏る＝日常局面への盲目、
  かつネット依存の選択はフィードバック相関で分布が漂流する）。
  例外は「合法手1つの局面は policy 教師を記録しない」（分布が自明で無情報）のみ。
- **探索ノイズ（ルート Dirichlet・温度）は full 手のみ有効**。fast 手はノイズ無しで淡々と進める
  （fast 手は教師にならないので探索多様化の意味がなく、速さと軌跡の質を優先）。

## 2. 教師データ設計（信用割当の正しい扱い）
- **Policy 教師**: **full 手の訪問分布のみ**（fast 手は Loss から完全マスク）。
  ※ L1模倣 warm-start を prior に使うのは禁止（悪化を実測済み・v4b⑦）。gen0 は uniform prior。
- **Value 教師**: **軌跡上の全局面を同列（重み1.0）**・ターゲットは最終結果 z∈{-1,+1}。
  - 私案「fast 局面を低重み」は**撤回**（レビュー反証: ①浅い手の大悪手は z を通じ軌跡全体を汚染
    するため局面重みでは浄化不能 ②80%の日常局面を捨てると基礎盤面評価が盲目化 ③PCR の増産×
    大数の法則がノイズを自然浄化し、人為重みはむしろ収束を妨害）。
  - **contingency**: z ノイズで収束しない場合は、局面重みでなく**ターゲットを z と V_root の
    ハイブリッド（TD 的）**へ変更する（これが正しい RL 的対処）。
- 補助損失（パワー差/ライフ差/1ターン巨大スイング予測）・L1 への KL アンカーは v4b どおり。

## 3. 評価・合格ライン（ペア単位検定）
- **1戦ごとの SPRT は禁止**（先手勝率がデッキ毎 0.17〜0.53 で振れ、i.i.d. 前提が崩壊）。
- **採用**: 対局は必ず**「先手1戦＋後手1戦」の2戦1ペア**で実施・集計（ペア内で先手バイアスを相殺）。
  ペア結果（2勝/1勝1敗/2敗）に対する尤度比検定。
- **合格ライン**:
  - 主目標: **平均勝率 ≥ 0.60**（SPRT: H0 p≤0.50 / H1 p≥0.60）。0.55 は「誤差/上振れ」の域で棄却。
  - 壊滅足切り: min≥0.5 は**撤廃**（1デッキの構造的微不利で世代更新が永久ブロック）→
    **各デッキ勝率の 95%CI 下端 ≥ 0.40**（真の崩壊 0.2 級のみ弾き、微不利 0.45 は許容）。
- 相手・held-out・席替え・リーク防止・カナリアは v4b どおり（最終ゲートは製品強度 L1）。

## 4. 本走ループ構成（確定）
- gen0: value=L1評価 warm-start（初期化のみ）・prior=uniform。
- 各世代: PCR 自己対戦（生成デッキ・公平化決定化）→ policy を full 手訪問分布で学習・value を z で学習
  → ペア単位ゲートで測定 → checkpoint（悪化なら rollback）。
- 停止: held-out 勝率の平坦化（目安3〜5世代）。合格＝§3。本番差し替えは別途 trace 目視＋Gen2 fallback 維持。
