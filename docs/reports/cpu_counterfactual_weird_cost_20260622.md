# CPU「変な手」反実仮想（手の差し替え）測定 報告（2026-06-22）

> 計測報告（点）。Phase 0 監査（`cpu_weird_move_audit.py`）の**非ペア**結果ラベルが本物率≒基準線50%
> （②自殺56%/n9・①53%/n184・③50%/n135）＝ほぼ無相関だった件を、**ペア（同一局面からの手差し替え）**で
> 因果的に確証する。計画は `cpu_weird_move_remediation_plan_20260622.md`、2a 不発の経緯は
> `cpu_value_blend_hard_ab_20260622.md` を参照。観測専用（評価・探索・パッチ・`value_model.json` 不変）。

## 1. 目的

「CPU の変な手は実際に勝敗を損ねているか？」を因果的に厳密判定する。非ペアの結果ラベルは「変な手以降を
完走したら負けたか」しか見ず、**その手のせいか**（局面がもともと不利だっただけか）を切り分けられない。本測定は
**同一局面 D から「変な手」と「最善の非変な手」の両方を打って終局させ、勝率差 Δ を取る**ことで、手の差し替えの
純効果を測る。

## 2. 反実仮想の設計

新規ツール `tests/cpu_counterfactual.py`（dev 専用・観測専用）。決定論自己対戦（hard・本番同一の
`decide_guarded`）を seed 範囲で回し、各局で `classify_decision`（監査と同一判定）が①②③のいずれかでフラグした
**変な手の決定点 D** を捕捉。各 D で同一状態のクローンを 2 つ作り、ペア比較する。

- **baseline 枝**: D のクローンに **CPU が実際に選んだ手（＝変な手）** を打ち、`_play_to_finish`（既定方策
  `decide_guarded`）で終局。手番側 P が勝てば win=1。
- **反実仮想枝**: D のクローンに **「最善の非変な手」** を打って終局。P が勝てば win=1。
- **差し替え基準（「最善の非変な手」）**: D の合法手のうち `classify_decision` に掛けても**いずれの変な手
  カテゴリ（①②③）にもフラグされない**手だけを候補にし、`_eval_move_settled`（相手 MAIN まで整流採点・監査の
  settle 評価と同一）が最大の手を選ぶ。**do-nothing（TURN_END）も候補に含む**ため「無理に動かない」も対抗手に
  なる（自殺攻撃の代替が「ターンを畳む」になるケースが多い）。全候補がフラグされる稀な D はスキップ。
- **RNG 制御（公平性の担保）**:
  - クローンは `GameManager.clone()`（deepcopy）で完全独立＝両枝が相互汚染しない。
  - `_play_to_finish` 内の `decide_guarded` は global `random` を消費する（タイブレークの `rng.shuffle`）。
    両枝の終局直前に **D ごとの同一固定 seed で global `random` を seed** するため、D 以降の downstream RNG 列が
    両枝で一致する（同じ seed 起点・同じ既定方策）。手適用は決定論なので「適用後に seed」で D 以降が同一列になる。
  - 差は「D で打った 1 手」だけ＝手の差し替えの純効果を測る。終局しなかった枝（None）はペアを計上しない
    （片側だけ完走すると不公平）。

実行（4 コアで seed を 0–99 に 25 ずつシャードし前景並列・1 局ごとに JSON フラッシュ。1 局あたりの
反実仮想は先頭 1 件＝②自殺優先で採取し 1 D=2 終局ロールアウトのコストを抑制）:

```bash
OPCG_LOG_SILENT=1 python tests/cpu_counterfactual.py --games 25 --seed <0|25|50|75> \
    --max-flags-per-game 1 --json /tmp/cf_sh<N>.json --progress
```

## 3. 数値（30 局完了・ペア 30・うち終局せず除外 0）

hard（cheat・info_policy=hard）。1 局 ~150s と重く、4 シャード前景並列で時間内に **30/100 局**完了。各シャードは
1 局ごとに JSON をフラッシュしたため部分でも回収可能（規模＝部分・小標本である点に留意）。

| 母集団 | 標本 n | baseline 勝率 | CF 勝率 | Δ=CF−base | 95%CI(Δ, 独立2標本正規近似) |
|---|---:|---:|---:|---:|---|
| 全体（①②③いずれか） | 30 | 0.400 | 0.367 | **−0.033** | [−0.279, +0.213] |
| ①差≤0で行動 | 30 | 0.400 | 0.367 | **−0.033** | [−0.279, +0.213] |
| ②自殺攻撃 | 12 | 0.500 | 0.417 | **−0.083** | [−0.481, +0.314] |
| ③無駄ドン | 8 | 0.375 | 0.500 | **+0.125** | [−0.357, +0.607] |

（注: この測定では母集団タグ「全体」と「①」がほぼ一致＝採取した D の大半が①フラグを伴うため。②③は①と
重複フラグで計上される。）

**ペア一致率（base と cf で勝敗が同じか）**:

| カテゴリ | n | 勝敗一致(base==cf) | CF が良化 | CF が悪化 |
|---|---:|---:|---:|---:|
| 全体（30ペア） | 30 | 23 (77%) | 3 | 4 |
| ②自殺攻撃 | 12 | 11 (92%) | 0 | 1 |
| ③無駄ドン | 8 | 5 | 2 | 1 |

②自殺攻撃 12 ペアのうち **11/12 が同一結果**。唯一の不一致（seed=26 t9 p1・regret=−400）はむしろ
**変な手のほうが勝った**側（base=1, cf=0）。②の代替手は「TURN_END で畳む」が多いが、それでも勝率は上がらない。

## 4. 結論

- **全体・カテゴリ別とも Δ≒0**（−0.083〜+0.125）で、いずれの 95%CI も 0 を広くまたぐ＝**統計的に Δ>0 とは
  言えない**。30 ペア中 23（77%）で勝敗が一致し、不一致の 7 ペアも良化 3／悪化 4 とほぼ拮抗。
- **②自殺攻撃**（本来いちばん「損していそう」な症状）でも、**12 ペア中 11 で結果が同じ・唯一の差は変な手有利**
  ＝差し替えても勝率は上がらない。「最善の非変な手（多くは TURN_END）に変えても勝てない」＝この自殺攻撃は
  **すでに敗勢の局面で出ている見栄えの問題**であって、勝敗の原因ではない可能性が強い。
- ③無駄ドンは +0.125 と符号は良化側だが n=8・CI が大きく 0 を含む＝ノイズ域。

**＝ペア比較は Phase 0 の非ペア結果（本物率≒50%・無相関）を裏づけた。** 監査がフラグする「変な手」は
**因果的にはほぼ無害**（勝敗を有意に損なっていない）。したがって、これらを直す価値は **強さの向上ではなく
「見栄え・納得感」の改善**にある。強化（勝率）目的でこの3症状を潰すことは、本測定の範囲では正当化されない。

## 5. 統計的限界

- **標本が小さい**: 完走 30 局・全体ペア 30・②自殺 12・③無駄ドン 8。Δ の CI は ±0.25〜0.6 と広く、**小〜中の
  真の効果（例 Δ=+0.1）を検出する検出力は無い**。「Δ>0 を否定」ではなく「30 ペアでは Δ>0 の証拠が出なかった・
  少なくとも大きな損ではない」と読むべき。確証には数百ペア（特に②は seed を大幅に広げて n を 3〜5 倍）が必要。
- CI は独立2標本比例の正規近似（ペア相関を使わない保守的＝実際の対比較 SE はこれより小さい可能性）。厳密には
  McNemar 検定（discordant ペアのみ）が適切だが、本測定の discordant は全体で 7・②で 1 と極小で検定力が無い。
- **情報方針**: cheat（hard・相手手札透視）で測定。フェア化（`--fair`）での再測は未実施。`cpu_cheat_carveout_ab`
  の知見では②自殺はカンニング無関係（eval_gap）なので結論は大きく変わらない見込みだが未確認。
- 1 D あたり 2 終局ロールアウトと重く、本ランは時間内 30 局で打ち切り（部分）。残 70 局・フェア再測・②の標本
  拡張は将来ランで積み増し可能（ツールは seed シャード＋毎局フラッシュで再開可能）。

## 6. 成果物・判断に迷った点

- **作成した報告**: `docs/reports/cpu_counterfactual_weird_cost_20260622.md`（本ファイル）。
- **作成したツール**: `tests/cpu_counterfactual.py`（dev 専用・観測専用・新規）。評価/探索/パッチ/`value_model.json`
  は一切変更していない。
- **判断に迷った点**:
  1. **④届かないカウンターを対象外**にした。④は「カウンターステップを締める PASS」で判定する戦闘単位の指標で、
     「同一状態から別の手を打って終局」というペア差し替えの枠（メイン手の選択肢集合）に乗りにくいため、本ツールは
     ①②③（メイン手の差し替え）に集中した。④の因果評価は別設計（カウンター札を切らない反実仮想）が要る。
  2. **「最善の非変な手」の採点を `_eval_move_settled`（settle）にした**。監査の①③判定と同じ物差しで「非変な手の
     中で最良」を選ぶ整合性を優先。settle は準備手（ATTACH_DON/PLAY）を構造的に過小評価する既知のクセがあるが、
     ここでは「差し替え先が変な手にならない」ことが主目的で、最良選択の僅差は終局ロールアウトが吸収する。
  3. **RNG 公平化を「手適用後に同一 seed で global random を seed」で実装**した。手適用自体は決定論なので、適用後に
     seed すれば D 以降の RNG 列が両枝で一致する。これにより両枝の差を「D の 1 手」だけに限定できる。
