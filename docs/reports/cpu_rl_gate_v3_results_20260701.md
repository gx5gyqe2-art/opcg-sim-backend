## 訂正（先行文書の誤り）
v4a/v4b/ゲート初測定の文書で「黒ひげ＝登場時効果無効メタ」と記述したが、**held-out デッキの
リーダー OP16-080 は登場時無効を持たない**（それは OP09-081 の別ティーチ）。OP16-080 の実効果:
①【相手のターン中】自分のキャラ全てコスト+1（防御パッシブ）②【相手のアタック時】【ターン1回】
手札のトリガー持ちを1枚捨ててアタック対象をリーダー/黒ひげキャラへ変更（リダイレクト防御）。
外部レビュアーの例示を検証せず取り込んだ私の誤り（ユーザ指摘で発覚）。**このデッキの本質は
「防御リソースの経済学」**であり、encode_v3 の特徴設計はこの訂正済み理解に基づく。

## 変更（encode_v2 → v3・+14次元）
エンジン計算の決定的な現在状態を注入: **実効コスト current_cost**（コスト+1パッシブが初めて可視）/
場の動的集約（現在パワー計・レスト数・付与ドン）/ **防御リソース**（自手札トリガー持ち枚数・
今出せる枚数・リーダー防御可用・相手の脅威）/ 登場時無効フラグ（一般特徴）。全て公開情報＋自分の手札由来。

## 解釈（正直に）
1. **狙った欠損（黒ひげ）は両seedで解消**: 0.10/0.39 → **0.45/0.48**＝ベースライン(0.42)同等以上に
   両seedで到達。「防御リソースが見える入力」への修正が、局在欠損を狙い通り動かした。
   キャンペーン開始時点の「実黒ひげで壊滅的盲目」から、bootstrap のみのパイプラインで
   L1 同等操縦まで来た。
2. **hancock も改善**（0.54/0.74 → 0.88/0.60・s0 は SPRT PASS）。
3. **nami は悪化が疑われる**（0.52/0.33 → 0.39/0.10）。s0 はベースライン(0.17)超だが s1(0.10・n=10
   早期FAIL)は下回る。SPRT 早期打切りの小標本ゆえ分散も大きいが、**v3 が nami に有害な可能性は
   否定できない**（特徴追加でL1模倣回帰の焦点が移った可能性）。過大評価しない。
4. 集計: **両seedでベースライン以上 = blackbeard・hancock の 2/3**（v2 では nami・hancock の2/3）。
   「どのデッキが残るか」が v2→v3 で入れ替わっており、単一 bootstrap 値ネットの分散も示唆。

## 次の含意
- 黒ひげ層の欠損解消は確認。**残る不安定性（nami の seed 依存・全体の分散）**は、
  次のレイヤ＝ **policy prior（Early Fusion・探索効率）** と **outcome self-play（L1模倣の天井超え・
  複数世代の安定化）**、および評価側の games 増で扱う段階。
- 表現（encoder）は v3 で「説明書＋今の盤面」が揃い、凍結設計の encoder 要素はほぼ実装済み。
