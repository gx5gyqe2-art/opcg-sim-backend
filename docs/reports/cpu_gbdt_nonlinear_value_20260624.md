# 計測: 学習価値の非線形化（GBDT）= 実装済み・但し現データ量では線形に未達（2026-06-24）

> 計測スナップショット（改変禁止）。天井上げ（学習価値を葉評価の中心に育てる）の第1段＝**モデルの容量を
> 上げる（線形→非線形 GBDT）**。pure-Python 勾配ブースティング木を実装し、fair 多シードデータで線形と比較。

## 実装

- `tests/train_gbdt.py`（新規・stdlib-only）: XGBoost 風の勾配ブースティング木（二値ロジ損失・gain 分割
  `GL²/(HL+λ)+GR²/(HR+λ)-G²/(H+λ)`・Newton 葉値 `G/(H+λ)`・特徴ごとの分位点候補で訓練を高速化）。
  `value_model.json` 互換の **gbdt-v1** 形式（木の入れ子 dict）を出力。
- `cpu_value_model.py`: gbdt-v1 推論（木の走査＋sigmoid）を追加。線形（logreg-standardized-v1）と
  両対応。**α=0 既定で推論は呼ばれず＝出荷挙動不変**（葉ブレンドは休眠）。
- 推論の単体テスト＝`tests/test_cpu_value_model.py::test_gbdt_tree_predict_and_format`。

## 結果（fair 多シード 2110 行・8:2 ホールドアウト）

| モデル | train acc | **val acc** | val logloss |
|---|---:|---:|---:|
| 線形（3a・現行同梱） | — | **0.725** | **0.537** |
| GBDT trees=120 depth=3 | 0.871 | 0.659 | 0.604 |
| GBDT trees=250 depth=3 lr=.05 | 0.874 | 0.668 | 0.595 |
| GBDT trees=120 depth=4 | 0.908 | 0.661 | 0.604 |
| GBDT trees=60 depth=2 minleaf60 λ8（強正則化） | 0.777 | 0.678 | 0.608 |
| GBDT trees=40 depth=2 minleaf100 λ10（最強正則化） | 0.758 | 0.661 | 0.605 |

## 判定

**全 GBDT 構成が val で線形(0.725)に未達**。train-val ギャップ大（0.87/0.66）＝典型的な**過学習**。強正則化で
ギャップは縮むが val は ~0.68 で頭打ち。**2110 行は GBDT には少なすぎる**（高分散モデルは小データで過学習・
高バイアスの線形が小データでは汎化で勝つ）。

→ **非線形の天井はモデル容量でなくデータ量が律速**。GBDT のコード/推論は正しく動く（基盤は完成）が、
現データ量では**採用しない**（`value_model.json` は線形 3a を維持）。

## 含意・次

- 非線形価値を活かすには**データの大幅拡充が前提**（fer 大量自己対戦＝この環境では長時間／**人間ログ**＝
  実戦分布のカバー）。WBS 将来項「人間ログ活用」が天井上げの本丸として効いてくる。
- GBDT 基盤は**データが揃った時点で即学習可能**な状態で同梱（`train_gbdt.py`＋gbdt-v1 推論）。
- 留保: 2110 行・単一 seed 群。データを 10x 級に増やせば GBDT が線形を超える可能性は残る（未検証＝要 compute）。
