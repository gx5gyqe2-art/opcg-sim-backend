# pre-flight ④b: 0/40 崩壊の機序トレース＝「バグでなく分布シフト」を確定

日付: 2026-07-01 / 対象: `cpu_rl_preflight4_mcts_results_20260701.md` の held-out(黄) 0/40 崩壊
コード: `tests/pre_flight4b_trace.py`
契機: 外部レビューの指摘「0/40 が スカラーOOD なのか トリガー期待値バグ なのか切り分け不能」

> スナップショット（改変しない）。憶測でなく実測で機序を確定する。

## 診断（net0＝encoder_v2 を L1評価で bootstrap・非黄で訓練）
- **A) value キャリブレーション**: net の予測が L1評価とどれだけ相関するか・異常値/NaN・life レンジ（黄 vs 非黄）。
- **B) 敗戦ゲームの value 軌跡**: 黄の負け試合で value が turn0 から張り付くか／盤面に反応するか。
- **C) 非対称性コントロール**: 黄で greedy-L1(p1) vs greedy-L1(p2) の先手勝率（交絡チェック）。

## 結果（2seed）
### A) キャリブレーション
| | 非黄(train分布) corr(net,L1) | 黄(held-out) corr(net,L1) | NaN/inf | life>5 |
|---|---|---|---|---|
| seed0 | **+0.849** | **+0.069** | 0 | 黄 3/474 |
| seed1 | **+0.813** | **−0.025** | 0 | 黄 11/642 |

→ **net は非黄では L1評価をよく予測（corr≈0.83）／黄では相関がゼロに崩壊（≈0）＝黄で value は“盲目”**。
NaN/inf 皆無（数値バグでない）。life>5 は黄で出るが少数（数%）＝主因でない。

### B) 敗戦ゲームの value 軌跡（黄）
value は **ほぼ一定・盤面非感応**（例: seed0 敗戦1 は life 5/4→0/4 の全過程で v=-0.91 固定／seed1 は
life 5/5→0/5 で v=-0.77 固定）。「turn0 から -1 に張り付く（諦める）」でも「過信」でもなく、
**盤面に反応しない≒ノイズ的定数**を出している（＝corr≈0 と整合）。

### C) 非対称性コントロール（黄・greedy-L1 ミラー）
先手 p1 勝率 = **0.567 / 0.500**（≈0.5）→ **盤面の先手/エンジン由来の交絡なし**。0/40 は学習 value の責。

## 確定した機序（バグ vs 分布シフト）
- **数値バグではない**（NaN/inf=0・エンコード/符号は健全）。
- **トリガー期待値バグでもない**（軌跡は特定イベントで崩れず、最初から盤面非感応）。
- **ハーネス交絡でもない**（コントロール≈0.5）。
- **＝真正の分布シフト（OOD）**: net は**黄を一度も学習していない**ため、黄の局面で value が
  L1評価と無相関の“盲目”状態になり、MCTS は無意味な value で探索して全敗する。
- **レビューのlife>5仮説は「存在するが少数＝主因でない」**。崩壊は単一スカラーでなく
  **value 関数全体が黄で OOD**（fingerprint→value の写像が黄領域で未訓練）という広い失敗。

## 含意（計画への反映）
1. **②（被覆）への投資は安全＝正当**: 崩壊は「黄を学習していないから」。黄の特徴領域を訓練分布に
   入れれば value が黄でも信号を持つはず。→ **pre-flight⑤（ミューテーションデッキで黄被覆→MCTS vs L1
   held-out が回復するか）** が正しい次の一手と確定。
2. **以前の私の見立て「value が -1 に張り付く（諦める）」は誤り**→正しくは**盤面非感応（corr≈0）**。
3. **信頼できるゲート(vs L1)の重要性を再確認**: R² は非黄で高い(0.66)のに vs L1 は 0/40。
   キャリブレーション(corr)も vs L1 勝率も、黄では揃って崩れる＝OOD の一貫像。
