# v11 実験報告 — 子盤面ラベルの失敗と教師過圧の発見（2026-07-23）

gen6 採用後の次イテレーション（v11）の実験記録。**採用候補は出なかった**（コーチ PASS 候補が
アリーナで敗退）。ただし複数の決定的な切り分けが得られた。スナップショット（追記・改変しない）。

## 1. 背景と仮説

gen6 の残課題 @68/@93 を 1-ply 分解した結果、「value 教師が root 決定点にしか付かず、decide が
比較する**初手後の子盤面**の序列を教えていない」を根本原因と仮説化（@68=TURN_END 子盤面の
過大評価 +0.64・@93=全子盤面フラットの実測）。対策として**子盤面ラベル**（プラン評価が既に持つ
初手族ベスト勝率を子盤面の value 教師化・referee_labeler の child_* 配列）を実装した。

## 2. 実験結果（コーチゲート・基準 gen6=3.8）

| 条件 | chall計 | 備考 |
|---|---|---|
| 子盤面 全量(2355) | 2.0 | @64/@137 崩壊 |
| 子盤面 25% | 1.8 | 量を減らしても毒 |
| 子盤面 pass-only(481) | 2.4 | TURN_END 系限定でも毒（@82 教師と衝突） |
| **子盤面なし（対照）** | **3.4** | @82=1.0・@93=0.6 が新規獲得（真の立役者は root 教師） |

**子盤面ラベルは全変種で有害＝全面撤退**。毒性の機構: (a) 攻撃/付与系の子盤面に「最善進行前提」の
楽観値が付く、(b) pass 系child は飽和負け局面由来の負ラベルが @82 型の「PASS が正解」root 教師と
正面衝突する。学習側の既定は `--child-frac 0`（生成側の child_* 配列は無害なので存続・読み手 opt-in）。

## 3. 真の発見①: @82/@93 を解いたのは root 反例教師の自然増

一晩で version5 教師 719→1,980・disagree 367→1,390 に成長した結果、これまで一度も動かなかった
@82（カウンター温存）=1.0・@93（展開）=0.8 が初めて成立（sd=0.9・子盤面なし）。
**データの中身（v10 特徴つき反例）が閾値を超えれば動く**ことが実証された。

## 4. 真の発見②: disagree×4 の過圧

disagree が倍増したのに複製倍率 ×4 のままだと実効圧が過熱し、@64/@137 を系統的に破壊
（sd 0.85〜0.95 の全域で 0.0＝アンカーで守れない）。**×2 へ冷却で @64=1.0 に完全回復**（chall 4.6）。
教師量に対して複製倍率は再調整が必要＝固定値でなく比率で考えるべきノブ。

## 5. コーチ PASS 候補とアリーナ敗退

sd=0.95・dw=2・ep8・lr5e-5（子盤面なし）で **コーチゲート PASS**（chall 4.4 vs 3.8・
@33/@64/@93/@115/@137 の5点・非退行ゼロ＝歴代最高プロファイル）。
しかし**アリーナ stage1 で 9/24（0.375）reject**＝実戦では gen6 に明確に負けた。

- コーチゲートは7つの検証局面のプロファイルであり、実戦の総合力を保証しない（3ゲート制が
  設計どおり機能した）
- マーク特化の獲得が大局の判断を歪めた可能性。gate 規則に従い再測定はしない（p-hacking 回避）

## 6. 結論と次

- **gen6 のまま**（採用なし）。教師蓄積（5枝・v5/child 生産中）は継続
- 次の再挑戦の材料: (1) v5 教師がさらに増えた時点で dw を比率ベースで再調整して再学習、
  (2) コーチ↔アリーナ乖離の診断（マーク外の実戦局面で候補が gen6 より悪い決定の採掘）、
  (3) @68（付与→リーサル）は依然未解決＝TURN_END 過大評価の矯正は子盤面ラベル以外の設計が必要
- 計器の教訓: 対照実験（frac=0）無しでは「子盤面が @82/@93 を動かした」と誤読していた。
  **新要素と自然増を必ず分離**する
