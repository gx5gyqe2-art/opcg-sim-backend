# 凍結設計 v4 補遺（v4a）: レビュー第6巡の反映（隠匿プール要約・二部サンプリング・機序ペア）

日付: 2026-07-01 / `cpu_rl_frozen_design_v4_20260701.md` への補遺（v4本文は変更せず本書で修正を記録）。
契機: 外部敵対レビュー第6巡（「色の完全退場」の見落とし指摘ほか）。

> スナップショット（改変しない）。

## 1. 点1「色退場で相手の隠匿領域への推論を喪失」→ 意図採用・実装は色でなく“プール要約”で
**指摘**: デッキはリーダー色に従うため、相手リーダーの色は「相手の手札/ライフに何が潜み得るか」の
事前分布。完全退場するとリーサル踏み込みやカウンターケアの期待値計算が崩壊。対案=リーダー色
マルチホットだけグローバルに残せ。

**採用形（より強い形で意図を満たす）**: 色は戻さず、**エンジン注入の「相手隠匿プール挙動要約」**を
グローバル特徴に追加する:
- 母集団 = 相手の 手札+山札 プール（**PIMC の決定化が既に使っている集合**＝`_determinize_opponent`
  が再サンプルする same pool。新たな情報チャネルを開かない）。
- 要約 = プールの集約 fingerprint 統計（カウンター密度・トリガー密度・効果カテゴリ配合・コストカーブ）。
- 論理: 色の情報価値の実体は「プール制限」そのもの。色はその粗い代理に過ぎず、プール要約は
  **色より情報量が多く、色を介さない**（＝shortcut 軸を再導入しない）。効果の色越境が進む
  デザイン潮流（ユーザ観察）にも、色でなく実プールを追うため自動追従。
- 万一プール要約が強すぎ/ノイジーなら、リーダー色マルチホットへ縮退する A/B をゲートで実施可。

## 2. 点2「members∪referencers の単一袋は構造的欠陥デッキを量産」→ 採用
Trait集中度 λ に加え、**構成比 α（members : referencers）の二部サンプリング**を導入。
「枠の α 割を members(T) から、(1-α) 割を referencers(T) から」引く＝シナジーの発動側/受け側の
バランス崩壊を防ぐ。α もパラメータ空間の一軸として振る。

## 3. 点3「Trait参照だけではメカニクス由来コンボが漏れる」→ 採用
コアパッケージ抽出に**機序ペア（Action-Condition Pair）インデックス**を追加:
- 例: コスト操作(COST_BUFF/SET_COST) ↔ コスト閾値参照効果(TargetQuery.cost_max 等)（黒の
  コストマイナス→コスト参照KO）、パワー低下 ↔ パワー閾値参照、レスト付与 ↔ レスト状態参照。
- パーサの構造化出力（ActionType×TargetQuery のスカラー条件）から機械抽出できる。

## 4. 閉め質問「PIMC の決定化母集団は？」への回答（実装確認済み）
`cpu_ai._determinize_opponent`（cpu_ai.py:1326）: **相手の 手札+山札（そのゲームの実ライブラリ）を
シャッフルし、手札枚数分を新手札に差し替える**。全カードプールの一様分布ではない。
- 生成デッキで訓練しても、実デッキゲートでは決定化は自動的に「その実デッキ」から引く＝
  **決定化にドメインギャップは無い**。ギャップが乗るのは value の訓練分布のみ（本キャンペーンの主題）。
- ライフは再サンプルせず据え置き（L1 の PIMC と同一の既知の簡略化。本キャンペーンの対象外）。

## 5. v4 からの差分まとめ
- グローバル特徴に「相手隠匿プール挙動要約」を追加（色は戻さない）。
- 生成器: λ に加え α（二部サンプリング）。コアパッケージは Trait両側 + 機序ペアの2系統インデックス。
- その他（ゲート・監査・補助損失・実行順）は v4 のまま。
