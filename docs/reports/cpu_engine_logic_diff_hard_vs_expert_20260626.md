# CPU 思考ロジックの差異まとめ: hard(α-β) vs expert(MCTS) — 2026-06-26

本番に出している2系統の CPU、`hard`（α-βミニマックス）と `expert`（マクロ MCTS）の**思考ロジックの違い**を
まとめる。背景: 本番構成（hard=PIMC K=4）で測り直した結果 **expert は hard より約 −211 Elo 弱い**ことが判明
（`docs/reports/` 同日の計測／チャット）。なぜ差がつくかは下表の探索構造の違いに帰着する。

正本の実装: `opcg_sim/src/core/cpu_ai.py`（hard）／`opcg_sim/src/core/cpu_mcts.py`（expert）。仕様は `docs/SPEC.md §2.5.2`（α-β）/`§2.5.7`（MCTS）。

## 1. 一言で

- **hard = 厳密な先読み**。「全部の良さげな手を、相手の最善応手込みで、4ターン先まで**漏れなく**読む」。α-β カットで値は厳密。
- **expert = サンプリングで当たりをつける**。「ターン丸ごとを1エッジにして、**確率的に**有望そうな手順を引いて2ターン先まで**ざっくり**読む」。訪問数で決める。

## 2. 構造の対比表

| 観点 | hard（α-β＋ビーム） | expert（マクロ MCTS） |
|---|---|---|
| アルゴリズム | αβ ミニマックス（厳密・カットで枝刈り） | UCB ベース MCTS（確率的・訪問数で選択） |
| ノード/エッジの粒度 | **1手**（move 単位の木） | **1ターン丸ごと**（turn 境界＝ノード／1ターン分の手列＝エッジ） |
| 先読み深さ | **4ターン**（`HARD_HORIZON=4`） | **2ターン**（`OPCG_MCTS_HORIZON=2`・大盤面発散抑制） |
| 探索幅 | ビーム（自分 `HARD_BEAM=3`／相手 `HARD_OPP_BEAM=4`）＋ルート深掘り `HARD_ROOT_BEAM=4` | progressive widening＋1-ply ソフトマックス（温度 `MCTS_MACRO_TEMP`）でターン候補を生成 |
| 計算量の決め方 | 深掘り1手あたり clone 予算（本番 `予算75×PIMC4≈300`） | **総反復 160**（`MCTS_MACRO_ITERS`）＋壁時計 2000ms 安全弁 |
| 値の質 | **厳密**（αβ は最善応手を仮定した真のミニマックス値・カットは値を変えない） | **近似**（サンプル平均＝訪問が少ない手順は粗い） |
| 隠れ情報の扱い | **PIMC K=4**（相手伏せ札を4通り想像→評価平均＝決定化アンサンブル） | **determinize 1世界**（相手伏せ札を1通り再サンプル・`worlds=1`） |
| 勝ち筋プラン | **自デッキplan供給**（aggro/control 分類で重み・逆算リーサル等・§2.5.5） | **plan 無し**（`_plan_segment` が expert に plan を渡さない） |
| 葉評価 | `evaluate`（ターン境界・整流済み） | `_value_boundary`（同じ `evaluate`＋任意で学習価値ブレンド・既定OFF） |
| リーサル認識 | ply 割引で**最短の詰み**を厳密に拾う | 葉の評価値に乗るのみ（厳密な詰み手順の保証は無い） |
| 決定性 | 固定（αβ＋同seed＝同手） | 固定反復なら再現可・壁時計デッドライン使用時は非決定 |
| 高速化 | make/unmake ジャーナリング・PV/killer 順序・**PyPy ワーカー**（~2.1x） | CPython **インプロセス**（PyPy 非使用）・計画キャッシュ＋ポンダリング |
| 出力 | 1手ずつ（`decide`） | **ターンプランを一括**生成→queue から即時 replay（`decide_mcts_macro`） |

## 3. なぜ hard が強いか（差の主因）

1. **深さ 4 vs 2**: hard は2ターンぶん余計に先を読む。テンポ/リーサル/守りの見通しが段違い。
2. **厳密 vs サンプリング**: αβ は「相手の最善応手」を仮定した真の値を返し、**精密な詰み・反証手順**を取りこぼさない。MCTS は160反復のサンプルなので、**鋭い一本道の手順を引き当て損ねる**ことがある（マクロ粒度でも本質は同じ）。
3. **隠れ情報 K=4 vs 1**: hard は相手の伏せ札を4通り平均＝1通り決め打ちの expert より**分散が小さく堅実**。
4. **plan の有無**: hard は自デッキの勝ち筋（逆算リーサル/温存方針）で評価が方向付けられる。expert は素の葉評価のみ。

→ 深さ・厳密さ・隠れ情報処理・plan の**4点すべてで hard が上**。PIMC（K=4・+53Elo）を hard に足したことで差が決定的になった。

## 4. expert が原理的に有利になり得る点（将来の伸びしろ）

- **マクロ粒度＝葉が常にターン境界**＝甘い途中局面を評価しない（整流不要）。深さを伸ばすほど clone 効率が良い。
- **determinize はフェアな隠れ情報処理の正攻法**（worlds を増やせば真の ISMCTS 近似）。
- 反復・horizon・worlds・学習価値ブレンドなど**スケールの余地**がある（ただし現状の −211 を埋めるには大幅強化が要る）。

## 5. 体感（レスポンス）

両者とも本番では1秒 envelope 内に収める設計（hard=PyPy＋plan-cache、expert=デッドライン＋ポンダリング）。
L1葉での同一局面1手レイテンシ実測は別計測（`tests/engine_latency.py`）参照。傾向: 中央値は同程度・**expert はテール（p95/max）が長い**（大盤面でターン計画が膨らむ）。

## 6. 含意

- **強さ目的なら hard が既定であるべき**（expert 既定は「ほぼ互角」前提の名残で、PIMC 追加後に再検証されていなかった）。
- expert は**マクロ MCTS の将来性**（深さ効率・ISMCTS）を残す上級者/実験トラックとして維持し、別途強化する位置づけが妥当。
