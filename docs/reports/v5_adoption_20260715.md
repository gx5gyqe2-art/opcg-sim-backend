## 3. 実装内容の反省（v6 設計の前提・機構レベル）

1. **mark-seed はマークの情報を学習信号に変換できない**: マークは手レベルの教師情報だが、実装は
   「マーク局面から自己対戦を開始する」＝分布の再形成であり、ラベルは勝敗のみ。@93 のような
   「勝率をほぼ変えない効率ミス」は勝敗ラベルの雑音床の下＝**原理的に直らない**（6点連続 0.0 が証拠）。
2. **蒸留とマーク矯正は綱引き**: 蒸留教師（凍結 gen4）は「マークされた誤りを持つネット」なので、
   マーク近傍で蒸留損失とシードデータが正面衝突。@82↔g3@33 の反相関シーソーはこの振動＋
   単一 value net への点制約干渉（計画 §課題(d)）の複合。
3. **符号化 v4 の集約特徴は低信号**: デッキ集計は既存特徴と相関が強く、ゲーム内変動が緩慢で、
   固定デッキプールでは準定数＝損失を改善せず動ける次元（ドリフト通路）を足しただけの疑い。
   マークが指す表現力の穴（決断文脈の判別特徴）とは無関係だった。
4. **温スタート前提では「早期ピーク」は「侵食開始が早い」**: round0 で既に v4 同等の強さから
   出発するため、cum2k の勝ちは「学習の登り」ではなく「継承強度＋序盤の微改善」。以降の学習信号は
   2・3 の注入する矛盾・雑音に負けた。
5. **注意（明示的な点制約損失も同じ罠）**: マークを policy CE / margin 損失で直接教える案は、
   点制約の誤汎化（似た構造でカードが違えば正解が反転する局面へ染み出す）を蒸留より強い経路で
   起こすため不採用。

## 4. 次期（v6）方針

1. **昇格ゲート（最優先）**: learner は candidate を出すだけにし、現行ベストと arena で対戦して
   **有意に勝った場合のみ昇格**（段階式: 24局で明確負け→即棄却、勝ち越し→100局で確定）。
   ピーク検出・退避・停止判断が仕組みに昇格する（今回 cum2k に救われたのは手動退避の偶然）。
2. **深い探索による部分再ラベル（mark-seed の置き換え）**: バッファ局面の一部を 10〜50倍 sims で
   再探索し、その訪問分布を policy ターゲットに上書き（expert iteration）。正解を局面ごとに探索が
   導出するため、**カードが違えば答えも変わる＝汎用性を保ったまま効率ミスを教えられる**。
   人間マークはラベルに使わず、(i) 評価（mark_gate）と (ii) 再ラベル予算のサンプリング優先度のみ。
3. **前提チェック**: 各マークを深探索（sims 1600〜3200）で解けるか分類。解ける→2 の守備範囲。
   解けない→value/表現の問題＝4 へ。
4. **特徴は失敗モード逆算＋事前アブレーション**: 新特徴は分散・相関・予測寄与を計測してから採用。
   集約5特徴も同チェックにかけ、寄与ゼロなら落とす（append-only なので互換可）。
5. **生成と serve の探索設定分離**: 生成側は枝刈りを無効化（刈った枝が学習データから消える
   自己強化盲点を断つ）。
6. **蒸留は廃止**（動機の劣化防止は 1 が構造的に担保）。MARK_SEED_FRAC は 0 に戻す。
7. スモークは「忘却が観測できる長さ」（candidate の arena 推移2点以上）を本走開始条件にする。
