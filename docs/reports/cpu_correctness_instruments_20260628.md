# CPU「正しさ」計器 設計（2026-06-28・外部AIレビュー合意済み・設計クローズ）

> **種別: 報告（点）**。設計合意のスナップショット（改変しない）。実装確定後は正本（SPEC §2.5）へ吸収。
> 前提＝`cpu_strength_plan_20260628.md` §K（自己対戦Eloは鏡均衡で頭打ち＝強化レバー枯渇）。

## 0. 目的の再定義
目的を「自己対戦で勝つ」から **「正しい選択ができる CPU」** へ。自己対戦 Elo は正しさの
**低解像度・鏡相殺・ノイズ床（σ≈0.05@100ペア）** な代理指標で、小・稀少・対称な正しさ改善を取りこぼす
（＝今までの「幽霊」判定の正体）。よって **正しさを直接・高解像度で測る計器群**を作り、自己対戦 Elo は
**検収から除外**する。

## 1. 第一軸：Provable / Heuristic（戦術/戦略の人間軸は棄却）
- **Provable（証明可能）**：終端 WIN/LOSS へ有界手数で到達、または全世界で他手以下（Dominated）。eval 非依存。
- **Heuristic（近似）**：探索が打ち切られ L1 eval で採点される手。「展開 vs 温存」等もここに還元。
- 隠れ情報下の Worst-case 保証はほぼ空（凍結バグ）。よって **Expected（期待値）**＝サンプラ品質に 100% 依存。

## 2. 計器①：Lethal / Survive / Dominated の Regret（連続値・客観 P_true 基準）
- **Regret = max_i P_true(i) − P_true(CPUの手)**。閾値 Pass/Fail は廃止、**Average Regret（CI付き）**で出す。
- **P_true ＝ オフライン全探索の客観勝率**。CPU が見たサンプル世界では採点しない（＝自己採点の循環回避）。
- **スコープ（確定）**：(a) 今ターンのリーサル／(b) 今ターンの生存／(c) Dominated。
  - **(d) 1ターン先のリーサル準備は破棄**：相手の1ターンを丸ごと挟む真の2ターン minimax＝終端のみ評価では
    枝刈りが効かず状態爆発、深さ制限すれば eval 依存で客観崩壊（両立不能）。
- **心臓部＝ターンソルバ**（eval 非依存・全探索の有界 minimax。攻撃側全手×守備側最適防御）。CPU本体と独立。
- **P_true 推定＝層化抽出**：純 MC は W≈2500 必要で非現実的。相手手札を **防御ボトルネック次元
  （カウンター総値・ブロッカー有無）でバケット化**し、各バケットを超幾何確率で重み付け → **W=100〜200**で
  低分散・高精度。

## 3. 計器(e)：被リーサル回避（客観ディフェクト検出器・②側に配置）
- (e)=「自分の死を作る手の回避」は2ターン minimax ではない：**自ターン終了時に渡す盤面**に対し、
  **(a) リーサルソルバを"相手側"として1回発火**するだけ（相手のドン補充+2／ドロー1を層化サンプル）。
  追加コスト＝(a)1回分・eval 非依存・有界。
- 連続 Regret 化は自分の全手順と結合して厄介 → **Dominated と同じ Pass/Fail のディフェクト検出器**として
  ②側に置く：「**回避可能なのに相手へ確定リーサル盤面を渡してターン終了**」＝絶対的ブランダー。
- **実装ガードレール**：相手のドローフェイズの未知1枚（速攻キャラ・除去/火力イベント）のリーサル寄与が高い。
  トラッシュ+公開情報の超幾何で**速攻/火力札バケットを正確に重み付け**してソルバに食わせる。

## 4. 計器②：単調性スイープ（eval 構造バグ・任意局面・新規ロジック不要）
- 無条件に得な摂動のみ（**自キャラの power+1000／ライフ+1／手札+1／アクティブドン+1**）で
  `V(S') ≥ V(S)` を検査。違反＝eval 系統バグ。
- **盤面追加（新キャラ展開）は除外**（過剰展開トリガ等の正当な非単調＝偽陽性源）。違反は**ハード失敗でなく
  トリアージ候補**として抽出。

## 5. 計器③：サンプラ較正 ／ 計器④：非対称ハンデ戦
- ③ 一様サンプラの含意「相手カウンター所持率」が実対局の開示頻度と一致するか（reliability diagram / Brier）。
- ④ CPU に「+1ライフ/+1ドン」を与え、ハンデ無し版への**勝率の跳ね幅**で基礎体力（eval 全体）を測る。
  **＝Elo を捨てた代わりの「正しさ→強さ」を非鏡で確認する最終ゲート**（鏡均衡の唯一の抜け道）。

## 6. データ健全性（計器の嘘を防ぐ）
- **局面選択バイアス**：自己対戦ログは温室（CPU は困難なリーサル局面を回避）→ Regret が誤って≈0 に張り付く。
  **ランダム/アグロ偏重エージェントを混入して盤面を荒らす＋人間終盤盤面を移植＋near-lethal の手続き生成**で
  困難局面を強制カバー。
- **難易度較正＝Greedy/Random エージェントの Regret を基準**に。Greedy 0.20 vs CPU 0.01 なら CPU 優秀／
  Greedy すら 0.01 ならデータセットがゴミ。
- **ターンソルバ完全性（オラクルがバグれば全滅）の3段検証**：
  1. 極小局面（ライフ1・アタッカー1・ブロッカー1・カウンター1）を紙で解いた解析解と100%一致。
  2. **二重実装**：最適化なしの愚直 BFS/DFS と 枝刈り版を小規模乱数局面で fuzz 一致（CI）。
  3. コーナー対称（全パス時に守備リソース非消費で Survive 等）。

## 7. 確定ロードマップ
1. **今すぐ：② 単調性スイープ**（新規ソルバ不要・既存 eval/局面生成で回る）→ eval 構造バグを即抽出。
2. **並行：① ターンソルバ**（(a)(b)(c)・層化抽出MC・二重実装 fuzz 検証）構築 ＋ (e) ディフェクト検出器。
3. **データ較正**：荒らしエージェント混入＋人間終盤盤面＋Greedy 基準 → 本番CPUの **Average Regret** 計測（Go/No-Go）。
4. ③ サンプラ較正 ／ ④ ハンデ戦。
- **自己対戦 Elo は検収から除外**（鏡均衡）。Pass/Fail 閾値 X は不使用（Regret 連続値）。

局面荒らし・Greedy 較正・②先行、の全点合意。「設計フェーズ完全クローズ・実装へ突入」で一致。
