# CPU 評価関数 L2 設計メモ — ホライズン外残差（強さ最優先・L1整合割引）v0（2026-06-25）

> 設計メモ（plan）。L1（コア葉評価・`docs/reports/cpu_eval_redesign_card_currency_20260625.md` v0.4）の**上に積む
> 加算補正レイヤー L2** の設計。確定後 `docs/SPEC.md §2.5` へ吸収する前提のスナップショット。
>
> 前提：L1 がスナップショット（カード単一通貨＋時間割引）を担い、SPSA 第1パスで成熟評価と**互角**（Elo −23）まで
> 到達。L1 は葉スナップショットしか見ないため**互角が天井に近い**。互角を超える強さの主な源泉が L2。
> 方針＝**強さ最優先（A）**／**割引は L1 と整合**（ユーザ決定）。

## 0. 要約

```
eval = L1(leaf) + L2(leaf)
L2   = Σ_term  w_term · value_term(leaf) · D(leaf)
```

- **L2 ＝ L1 への小さな加算補正**。各項は「**ホライズン外で初めて実る潜在価値** × **残ターン割引 D**」。
- **D は L1 の生存状態を再利用**（`T_rem = min(自ライフ/相手clock, 相手ライフ/自clock)`）＝レースで自動 0・グラインドで作動。
- **既定は空（OFF）**。1 項ずつ ablation（単独 ON/OFF A/B ＋「変な手」監査）で**稼いだ項だけ**足す＝キーワードスープへの先祖返り防止。
- 強さ最優先の第1項＝**ブロッカー（仮想ライフ・複数ターン）**。

## 1. L2 の仕事と 3 者の分業

| 担当 | 何を見るか |
|---|---|
| 探索 | ホライズン内（~4T）で決着する戦術＝結果盤面まで計算 |
| L1 | 葉盤面のスナップショット＝単一通貨＋時間割引 |
| **L2** | **どちらも見えない「ホライズン外で初めて実る潜在価値」だけ** |

L2 ＝「ホライズン（~4T）より先で materialize する価値を、そこへ到達する確率で割り引いたもの」。

## 2. L2 の鉄則：二重計上の禁止（W_RECUR_ENGINE の教訓）

L2 設計の**最重要制約**。過去、毎ターン価値を生むエンジンに静的加点（`W_RECUR_ENGINE`）を入れたら、ablation で
**正味マイナス（バイアス）**＝「horizon=4 が将来発動を結果盤面で既に拾う＝二重計上」と判明し撤去された。

→ **L2 の各項は「探索がホライズン内で拾う分を差し引いた、純粋にホライズン外の残差」でなければならない**。
厳密計算は不能なので、L2 は**保守的（小さく）**＋**各項を単独 ON/OFF A/B で検証**し、ノイズに勝つ項だけ残す。
合格条件は W_RECUR_ENGINE 並みに厳しく＝**勝率非劣化＋該当カテゴリの「変な手」非増加**。負けたら即撤去。

## 3. 割引 D（L1 と整合・L1 amp の鏡像）

L1 の生存状態（両者ライフ・clock）をそのまま再利用する：
```
T_rem = min( 自ライフ/(相手clock+ε), 相手ライフ/(自clock+ε) )   # 想定残ターン数（= L1 の γ と同じ量）
D     = clamp( (T_rem − H) / W_d , 0, 1 )                       # H≈ホライズン, W_d=ランプ幅
```
- **D=0**：試合がホライズン内で終わる（レース）＝L2 は効かない（L1 のリーサル判断を邪魔しない）
- **D→1**：長期戦＝潜在価値が実る時間がある＝L2 が効く

L1 の amp（lethal 接近で跳ね上がる＝短期）と **D（残ターンが多いほど大きい＝長期）は鏡像**。
→ L2 は「**レースで自動 0・グラインドで作動**」という正しい性質を構造的に持つ。`my_life/opp_clock` 等を流用＝完全整合。

## 4. アーキテクチャ

- L2 は L1 とは**別の加算関数**＝`eval = L1(leaf) + L2(leaf)`。
- 各項は独立フラグ（`OPCG_EVAL_L2` 配下）で ablation 可能。既定 OFF＝L1 のみ（現状）と完全同値。
- 実装は L1（`cpu_eval_v2`）と同じく stdlib-only・読み取りのみ決定論・四則演算＋clip（1 手 1 秒制約に無影響）。

## 5. 項の優先順（強さ最優先＝Elo 期待の大きい順）

| 順 | 項 | Elo 期待 | 二重計上リスク | 採否 |
|---|---|---|---|---|
| **1** | **ブロッカー（仮想ライフ・複数ターン）** | **大** | 中（L1 Tele が“次の 1 ターン”分は既に拾う） | **まず実装** |
| 2 | KO 耐性（効果耐性「KOされない」） | 中 | **低**（除去は地平線の先・L1 は除去を一切モデル化しない＝きれい） | 次 |
| 3 | 毎ターンエンジン（起動/常時の反復価値） | 大（長期戦） | **高**（W_RECUR_ENGINE で実証済の罠） | 後・厳しく割引 |
| 4 | 将来クロック率（ダブルアタック/アンブロッカブル） | 中 | 中 | 後 |

強さ最優先なので**ブロッカーを第1項**に（最高 Elo・レビュー指摘の「守り過小評価＝特攻」の是正にも直結）。

## 6. 第1項：ブロッカー L2 v0

```
L2_blocker = w_blk · Σ_(自アクティブブロッカー) (パワー/1000) · D
```

**二重計上の切り分け（最重要）**：
- L1 の Tele は「**次の 1 ターン**、自ブロッカーが相手の対自リーサルを縮める」分を**既に**拾っている。
- よって L2 が足すのは「**2 ターン目以降も毎ターン壁になる**」反復防御価値だけ ＝ `D`（残ターン）で割り引かれた残差。
- **ablation 合格条件**：L1（v2）固定で L2_blocker のみ ON/OFF を自己対戦 A/B し、**勝率非劣化 ＋「変な手」非増加**。
  負けたら即撤去（W_RECUR_ENGINE 型の二重計上の再来を警戒）。

> 精緻化の余地（v0 では保留・必要なら後で）：反復防御を「パワー和」でなく「**相手の打点を N 回止める期待**」
> （相手の攻撃本数・有効パワー分布から）で測る。v0 はパワー和×D の軽い近似で始め、ablation の結果で精緻化判断。

## 7. ロールアウトの規律（キーワードスープに戻さない）

L1 同様、**L2 は空から始めて 1 項ずつ稼ぐ**：
1. その項が**厳密にホライズン外**だと論証（探索と二重計上しないこと）。
2. **期待値 × 残ターン割引 D** として表現。
3. **単独 ON/OFF の A/B ＋ 変な手監査**でゲート。
4. ノイズに勝った項だけ採用。負ければ捨てる。勝ったら次の項（KO 耐性）へ。
5. 採用が貯まったら、L2 の重み（`w_blk` 等）も含めて **SPSA で L1 と co-optimize**。

## 8. 検証ゲート（CLAUDE.md 準拠）

- 構造監査 `EXCEPTION/CARD_LOSS/TEMP_LEAK = 0`・カード挙動ベースライン不変（評価変更は挙動に影響しないはずだが確認）。
- 「変な手」Phase0 監査（null-move regret）でカテゴリ別カウンタが**非増加**（特に守り関連）。
- アリーナ A/B で**勝率非劣化**（±35〜58 Elo のノイズ帯を破るペア局数で）。各項を段階導入し、退行は差し戻し。

## 9. 未決事項（実測で詰める）

- **D の形**：`H`（しきい残ターン）・`W_d`（ランプ幅）・`T_rem` の clock 推定の具体値。
- **ブロッカー第1項の formulation**：パワー和×D の近似 vs 「N 回止める期待」。ablation で精緻化判断。
- **項間の相互作用**：ブロッカーと KO 耐性を同時採用したときの二重計上（どちらも「除去/攻撃への耐性」）。
- **co-optimize のタイミング**：各項を個別検証してから SPSA に入れるか、早めに合同最適化するか。
