## 2. 系統課題

### S1. 初版 LCB 読み出し（z=1, min_frac=0.2）の較正不足＝楽観乗り換え退行【本PRで修正】

ターン4の「リーダー4ドン付与」暴走のうち **3手（@20/@22/@23）は LCB 読み出しが argmax(N) から
乗り換えた手**で、@20 の argmax(N) は PLAY OP16-056（ボンクレー）＝人間指摘そのものだった。

根拠（同一ターン内・sticky 世界線＝同一世界での連続 decide の実測）:

| decide | 選んだ手の Q | 次 decide の root value | 楽観ギャップ | 1/√n（z=1の想定誤差） |
|---|---|---|---|---|
| @20 ATTACH q=+0.184 (n=26) | → @21 | −0.359 | **+0.543** | 0.196 |
| @21 ATTACH q=−0.359 (n=70) | → @22 | −0.667 | **+0.308** | 0.120 |
| @22 ATTACH q=−0.667 (n=54) | → @23 | −0.807 | **+0.140** | 0.136 |

低訪問枝の Q は PUCT の**選択バイアス**（高く出た枝ほど追加訪問される→低訪問で高 Q の枝は
「序盤の引きが良かった」枝）で楽観に歪み、その大きさは 1/√n スケールを大きく超える。
z を上げるだけでは @22/@23（訪問比 0.86-0.95・Q差 0.02）型を止められないため、**二重ゲート**へ置換:

> 乗り換えは「① 訪問が競っている（n ≥ 0.4·n_top）」かつ「② Q 差が明確（q ≥ q_top + 0.05）」の
> 代替に限る（該当複数なら最大 Q・`SERVE_ROOT_SWITCH_*`・min_gap=inf で argmax(N) に一致）。

**較正の妥当性＝2局×16マーク全件の回帰**（`test_learned_root_readout.py` に固定化）:
g1 の良い乗り換え2件（@12→ドン付与・@24→波絡み）は維持、g2 の誤乗り換え3件は全て解消、
残り11件は不変。

### S2. 低訪問 Q の楽観バイアス（S1 の根本現象・計測資産化の提案）

上表の「選んだ手の Q → 次 root value」ギャップは、読み出しだけでなく**探索そのものの推定品質**の
問題（160 sims の親推定 26 visits vs 次手番の 160 sims root は後者が信頼できる）。
このギャップは trace から機械抽出できる＝**楽観ランプ計器**（decide 間 TD 誤差）として
蓄積すれば、(a) 読み出しゲートの再較正データ、(b) 学習側（MCTS改善値ラベル・混合損失）の
教師信号になる。第1回 §5-1 の学習ルートへの具体的入力。

### S3. 防御（カウンター）の残課題 — 探索は既に counter を上位に置いている

g2@32 は q_counter(−0.702) > q_pass(−0.739) と**探索の Q 順位は人間指摘と一致**したが、
訪問比 0.23・Q差 0.037 でゲート未満（そもそも初版 LCB でも乗り換えない）。g1@63 の net −0.998
飽和より一歩手前の形＝**net が防御価値を過小評価**し、priors が PASS に寄る（F4）。
読み出しゲートを防御専用に緩めるのは g2@20 型の楽観を招くため採らない。学習側で扱う
（防御局面の value/policy 改善＝マーク回帰ゲートの主要ケースに @32/@63 を含める）。

### S4. ドン付与の net 系統誤差の続き（F4・学習側）

- g2@21: argmax(N) でも付与＝priors/value が leader 付与を過大評価（g1@16 と同型）。
- g2@38: 防御目的のリーダードン（ナミ【ドン!!×1】【相手のアタック時】+2000 の条件を開ける付与）が
  **候補上位にすら現れない**＝policy prior の枯渇（SERVE は dirichlet 無し＝低 prior 枝は訪問 0 近傍）。
  攻め付与の過大評価と防御付与の過小評価が同居＝「付与の価値は文脈（後続プラン）依存」を
  net が表現できていない。EffFeat v3 には HAS_DON 系特徴が存在する＝学習データ/損失の問題。

  PLAY OP16-056 ×9/9（＝実対局の ATTACH は当該世界サンプル＋LCB の合せ技だったことの傍証）。
  @21-23 の付与連打は両版・全シードで再現＝**readout ではなく net/prior 起因（F4）**の帰属が確定。

  退行なし（第1回の同シード実測 0.417 に対し点推定は改善方向・N が小さいため有意差は主張しない）。
- **品質ゲート**: `make test` フルスコープ green（1326 passed）。

  復元可能になり、8/8 マークが盤面復元評価の対象になった。
