# 反実仮想レフェリー 初回検証（2026-07-16 スナップショット）

「人間の教師になれるCPU」構想の初回検証。何万局の勝率でなく、**同一世界（CRN）×終局ロールアウトの
対照実験**で1手/1プランの因果効果を少数試行で判定する計器（`tests/scripts/counterfactual_referee.py`）
を実装し、人間マーク6件と突き合わせた。教師ネットは固定 gen5（学習ドリフトの影響を受けない錨）。

## 1. 計測の系譜（同日中に3段階で精密化）

### 1-1. root全数モード（勝敗のみ・6世界×sims64）

一致 5/6。ただし内訳は「引き分けによる一致」が大半＝勝敗だけでは飽和局面（どう打っても勝つ/
負ける）で判別力ゼロ。g3@93 は全18手が 0/6（どう打っても負け）＝この「無駄ドン付与するな」
マークは**勝敗に文字通り影響しない純粋な作法**であることが対照実験でも確定。

### 1-2. 勝ち方の質タイブレーク追加（残ライフ差・決着ターン）

飽和局面にも序列が付いた。例:
- g3@82: PASS(L+1.17) と OP15-105カウンター(L+1.50) は同価値圏・**EB03-053 で切るのは明確に損**
  （4/6・L+0.50）＝「切るなら105、EB03温存」という人間指摘より細かい答え。
- g3@93（負け局面）: 展開(L−2.0) ＞ 攻撃(−2.67) ＞ TURN_END(−3.0)＝**負け方の質**まで判別。
- 教訓: ±0.3程度のライフ差はノイズ＝**同価値バンド**で判定すべき（厳密辞書式は過敏）。

### 1-3. g3@64 の不一致の解剖（2つの汚染源を特定）

root全数では「素攻撃 ≫ 付与」で人間と逆。原因を追うと:

1. **ロールアウト盲点（主因・トレースで実証）**: 付与を選んだ後のロールアウト役（gen5）が
   `付与→付与→TURN_END` と**一度も攻撃せず**にターンを終える＝マークの対象である盲点そのものが
   ロールアウトに混入し、「付与プランの後半」を実行しない。root prefix 比較は
   「その後の実行力」を測ってしまう。
2. **復元忠実度ギャップ（副次）**: フレーム復元はパワー修正を持たない（実対局で 1000 に
   デバフされていた OP15-119 が素の 7000 で復元される等）。

**対策=プランモード**（手順列を固定して適用→以降ロールアウト）を実装し再対決:

| プラン（g3@64・8世界） | 勝ち | 平均残ライフ差 |
|---|---|---|
| 素で攻撃（実際の手） | 8/8 | L+2.38 |
| 付与→攻撃（人間指摘） | 8/8 | L+2.00 |

＝**復元盤面上ではどちらでも勝ち・差は好みの範囲（わずかに素攻撃側）**。実対局の真の盤面での
最終判定は復元忠実度の制約により保留（→§3）。

## 2. 確立したこと

1. **少数対照実験は成立する**: 同一世界CRNにより、勝敗に効く差は6〜8世界で判定できる
   （@82 の EB03 温存、@16 の無駄付与、@93 の負け確定など）。
2. **教師CPUの回答形式**: 「勝敗が変わる → 断定＋数値根拠」「変わらない → 同価値と申告し、
   勝ち方の質（ライフ・速度）で序列を提示」。人間と割れたら再現可能な根拠つきで反論できる。
3. **プラン比較が正しい単位**: 1手 prefix はロールアウト実行力に汚染される。人間の指摘も
   実際は「プラン」（付与→その駒で攻撃）なので、比較単位を揃えると検証がクリーンになる。
4. 固定教師（gen5）を使うため、v7 で確定した「深探索オラクルが value ドリフトを継承する」
   問題を回避する**外部の錨**として学習側（再ラベル）にも転用可能。

## 3. 残課題（次の実装）

- **リプレイ真盤面化**: フレーム復元でなく、記録された全手順を先頭から適用して
  マーク時点の真の盤面（パワー修正・レスト状態込み）を作る（`resolve_recorded_action` 基盤を流用）。
  復元忠実度ギャップを閉じ、@64 の最終判定と全マークの再検証を行う。
- 同価値バンド（|ライフ差|<0.5 は引き分け扱い）の判定既定化。
- ターンプラン**自動列挙**モード（現状は手指定・等価手順の重複除去＋ビームで全数化）。
