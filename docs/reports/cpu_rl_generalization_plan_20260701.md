# 学習型CPU「1セットで汎化させる」学習計画（設計正本）

日付: 2026-07-01 / 対象: 学習型CPU（Dual-Net + NN誘導MCTS）を**実デッキ全般へ汎化**させる次期学習
関連: `cpu_rl_pilot_p3_results_20260630.md`（Gen2本走）／`cpu_rl_genprobe_results_20260701.md`（本計画の pre-flight 実測）

> 本書はスナップショット（後から改変しない）。以降の実測・変更は新しい日付の報告に追記する。

## 0. 背景と要求

Gen2（合成の色寄せデッキで自己対戦学習）は製品L1に 0.925 だったが、**実アーキタイプ（黒ひげ
OP16-080 等）では value が初手から ≈-1 に張り付く＝分布外(OOD)**。原因は「合成デッキで学習した
ネットが未知の構築を評価している」分布シフト。

ユーザ要求: **実デッキは無限にあるので個別収集ではなく汎用的な学習にしたい。かつ次の学習は
1回（=数世代で1セット）で終わらせたい**（train→OODで失敗→再学習、のループを避ける）。

## 1. 方針: 「1セットで終わる」= 世代ループで直せない“不可逆な選択”だけを gen0 前に確定する

| 区分 | 中身 | 世代を跨いで直せるか |
|---|---|---|
| **不可逆（gen0前に確定必須）** | エンコーダ表現・デッキ生成分布・行動空間・held-outゲート定義 | ❌ 途中変更＝全重み破棄＝セットやり直し |
| **世代ループが自己修正** | value/policy の重み | ✅ これを直すのが世代ループの仕事 |

→ 不可逆な3つ（**表現・デッキ分布・ゲート**）をセット開始前に**安く検証して固定**すれば、あとは
世代ループが強くする。ここを外すと数世代回してから気づいて全やり直し＝最悪。**pre-flight の
オフライン検証が「1セットで済む」担保**。

## 2. 汎化の3レバー（不可逆・gen0前に確定）

### レバー① カードを“効果フィンガープリント”で表す（表現）
現行 `encoder.py` の `card_idx`（card_id→Embedding）は**丸暗記**。未知/低頻度カードの埋め込みは
未訓練＝ゴミ入力で V が崩れる。パーサ産出の構造化効果（TriggerType/ActionType/TargetQuery）を
**固定長 behavior ベクトル**へ落とす（`tests/rl_fingerprint.py`、CARD_DIM=50）。「コスト5以下をKO
して1回復する未知カード」は同じ振る舞いの既知カードと同じ入力＝分布内になる。
ID埋め込みは**捨てず低次元＋強dropout の残差**として残す（既知カードの個性は保ち、未知は特徴へ縮退）。

### レバー② 自己対戦デッキ生成のドメインランダム化（データ分布）
現行 `build_deck` は決定的な色寄せ1種。代わりに**特徴空間を張る手続き生成**: 色数・カーブ・
キーワード密度・効果種別ミックスを振り、アーキタイプ形状クラスタ（アグロ速攻/コスト増コントロール/
大型フィニッシャー/回復コントロール/除去多め/リダイレクト壁）を混ぜる。実デッキが落ちる特徴領域を
訓練分布が覆えばよい。**pre-flight で「特徴があっても未見領域の外挿は崩れる」ことが実測された（§下）＝
②は任意ではなく必須**。

### レバー③ 汎化を測る held-out ゲート（受け入れ基準）
訓練デッキ集合Aで学習 → **Aと素で交わらない集合B**（手組みの実アーキタイプ＝黒ひげ等を含む）で
**勝率**を測る。この「OOD勝率」を生成ゲートに追加し、**セットの受け入れ＝分布内交差評価＋held-out
OOD勝率**（後者が本命）。定義は開始前にfreeze。

## 3. 推論時の安全網（移行期・可逆・低コスト）
- **value アンサンブル**: 小さめの value を数個（共有trunk＋K ヘッド）。予測のばらつき大＝OOD検知 →
  その局面だけ sims を増やす／L1ルール評価へブレンド。アンサンブルは後付け不可なので**セット内で**。
- **探索でごまかす**: MCTS遷移は正確・deck非依存。sims↑で葉V依存↓。miscalib V の部分的洗濯機。

## 4. pre-flight ゲート（自己対戦前・安い GO/NO-GO）
- **①表現の汎化プローブ** … `tests/probe_generalization.py`。**実施済み（§下・結果別紙）**。
- **②デッキ生成カバレッジ** … 生成デッキ群と実アーキタイプの特徴分布を比較、実デッキが生成分布の
  内側に入るか。**未実施（②実装時に同梱）**。
- **③全カード fingerprint 監査** … 2652枚を fingerprint に通し退化（全ゼロ/衝突過多）が無いか。
  **一次確認済み**（全ゼロ0件・unique 1820/2652＝類似カードが正しく縮退）。

## 5. pre-flight ① の実測サマリ（詳細は別紙）
線形プローブ（held-out=黄＝黒ひげ色を全除外、教師=L1評価/対局結果）で:
- **識別子(card_id)表現は汎化価値を足さない**（held-out ΔR²≈0〜負）＝OOD負債。→ **① 実施＝ID埋め込みは残差へ降格を確定**。
- **フィンガープリントは実価値信号を足し、信頼できる密教師で held-out へ転移**（ΔR²≈+0.12〜+0.19）。→ **① GO**。
- **ただし色を丸ごと未見にする極限＋ノイズ教師では特徴表現でも外挿が崩れる**。→ **② ドメインランダム化は必須**、
  かつ**決定的な OOD 判定はランダムrolloutでは代替できず本走の held-out 勝率ゲートで行う**（③）。

## 6. セット設計（gen0前に freeze する数値）
- 世代数N: 交差評価が頭打ちする所（Gen2 vs Gen1=0.568 で既に接近）＋**held-out OOD勝率の平坦化で停止**（目安3〜5世代）
- sims / replay buffer / games per gen / Dirichlet（Gen2 実績を踏襲）
- アンサンブル K（入れるなら now）
- チェックポイント: net-only git checkpoint（reclaim耐性は P3 で実証済み）
- **受け入れ基準**: 分布内交差評価 GO かつ **held-out(黄等)勝率 > しきい（要 freeze）**

## 7. 実行順（推奨）
1. 本計画を正本化（本書）＋ pre-flight ①（済） … **完了**
2. レバー① fingerprint を**本番 encoder へ移植**＋ドリフトテスト更新（`test_cpu_learned.py`）
3. レバー② デッキ生成器を実装＋ pre-flight ②③ を通す
4. レバー③ held-out 集合 B を freeze・勝率ゲート実装
5. 全ゲート GREEN で初めて**1セット起動** → 世代ごとに held-out 勝率記録 → 平坦化で停止
6. 移行期は現行 Gen2 を fallback（新系統が held-out ゲートを通るまで本番は Gen2 のまま）

## 8. 残リスクと退避
オフラインプローブを通しても RL の分散は残る。1セットを潰し得る唯一の要因＝**表現の天井**は①が早期に
捕まえた（GO）。**②の未見領域外挿リスク**も実測済み＝ドメインランダム化で被覆する設計に織り込み済み。
移行期は Gen2 が常に fallback。
