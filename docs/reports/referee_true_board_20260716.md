# 反実仮想レフェリー 真盤面判定（2026-07-16 スナップショット・第2報）

第1報（`counterfactual_referee_20260716.md`）の残課題「リプレイ真盤面化」を実装し、
マーク局面の判定をフレーム復元から**記録全手順の再実行**（`replay_runner.state_at_action`）に
切り替えた結果。真盤面＝パワー修正・一時効果・レスト状態など全内部状態が実対局と同一。

## 1. 実装（何を作ったか）

1. **`state_at_action`**: seed からデッキ構築→start_game→記録アクションを両席 scripted で
   順に再実行し、action index 直前で停止した GameManager を返す。CPU 再 decide なし＝
   方策ドリフトの影響を受けない。ルール分岐は miss として検出・報告（黙って誤再生しない）。
2. **効果対話リゾルバ**: エンジンは効果対話で「機械的既定解決の1手」しか合法手列挙しないため、
   記録の `selected`（card_id 列）から pending request の候補へ**列挙順・重複消費**で写像し
   payload を直接構築。uuid でしか記録されない候補（ドン!!等）は残候補が同質のときのみ充当。
3. **対象欠落 ATTACK_CONFIRM のフレーム差分特定**: 「リーダー優先」推測は g3 の
   rested キャラ攻撃（84/86/88）をライフ攻撃に化けさせ、幻のトリガー対話で分岐した（実測）。
   宣言フレームと戦闘解決フレームの相手側差分（ライフ減=リーダー／消えた札=その対象）で特定。
4. レフェリーに `--true-board` を追加（既定＝従来のフレーム復元・挙動不変）。

これで **g3 の全159手が最後まで真盤面で再生可能**。フレーム照合テスト＋デバフ再現テストを
`tests/test_replay_true_state.py` に固定。

**既知の限界**: g1/g2 の録画は現行ビルドと**セットアップ時の乱数消費が異なり**初期手札から
分岐（録画時ビルドとのエンジン乖離＝設計どおり miss として検出）。現行ビルドでの録画からは
全手再生できる。旧録画はフレーム復元に fallback。

## 2. 判定結果（フレーム復元 → 真盤面でどう変わったか）

### g3@64「ドン付与→攻撃すべき」— **逆転・人間指摘が正しい（最重要）**

フレーム復元盤面では両プラン 8/8 勝ち＝「同価値・わずかに素攻撃」だった。真相は、実対局で
**1000 にデバフされていた相手ルフィ（OP15-119）が素の 7000 で復元**され、局面が実際より
楽に見えていた。真盤面（接戦の負け気味局面）でプラン対決をやり直すと:

| プラン（真盤面） | 8世界 | 16世界 |
|---|---|---|
| 素で攻撃（CPU の実際の手） | 1/8・L−1.75 | 6/16・L−1.00 |
| **ドン付与→攻撃（人間指摘）** | **3/8・L−0.62** | **7/16・L−0.25** |

勝ち数・残ライフ双方で人間プランが優位（ライフ差 +0.75 は同価値バンド ±0.5 を超える）。
**判定: 人間指摘が正しい。** ユーザの直感（「ドンつけない方が勝てるなんてことはない」）どおり。

### g3@93「負け局面で無駄ドン付与するな」— **作法ではなく実利だった**

フレーム復元では全18手 0/6（どう打っても負け）＝純作法と判定していた。真盤面では
**PLAY/OP16-056（展開）に 1/6 の現実の勝ち筋**が残っており、ドン付与2手は最下位タイ:

- ◆PLAY/OP16-056: **1/6**・L−2.33（人間方向・最善）
- ATTACH_DON 2手: 0/6・L−3.00（ワースト帯）

**判定: 人間一致○（勝ち +1/6・ライフ +0.67）。** 「無駄付与せず展開に回せ」は勝機を保つ
実利の指摘だった。盤面忠実度が判定の質を直接変えた第2の実例。

### g3@82「カウンターは105で切れ・EB03温存」— 真盤面でも一致○

◆PASS 5/6・L+2.00 ＞ OP15-105 4/6・L+0.83 ＞ EB03-053 3/6・L+0.50。
フレーム復元時と同じ結論（この局面はパワー修正が結果に効かなかった）。

### g3@68（付与マーク・root全数モードは✗）

root 全数（1手 prefix）比較では素攻撃 5/6 が付与 3〜4/6 を上回り✗。ただしこれは第1報で
特定済みの**ロールアウト盲点汚染**（付与後にロールアウト役が攻撃しない）が残る比較単位で、
付与系マークはプランモードで判定すべき（@64 と同構造）。プラン対決の結果は §3。

## 3. g3@68 プラン対決（真盤面・8世界）— レフェリーが人間に反論するケース

| プラン | 勝ち | 平均残ライフ差 |
|---|---|---|
| **素でリーダー攻撃（CPU の実際の手）** | **6/8** | **L+1.38** |
| 付与→OP11-106 で攻撃（人間指摘方向） | 5/8 | L−0.38 |
| 付与→リーダーで攻撃（人間指摘方向） | 3/8 | L+0.12 |

プラン単位で揃えても素攻撃が優位（勝ち +1〜+3・ライフ +1.26〜+1.75）。@64（負け気味＝
付与で1点を太らせる価値が高い）と対照的に、@68 は**勝ち気味局面でテンポ攻撃が正解**という
分岐で、同じ「付与しろ」でも局面文脈で答えが割れる——これは第1報の狙いどおり
「人間と割れたら再現可能な根拠つきで反論する」教師CPUの挙動そのもの。

## 4. 結論

1. **真盤面は判定を変える**: @64 は逆転（人間が正しい）、@93 は作法→実利に格上げ。
   復元忠実度は教師CPUの信頼性の土台であり、フレーム復元では不十分だった。
2. **レフェリー＋真盤面＝人間マークの検証器が完成**: 勝敗が変わる差は6〜16世界で判定でき、
   同価値なら勝ち方の質（ライフ・速度）で序列を出す。一致だけでなく**反論**（@68）も
   再現可能な数値根拠つきで出せる。
3. **比較単位はプラン**: 1手 prefix はロールアウト実行力に汚染される（@68 root ✗ と
   プラン対決の乖離が好例）。付与系の指摘は「付与→その駒で攻撃」のプラン全体で比較する。
4. 今後: 現行ビルドで録画された対局は全手真盤面化できる＝**今後の人間対局マークは全て
   真盤面で判定可能**。旧録画（g1/g2）はフレーム復元 fallback。ターンプラン自動列挙
   （ビーム＋等価手順の重複除去）と同価値バンド既定化が次の実装。
