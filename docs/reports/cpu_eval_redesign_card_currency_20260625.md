# CPU 評価関数 再設計メモ — 単一通貨「カード」＋時間割引＋2層化（2026-06-25）

> 設計メモ（plan）。確定後に `docs/SPEC.md §2.5` へ吸収する前提のスナップショット。
> 経緯：評価関数の学習化（価値ブレンド）が実測で頭打ち（線形 val_acc ~0.83 でも人間ドメインへ非転移・
> GBDT も線形を超えず＝モデルの表現力はボトルネックでない）と判明したため、強化の主軸を
> 「**評価パラメータの構造的な再設計（統合・次元削減）**」に置き直す。

## 0. 要約（結論）

評価関数を次の原理に畳む：

> **評価値 ＝（自分のカード資源 − 相手のカード資源）を、残りライフ（＝lethal までの時間）で割り引いたもの。**
> 盤面＝割引ゼロの即時カード、手札＝割引付きの将来カード、ライフ＝非線形に割引率を支配する銀行残高。

その上で評価を **2 層**に分離する：

- **L1（コア葉評価・本命）**：単一通貨「カード」＋時間割引を、**探索がシミュレートした葉盤面**に対して計算。
  ホライズン内の効果はすべて葉盤面に表現済み＝この層が自動で値付けする。**チューニング対象はこの層だけ。**
- **L2（ホライズン外残差・当面OFF）**：葉スナップショットに映らない潜在価値（ブロッカーのオプション価値、
  KO耐性、将来ターンのクロック率）だけを、残ゲーム長で時間割引して加算。独立フラグ・単独A/B・初期は無効。

## 1. 核心命題：OPCG では「カード」が唯一の通貨

OPCG はライフが文字どおりカードの山で、被ダメージでライフ上端が**手札に入る**。資源の変換が一本の鎖になる：

```
ライフ ──(被ダメージ)──▶ 手札 ──(ドンを払う)──▶ 盤面 ──(攻撃)──▶ 相手ライフを削る
   ▲                      └──(カウンター)──▶ 防御
   └────────────────（相手の盤面が自分のライフを削る）
```

- 手札 ＝ 展開にも防御（カウンター）にも使える純資源
- 盤面 ＝ 手札から変換され、毎ターン相手のライフ（＝相手のカード）を削る
- ドン!! ＝ 手札→盤面→打点の**変換スループット**（＝「将来どれだけ手札を運用できるか」）

→ すべてカード換算に還元できる。実データもこれを支持：学習線形モデルで `life_diff` が支配的(+0.87)、
盤面パワー差が次点。「カード資源の差」が勝率の本体。

## 2. 唯一の補正：時間（クロック）＝割引率

純粋な「カード枚数」だけでは **damage race（先に相手ライフを 0 にする競争）** を取りこぼす。
カード総量が同じでも、**盤上のカードは「今」働き、手札のカードは「ドンを払って数ターン後」にしか働かない**。
残り時間が短い（自ライフが薄い）ほど、遅い資源（手札・将来のドン）の価値は下がる。

したがってクロックは「第2の通貨」ではなく、**単一通貨に掛ける状態依存の割引率**として持つ：

- **盤面のカード**：割引ゼロ（今働く）＋相手プールを削る速度
- **手札のカード**：割引付き ＝ P(展開まで生存) × ドンで回せる量
- **自ライフ**：割引率を決める残り時間。lethal 近傍は**非線形プレミアム**（最後の数枚は「将来の手札」でなく
  「負けまでの距離」）

> 「枚数だけの 1 スカラー」は行き過ぎ（テンポを誤評価する）。「1 通貨＋ライフ駆動の 1 割引」が最小で忠実な形。

## 3. 探索と評価の分業（なぜ効果に静的重みを足さないか）

α-β の鉄則：**探索＝結果盤面まで計算できる戦術／静的評価＝探索が見られない残りだけを値付け**。

- ホライズン内で決着する効果（バニッシュ・除去・登場時・ドロー・今ターンの打点）は、**手を打った結果盤面に
  既に表現される**＝L1 が自動で正しく値付け。**別途の静的重みは二重計上**。
  - 例（バニッシュ）：通常ダメージ＝相手「ライフ−1・手札+1」（変換）／バニッシュ＝相手「ライフ−1・手札±0」
    （相手にカードを返さない＝真のアド）。その攻撃がホライズン内で通れば、結果盤面の「相手手札が増えていない」
    状態を L1 が拾う。固定 `W_KW_BANISH` は冗長。
  - 実証：`W_RECUR_ENGINE` の ablation で「horizon=4 が将来発動を結果盤面で拾う＝二重計上」が確認され撤去済み。
    データでも `removal_x_oppfield ≈ 0`（除去は探索が処理済み＝静的寄与ゼロ）。
- 静的重みが正当なのは **ホライズン外の潜在価値だけ**（→ L2）。しかもそれも残ゲーム長で時間割引した残差。

**禁じ手**：効果がホライズン内なのにビーム剪定で防御ラインを探索が捨てている場合、それを静的重みで埋めるのは
パッチC（ハック）。正しい対処は探索側（ビーム拡張・着手順序）。L1/L2 を分けるのは、この区別を可能にするため。

## 4. 2層アーキテクチャ

### L1：コア葉評価（探索が毎葉で使う）
- 入力＝**シミュレート済みの葉盤面**（カードレースのスナップショット）。
- `eval = discount(life) · ( cards_me − cards_opp )`
  - `cards_*` ＝ 手札（割引付き）＋盤面（割引ゼロ・打点込み）＋ライフ（非線形）。
  - `discount` ＝ 残りライフ（lethal までの時間）で決まるカーブ。
- ここだけを SPSA / 学習でチューニング。パラメータは少数・ほぼ直交。

### L2：ホライズン外残差（別ロジック・初期OFF）
- 葉スナップショットに映らない潜在価値のみ：ブロッカーのオプション価値／KO耐性／将来ターンのクロック率
  （ダブルアタック・アンブロッカブルの「次ターン以降も止められない」分）。
- 各項は残ゲーム長／ホライズン距離で**時間割引**。独立フラグで ON/OFF、**単独 A/B で効果を測る**。
- 当面は無効（OFF）。ノイズに勝てない残差は OFF のまま。

### 1 効果が両層に割れることの扱い
「効果＝L1 か L2 か」ではなく層をまたぐ。定義を固定する：
- **L1 ＝ 葉盤面を“カードレースのスナップショット”として読んだ値**
- **L2 ＝ そのスナップショットに映らない将来修飾だけ**
（例：アンブロッカブルの「今ターンの一撃」は L1＝葉の打点／「次ターン以降」は L2＝将来率。）

## 6. データ裏付け（自己対戦 6426 行・学習線形モデル）

- 実効重み上位：`life_diff`(+0.87) ≫ 盤面パワー差(±0.6) > 絶対ライフ・相手手札・リーダーパワー。
  → **核は L1 の少数項**。
- ほぼ寄与ゼロ：`removal_x_oppfield`・`hand_removal_me`・`blocker_x_oppfield` ≈ 0。
  → option-value 交互作用は静的には効かない（＝探索が拾う／L2 の残差にすぎない）。
- 高相関（|r|≥0.6 が 27 ペア）：`field_n↔field_pow`(0.89)、`hand_n↔hand_char↔hand_counter`(~0.9)。
  → **冗長＝統合余地が大きい**。これらを畳むとパラメータが直交化し、SPSA / 手調整が低ノイズで効く。

## 7. チューニング方針

1. **統合（次元削減）を先に**：上表どおり L1 を「カード基礎価値・割引カーブの形・盤面→打点変換レート」の
   少数パラメータに畳む。死に項と探索が拾う効果重みは撤去。
2. **L1 を SPSA で同時最適化**（自己対戦 Elo）。座標降下（1個ずつ）はノイズ＞効果・相互作用無視で非効率。
   直交化後なら SPSA が有効。
3. **L2 は当面 OFF**。後で単独 A/B（ON/OFF）で「効くか」を測ってから少数だけ採用。
4. **パッチC（補償ハック）は L1 が賢くなった分だけ撤去**（WBS「変な手」Phase3 と統合）。

## 8. 検証ゲート（CLAUDE.md 準拠）

- 構造監査 `EXCEPTION/CARD_LOSS/TEMP_LEAK = 0`・カード挙動ベースライン不変（評価変更は挙動に影響しないはずだが確認）。
- 「変な手」Phase0 監査（null-move regret）でカテゴリ別カウンタが**非増加**。
- アリーナ A/B で**勝率非劣化**（±35〜58 Elo のノイズ帯を破るペア局数で）。
- 統合の各ステップを上記ゲートで段階導入。退行は差し戻し。
