## 0. 要約（結論）

評価関数を次の原理に畳む：

> **評価値 ＝（自分のカード資源 − 相手のカード資源）を、残りライフ（＝lethal までの時間）で割り引いたもの。**
> 盤面＝割引ゼロの即時カード、手札＝割引付きの将来カード、ライフ＝非線形に割引率を支配する銀行残高。

評価は 2 層に分けるが、**本メモが設計するのは L1 のみ**：

- **L1（コア葉評価・本メモの対象）**：単一通貨「カード」＋時間割引を、**探索がシミュレートした葉盤面**に対して
  計算する、探索が毎葉で使う評価。ホライズン内の効果はすべて葉盤面に表現済み＝L1 が自動で値付けする。
- **L2（ホライズン外残差・本メモ対象外）**：葉スナップショットに映らない潜在価値を扱う別レイヤー。
  **中立な状態から後日あらためて設計する**。本メモでは内容を定めない。

> 本メモの L1 数式は外部AIレビュー **4ラウンド**＋スコープ自己監査を経た **v0.4**（§4）。骨子(§1–3)は不変、
> 数式の詳細は §4、現行 `W_*` の仕分けは §5、残課題（以降は実測で決める）は §9。

## 1. 核心命題：OPCG では「カード」が唯一の通貨

OPCG はライフが文字どおりカードの山で、被ダメージでライフ上端が**手札に入る**。資源の変換が一本の鎖になる：

```
ライフ ──(被ダメージ)──▶ 手札 ──(ドンを払う)──▶ 盤面 ──(攻撃)──▶ 相手ライフを削る
   ▲                      └──(カウンター)──▶ 防御
   └────────────────（相手の盤面が自分のライフを削る）
```

- 手札 ＝ 展開にも防御（カウンター）にも使える純資源
- 盤面 ＝ 手札から変換され、毎ターン相手のライフ（＝相手のカード）を削る
- ドン!! ＝ 手札→盤面→打点の**変換スループット**（＝「将来どれだけ手札を運用できるか」）

→ すべてカード換算に還元できる。実データもこれを支持：学習線形モデルで `life_diff` が支配的(+0.87)、
盤面パワー差が次点。「カード資源の差」が勝率の本体。

## 2. 唯一の補正：時間（クロック）＝割引率

純粋な「カード枚数」だけでは **damage race（先に相手ライフを 0 にする競争）** を取りこぼす。
カード総量が同じでも、**盤上のカードは「今」働き、手札のカードは「ドンを払って数ターン後」にしか働かない**。
残り時間が短い（自ライフが薄い）ほど、遅い資源（手札・将来のドン）の価値は下がる。

したがってクロックは「第2の通貨」ではなく、**単一通貨に掛ける状態依存の割引率**として持つ：

- **盤面のカード**：割引ゼロ（今働く）＋相手プールを削る速度
- **手札のカード**：割引付き ＝ P(展開まで生存) × ドンで回せる量
- **自ライフ**：割引率を決める残り時間。lethal 近傍は**非線形プレミアム**（最後の数枚は「将来の手札」でなく
  「負けまでの距離」）

> 「枚数だけの 1 スカラー」は行き過ぎ（テンポを誤評価する）。「1 通貨＋ライフ駆動の 1 割引」が最小で忠実な形。

## 3. L1 が値付けする対象：葉盤面のカードレース・スナップショット

α-β の鉄則：**探索＝結果盤面まで計算できる戦術／L1＝探索がシミュレートした葉盤面を「カードレースの
スナップショット」として読む**。ホライズン内で決着する効果は手を打った結果盤面に既に表現されるので、
**L1 は盤面スナップショットを読むだけで足り、効果ごとの静的項を持たない**。

- 例（バニッシュ）：通常ダメージ＝相手「ライフ−1・手札+1」（ライフ→手札の変換）／バニッシュ＝相手
  「ライフ−1・手札±0」（相手にカードを返さない＝真のアド）。そのバニッシュ攻撃がホライズン内で通れば、
  結果盤面の「相手手札が増えていない」状態を **L1 が自動で正しく値付け**する。
- したがって L1 の設計は「個々の効果に重みを付ける」のではなく「**葉盤面をカード通貨で読む式を 1 つ作る**」こと。

> 葉スナップショットに映らない潜在価値（＝ホライズン外の残差）は L1 の対象外。L2 で後日あらためて扱う。

## 4. L1 アーキテクチャ（v0.4・外部レビュー4ラウンド反映）

入力＝**シミュレート済みの葉盤面**（カードレースのスナップショット）。

```
eval   = R_me − R_opp + (Tele_me→opp − Tele_opp→me)
R_side = R_life + R_board + R_hand + R_don
```
```
R_life  = w_life·L + Surv_life(L)                                   # ライフ枚数＋lethal非線形のみ / デッキ切れ同形
R_board = Σ_body max( 顔打点寄与_i , トレード価値_i(power, rested) ) # 1体1アクション=一度だけ計上 / レストは葉手番依存で割引
R_hand  = greedy_alloc(手札, Don予算):                              # 1枚は排他資源（展開 or カウンター）
            Δ_i = dev_i·γ_surv − ctr_i·amp を降順、Don予算まで展開・残りカウンター
            dev_i は「汎用の展開可能カード価値」（効果別にしない＝§4.1）
            カウンターイベントは葉のアクティブドンでクリップ（撃てない防御札を過大評価しない）
            相手手札は中身不可視→枚数ベース期待値×上振れ補正
R_don   = w_don · アクティブドン                                    # ランプ＋「構え」防御オプション
Tele_X→Y= g( EffReach_X , Y_life )                                  # リーサル距離（防御控除はここ“だけ”・§4.2）
            EffReach_X = max(0, Σ_atk(X) 有効パワー − Yブロッカー吸収 − Y期待カウンター緩衝)
```

割引と圧力は**共有する生存“状態”（両者ライフ・平均clock）の別関数**（束ね過ぎず・共線でもなく）：
```
γ_surv = f(想定残ターン数)            # 展開の割引：あと何ターン生きて使えるか
amp    = 1 + λ·decay(即時圧力)        # カウンター増幅：相手Teleとの差分。確殺接近で連続的に 0 へ（崖なし）
```

ノブ ≈8〜9（汎用項のみ・ほぼ直交）。**チューニング（SPSA）対象はこの層だけ**。
非リーサルのオーバーステイ抑止は**探索の相手モデル**（期待カウンター応手）が担当し、**L1静的評価には入れない**
（入れると探索のモデル済みカウンターと二重控除になる）。

### 4.1 スコープ規律：`R_hand` の `dev_i` は汎用に限定
「その札を出したら何が起きるか」を**効果別**に静的評価すると (C) に反する（＝探索の仕事を二重計上）。
`dev_i` は「**展開可能なカード1枚**」の汎用価値（γ と Don で割り引くだけ）に限定し、特定の札を出す価値は
**探索が出した結果盤面で評価**する。`ctr_i`（刷りカウンター値）・コストは札の静的属性なので読み取り可。

### 4.2 スコープ規律：`Tele`（リーサル番兵）はホライズンと整合
`Tele` は「次ターンに殺されるか／殺せるか」の 1 手先予測で、**厳密には L1（純スナップショット）と L2 の境界**。
探索のホライズンが相手の反撃を含む場合、ホライズン内の致死を `Tele` でも数えると**二重計上**。
よって `Tele` は**探索がシミュレートする範囲を超えたリーサルだけを見る端数項**として扱う
（ホライズン内の致死は探索に任せる）。相手の期待カウンター緩衝（隠れ情報の枚数ベース推定）は、
**この `Tele` 一箇所にのみ**置く。

> L2＝ホライズン外残差（ブロッカーの2ターン以上先のオプション価値・KO耐性・メタ/制圧効果・遅延誘発など）は
> 別レイヤーとして中立な状態から後日設計する。本メモでは内容を定めない。
> なお v0.4 には**キーワード固有の静的重みは一つも無い**（ブロッカーも `Tele` の控除としてのみ作用）＝L2へ正しく分離済み。

## 6. データ裏付け（自己対戦 6426 行・学習線形モデル）

- 実効重み上位：`life_diff`(+0.87) ≫ 盤面パワー差(±0.6) > 絶対ライフ・相手手札・リーダーパワー。
  → **L1 の核は少数項**（ライフ差・盤面パワー差）。
- 高相関（|r|≥0.6 が 27 ペア）：`field_n↔field_pow`(0.89)、`hand_n↔hand_char↔hand_counter`(~0.9)。
  → **冗長＝統合余地が大きい**。盤面系・手札系をそれぞれ 1 系に畳むとパラメータが直交化し、
  SPSA / 手調整が低ノイズで効く。

## 7. チューニング方針

1. **統合（次元削減）を先に**：上表どおり L1 を「カード基礎価値・割引カーブの形・盤面→打点変換レート」の
   少数パラメータに畳む。
2. **L1 を SPSA で同時最適化**（自己対戦 Elo）。座標降下（1個ずつ）はノイズ＞効果・相互作用無視で非効率。
   直交化後なら SPSA が有効。

## 8. 検証ゲート（CLAUDE.md 準拠）

- 構造監査 `EXCEPTION/CARD_LOSS/TEMP_LEAK = 0`・カード挙動ベースライン不変（評価変更は挙動に影響しないはずだが確認）。
- 「変な手」Phase0 監査（null-move regret）でカテゴリ別カウンタが**非増加**。
- アリーナ A/B で**勝率非劣化**（±35〜58 Elo のノイズ帯を破るペア局数で）。
- 統合の各ステップを上記ゲートで段階導入。退行は差し戻し。

## 9. 残課題（紙上は確定・以降は実測で詰める）

外部レビュー4ラウンド＋スコープ自己監査で**構造は確定**（§4 v0.4）。残るのは**数値（カーブの形・係数）で、
いずれも紙上でなくアリーナ計測で決める**性質のもの：

- **`Surv` / `γ_surv` / `amp` カーブの形と強さ**：lethal非線形プレミアム・展開割引・カウンター増幅の係数。
- **`Tele` のホライズン端しきい値**：探索が見る致死とどこで切り分けるか（§4.2）。二重計上回避の運用点。
- **相手手札の上振れ補正係数**：自分は貪欲最適・相手は平均のスケールズレを埋める係数（暫定~1.2）。
- **レスト割引量**（§Q3）／**`R_don` の構え防御オプションの重み**／**`W_SETTLE_PRESSURE` の統合可否**。
- **探索の相手モデル健全性の確認**：非リーサルのオーバーステイ抑止が探索側の期待カウンター応手で十分か
  （静的バッファをL1に入れない方針の前提）。← **ON/OFF の A/B で即決着**。

> **設計レビューの収束判定**：ラウンド4で論点が「カウンター緩衝をL1に置くか探索に置くか」の**再燃**に入った
