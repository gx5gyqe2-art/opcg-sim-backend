# Phase 2 学習評価 hard ブレンド A/B（2a 不発）＋ 手調整への転換（2026-06-22）

> 計測報告（点スナップショット・改変しない）。CPU の「変な手」（100% 評価関数起因＝Phase 0/1 で確定）を
> 学習価値モデルのブレンドで直す試み（2a）の結果。**この規模・線形モデルでは強さ・変な手とも改善せず**、
> 計測ゲート付きの手調整（option 3）へ転換する根拠を記録する。計画＝`cpu_weird_move_remediation_plan_20260622.md`。

## 実験条件

- **データ**: フェア hard 自己対戦 320 局（`info_policy="fair"`・seed 0/1000/2000/3000 各80局）＝**6,992 行・40特徴**。決定論。
- **学習**: pure-Python 標準化ロジスティック回帰（`tests/train_value.py`・epochs 400）。**val logloss 0.603 / val acc 0.645**（ホールドアウト）。
- **配線**: `cpu_ai.evaluate` に winprob ブレンド（`(1-α)·base + α·winprob`・base は tanh↔atanh で eval スケール整合）。**`OPCG_VALUE_BLEND_HARD` 既定0＝ビット一致・決定論不変**（監査2局が α=0 で従来と完全一致＝①23 ②1 ③20 ④0 を実測確認）。winprob は常にフェア特徴。
- **測定**: 強さ＝`cpu_value_blend_ab.py`（α版 hard vs α=0 hard・席交互・各30局）。変な手＝`cpu_weird_move_audit.py`（`OPCG_VALUE_BLEND_HARD=α`・各12局）。

## 結果

### 強さ（α版 hard vs 現行 α=0 hard・30局）

| α | 勝率 | 概算 Elo | レイテンシ mean/max |
|---|---|---|---|
| 0.1 | 12/30 = 0.400 | −70 | 298ms / 1.05s |
| 0.25 | 12/30 = 0.400 | −70 | 297ms / 1.11s |
| 0.5 | 16/30 = 0.533 | +23 | 299ms / 1.07s |

### 変な手（α別監査・12局・件数/100局）

| α | ①差≤0 | ②自殺攻撃 | ③無駄ドン |
|---|---|---|---|
| 0（基準） | 1292 | 42 | 983 |
| 0.1 | 1708 | 75 | 1317 |
| 0.25 | 2317 | 42 | 1792 |
| 0.5 | 2108 | 50 | 1625 |

## 結論

- **強さ**: 低α（0.1/0.25）はむしろ −70 Elo で悪化、高α（0.5）でようやくノイズ帯の互角。**合格ゲート（勝率>55%）を満たす α は無い**。
- **変な手**: 全α で①③が増加（②も改善せず）＝ブレンドは変な手を**増やす**。
- **原因**: val acc 0.645＝フェア盤面特徴は勝敗をほとんど予測できていない（OPCG は分散大・静的盤面の線形予測が弱い）。手作り評価が内包するタクティカル知識を、弱い勝率予測器で薄めると素直に劣化する。
- レイテンシは問題なし（mean ~298ms）。

これは計画で想定した分岐（「2a＋が頭打ちなら 2b 昇格 or 別ルート」）。**2a（320局・線形）は不発**と確定。

## 転換: 計測ゲート付きの手調整（option 3）

学習モデルは**評価としては使えない**が、**重みは診断に使える**（どの特徴が実際に勝ちと相関し、手作り評価がどれを過大評価しているかの根拠）。Phase 0 監査＋アリーナという計測ゲートが揃った今、**根拠に導かれ・全変更を数値で受理/却下する手調整**は安全に回せる（過去の `normal<easy` 沼との違いは無計測でない点）。

次手＝診断 A（学習重み vs 手作り評価重みの突き合わせ＋結果ラベル付き「本物の変な手」抽出）→ 標的 TOP3 を確定 → 1項ずつ計測ゲートで是正（②自殺攻撃から）→ Phase 3 のパッチ撤去と統合。
