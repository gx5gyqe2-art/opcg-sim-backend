# 実対局リプレイ R0: 記録アクションの一意復元可否（曖昧性計測）

日付: 2026-07-04 / 計画: `docs/replay_verification_plan.md` §3・§6-1 の設計判断（(A) 逆引きタイブレーク vs
(B) 記録高解像度化）を確定するための実測。**報告（点）＝以後改変しない**。
計器: `tests/scripts/replay_ambiguity_probe.py`。

## 1. 問い

人間アクションは `cpu_ai._describe_move`（`{action_type, card=card_id, targets=[card_id]}`・**uuid 非依存**）で
記録される（`services/replay.py`）。再生時これを合法手へ逆写像する必要があり、**同 card_id の複製カード**
（実デッキは 4-of）で「同じ記述子の合法手が複数」＝一意復元不能になりうる。その発生率と分布を測る。

## 2. 方法

自己対戦の各意思決定点で、採用手の記述子 `d` を作り、合法手のうち `_describe_move(mv)==d` を満たす数を数える
（>1 なら記述子だけでは一意に戻せない）。アクション種別ごとに集計。**held-out 実デッキ**（複製あり）で計測
（合成デッキ＝distinct card_id では複製が無く 0%＝実態を測れないため）。

## 3. 結果

| 条件 | 意思決定点 | 一意復元不能 | 一致手数分布 |
|---|---|---|---|
| random policy・実デッキ・11局 | 1429 | **51（3.57%）** | 1手=1378 / 2手=48 / 3手=3 |
| ai(L1) policy・実デッキ・6局 | 964 | **44（4.56%）** | 1手=920 / 2手=42 / 3手=1 / 4手=1 |
| （参考）合成デッキ・random・12局 | 1360 | 0（0.00%） | 1手=1360 |

アクション種別ごとの曖昧率（ai／random）:

| action_type | ai | random | 備考 |
|---|---|---|---|
| PLAY | 14.2% | 16.2% | 手札の同名複製を出す |
| SELECT_COUNTER | 16.9% | 9.6% | 手札の同名カウンター |
| SELECT_BLOCKER | 7.7% | 9.7% | 場の同名ブロッカー |
| ATTACH_DON | 5.9% | 5.2% | 場の同名キャラへ付与 |
| ATTACK | 6.8% | 4.7% | 場の同名アタッカー |
| **RESOLVE_EFFECT_SELECTION** | **0%**(0/231) | **0%**(0/284) | 効果選択は一意（合計 0/515） |
| TURN_END / PASS / MULLIGAN / KEEP_HAND / ACTIVATE_MAIN | 0% | 0% | カード同定を伴わない |

## 4. 所見

1. **曖昧性は実在するが低く（3.5〜4.5%）、fan-out は小さい**（ほぼ 2手・稀に 3〜4手）。巨大な分岐は無い。
2. **カード同定を伴う手（PLAY / SELECT_* / ATTACK / ATTACH_DON）に集中**。制御手（TURN_END/PASS 等）は 0%。
3. **`RESOLVE_EFFECT_SELECTION` は 0/515 で完全一意** ＝計画 §3-2 で懸念した「effect 選択 payload の欠落で曖昧化」は
   **実測で低リスク**と判明（現行記録で足りる）。
4. **合成デッキでは 0%** ＝曖昧性計測は**必ず実デッキ**で行う（この計器の既定は実デッキ側を使う）。

### 挙動等価性の内訳（重要な区別）

- **手札の複製（PLAY / SELECT_COUNTER）**: 2枚は同一カード＝どちらを出しても**結果盤面は同一**（uuid が違うだけ）。
  → 決定論タイブレークで**挙動等価**に解決できる。
- **場の複製（ATTACK / ATTACH_DON / SELECT_BLOCKER）**: 場の 2 体は**ドン付与・パワー・レスト状態が異なりうる**。
  記述子は card_id しか持たないので、どちらの個体かは**記録から復元できない**＝タイブレークが誤個体を選ぶと
  再生が分岐しうる（真に危険な少数部分）。

## 5. 判断（計画 §6-1 の確定）

**(A) 決定論タイブレーク逆引きを主とする**。実測が支持:
- 曖昧率 4% 前後・fan-out 小・effect 選択は 0%＝記録の高解像度化（(B)・契約更新）は**原則不要**。
- 手札複製は挙動等価に解ける。場複製の一部だけが真の分岐リスク。

**具体方針（R1 へ）**:
1. リゾルバは「記述子に一致する合法手を列挙し、**合法手の決定論列挙順の先頭**を採る」（録画・再生で列挙順が
   同一＝決定論なら安全）。
2. **場複製の分岐は round-trip テストで検出**する（R2/R3）。再生結果が録画と食い違ったら失敗＝どの局でどれだけ
   起きるか実測できる。残差が無視できなければ、その手種**だけ** (B)-lite（記録に個体弁別子＝付与ドン数/パワー等を
   追加。`REPLAY_SCHEMA` 更新・§5 の契約手順）へ escalate。
3. effect 選択は現行記録のまま（0% 実測）。

## 6. 付随観測（R0 スコープ外・別途フォロー）

- random policy・実デッキの 1 局（12局中）で `apply_counter` 内 `pay_cost` が「ドン!!が不足」で
  `ACTION_EXCEPTION`。random 方策が支払い不能な `SELECT_COUNTER` を選んだ可能性（合法手生成が支払い可能性を
  絞れているか）＝**エンジン側の要確認事項**として切り出す（本 R0 の結論には影響しない・ai policy では未発生）。
