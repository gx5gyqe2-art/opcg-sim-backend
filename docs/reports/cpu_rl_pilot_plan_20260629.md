## 0. 結論（先に）

- **メカニズムは陽性**＝学習評価器は value 予測で L1 を超え、データ規模で伸びる（追補2のスケールカーブ）。
- **計算資源は律速でない**＝本セッションで実測（§2）。NN推論はL1と同オーダー、探索律速はエンジンの
  clone+apply（＝Rust化で効く層）。**現ネット規模ではGPU不要**。レビュアーも「GPUなしは無理」を**撤回**。
- よって方針＝scale-curve が測れない唯一のもの＝**「value予測の優位がプレイ強度に転換されるか」**を、
  9〜12人月のRust移植を払う前に、**数日CPUのPythonパイロット**で測る。
- 本書はそのパイロットの**確定手順＋判定ゲート＋交絡の切り分け設計**。

**重要**：本計画は過去に撤去した施策と一部重なる。下表で「何を試して何故ダメだったか」と
「新計画が各失敗をどう回避するか」を明示する。**新規性の核は④フルRLループであり、これは一度も試していない**。

| # | 過去に試した施策 | 結果 | 撤去コミット | 新計画との関係 |
|---|---|---|---|---|
| ① | **学習価値モデル（NNUE型）**＝極小ブレンドα×極小データ(2110行・val_acc0.645)を**α-βの葉**に差す | **Elo中立/−70（幽霊）** | `9b7d06e`/`a3d9cfb`、#149で削除 | **重なるが回避**：大規模データ＋半生表現＋**α-β禁止(MCTS)** |

| ④ | **フルRL（Dual-Net＝value+policy ＋ NN誘導MCTS ＋ 自己対戦反復 ＋ クロス評価）** | **未実施** | — | **本計画の核。新規。** |

### 過去の3つの致命傷と、新計画の回避策（レビューで確定）
1. **α-β+NN は water-oil**：minimax が NN の外れ値（幻覚）を能動的に極大化＝偽陰性ゲート。
   → **回避：評価器をα-βでなくMCTSに載せる**（平均化で幻覚を吸う。AlphaZeroがMCTSを採る理由）。
2. **模倣の天井**：教師あり×現行CPU outcome は教師の方策歪みを蒸留＝教師超え不可。
   → **回避：RLループ（policy improvement）で「教師超え」を機能証明**。SLだけでは偽陰性。
3. **入力表現の天井**：L1相当特徴ではL1の関数再現が上限。
   → **回避：半生表現＋カードID Embedding**（L1が捨てる相互作用・カード固有シナジーを学べる粒度）。

> ＝過去の不発は「①②を**安いやり方**で試して幽霊化した」もの。本計画は同じ轍を踏まないよう、
> 3失敗モードを設計で個別に潰した上で、**未実施の④**を最小コストで機能証明する。

|---|---|---|
| 1,000 | 0.586 | +0.004 |

### 2.2 計算は律速でない（本セッション実測）
- **1局面評価コスト**（中盤盤面・`tests/rl_evalcost.py`）：
  L1 `evaluate` **35.3µs** / NN forward batch=1 **31.2µs（×0.88）** / batch=64 **3.4µs（×0.10）**。
  → 「numpy極小NN推論はL1より桁違いに重い」は**偽**。探索律速はエンジンのclone+apply＝Rust層。
- **自己対戦スループット**（4コア・`tests/rl_throughput.py`）：探索付き ai40＝83,647局/日（現Python）、
  Rust×50で4.18M局/日。**10⁶局＝現Pythonで12日／Rust×50で6時間**。律速はサンプル複雑性であり計算資源でない。

## 3. パイロット実行計画（レビュー最終サインオフ反映・順序入替済）

レビュアー最終助言で**実装順序を入れ替えた**＝最も安価で情報利得の高いマイルストーンから踏む。

```
[GATE A] 三目並べでAZループ(自己対戦→学習→重み更新)が既知最適へ収束   … RLインフラの実装正しさ（数十分）
   ↓
[GATE B] OPCG + 教師ありSL-net で MCTS playout単調性(50→200→800)     … 探索の健全性＋PIMC挙動確認
   ↓
[P2/Gen0] SL-net + MCTS vs L1+α-β を最低400戦CRNで対戦              … 必要条件チェック（最優先・最重要）
   ├─ 勝率45〜50%+ → 「value優位はプレイ強度に転換」がRL前にほぼ証明 → [P3]へ
   └─ 勝率20%以下のボロ負け → RL前に [P4] でSL-netの表現力を先に診る
   ↓
[P3] OPCG自己対戦 数世代 + 世代間クロス評価(gen N+1 vs gen N)       … プレイ強度転換の本測定
   ↓
[P4] 容量ラダー(hidden 128→256→512) で表現力仮説を直接テスト        … 陰性時の帰属切り分け
```

### 3.1 アーキテクチャ上の確定事項（レビュー助言）
- **PIMC×MCTS の water-oil 回避**：各MCTS探索（例800 playout）中は**決定化した1つの世界線を固定**する
  （Information Set MCTS の簡略版）。プレイアウト毎に非公開情報を引き直すと、ノードのバリュー移動平均が
  異なる並行世界の勝率を混ぜ、分散爆発で木が崩壊する。**これを怠ると GATE B が PIMC ノイズで確実に失敗する**。
- **較正のドリフト対策（鶏卵問題）**：SL初期netで較正した c_puct は、RLで方策が尖ると不適になる。
  各世代フル再較正はコスト不見合いゆえ不要だが、**MCTSルートノードの訪問回数分布（探索手のばらつき＝エントロピー）
  をロギング**し、世代進行で1〜2手に潰れたら c_puct を少し上げる**軽微な介入ルール**を設ける。

### 3.2 交絡を断つ3 instrument（陰性時の「表現力 vs ハイパラ vs バグ」切り分け）
- **① 探索ハイパラのOPCG事前較正（P2前段）**：三目並べの定数をOPCGに**持ち込まない**。
  既存N=22623で教師あり学習した固定netをMCTSに挿し、対L1勝率最大化で c_puct・Dirichlet α を
  **OPCG上で粗較正**してからRL開始。
- **② playout単調性診断（GATE B）**：固定netで playout 50→200→800、勝率が単調増か。
  単調増=探索健全／平坦=探索ハイパラ不適合（＝表現力に帰属してはいけない、と即判定）。
- **③ 容量ラダー（P4）**：陰性時 hidden=128→256→512 で強度を測る。
  伸びる=表現力律速（大型net＋GPUへのGOサイン）／平坦=手法限界寄り。
  「表現力 vs 手法限界」を**断定でなく測定可能な勾配**に変換。

### 3.3 クロス評価の有意性要件（レビュー確定）
- **最低400戦**（有意水準5%・勝率ブレ±5%以内の統計的下限）。
- **CRN必須**：同一デッキ・同一山札並び（シード固定）の**初期盤面ペアを200個**用意し、
  **先攻/後攻を入替えて計400戦**。これで「先後の絶対的有利不利」「初期手札・ライフの引きムラ」を相殺し、
  **評価器・探索の純粋な差**を抽出する。単なるランダム対戦は無意味。

## 4. 判定ゲートと意味

| ゲート | 不通過の意味 | 通過の意味 |
|---|---|---|
| **GATE A**（三目並べ） | RLループ実装にバグ（backprop/データパース等）。**OPCGへ進まない** | 自己対戦→学習→更新ループは健全 |
| **GATE B**（playout単調性） | 探索ハイパラ不適合 or PIMC統合ミス。表現力に帰属してはいけない | MCTS探索は健全 |
| **P2/Gen0**（対L1 400戦CRN） | RLの初期状態が弱すぎ＝離陸困難。先にP4で表現力を診る | value優位のプレイ転換がRL前にほぼ証明 |
| **最終（P3/P4）** | 較正済みでも世代で強くならず容量ラダーも平坦＝**手法限界寄りNO-GO** | 世代で上昇＋55%突破＝**強GO**（Rust＋大型NN＋GPUへ投資） |

> **GATE A は GATE B 導入後も冗長でない**：GATE Bは「固定NNでのMCTS探索アルゴリズム」の正しさのみ保証。
> GATE Aは「自己対戦生成→バッチ学習→重み更新」という**RLループ全体**の健全性を保証する。役割が別。

## 5. スコープ・コスト・正直な限界

- **コスト**：数日CPU＋数百行（既存 `rl_encoder/rl_datagen/rl_net` 資産を拡張）。Rust移植は本結果が出るまで払わない。
- **環境**：dev sandboxは揮発性。本走は**安価な常設CPU VM（GPUなし）**を想定。
- **残る偽陰性リスク（レビュアーと合意済・非対称）**：
  - **陽性（世代で強くなる）＝強い転移**：極小NN＋限定playoutのハンデ下でL1超えなら、Rust化・大型化で更にスケール確実＝**確実なGO**。
  - **陰性（強くならない）＝転移不可**：表現力不足/ハイパラ不適合に吸収され「Rustで大型化してもダメ」へは飛躍できない。
    ゆえに陰性は「クローズ」でなく**容量ラダー③の勾配に応じた次手**として扱う。**確実な結論は陽性のみ**。
- **将来のGPU必須化分岐点**：表現力不足で学習が飽和し、ネットをResNet級に大型化せざるを得なくなった時。
  ただし大型化は**アーキテクチャ問題**でありGPUは弱い表現を救わない。「強い表現力＝大型NN＝GPU要求」は本走（GO後）の話。
