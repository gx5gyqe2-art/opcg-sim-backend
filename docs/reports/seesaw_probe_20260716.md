# 付与系マークのシーソー干渉 直接測定（2026-07-16 スナップショット）

v6 run 中に観測された「ドン付与系マークの 0↔1 往復（シーソー）」の機構特定。
計測: 付与系3マーク（g1@12・g3@64・g3@68）＋@82 の復元盤面で、gen5／r82（v6 best・付与マーク
1.0側）／r96（candidate・g1@12=0.67側）の3ネットの (a) 人間指摘方向への policy prior 質量、
(b) 人間手適用後 vs prior最大手適用後の value を分解（スクリプトは scratchpad・mark_gate の
復元/述語と `cpu_learned._priors_fn/_value_fn` を使用）。

## 結論（シーソーの機構・確定）

1. **value はこれらの決断をほぼ区別していない**: 人間手/top手の適用後 value 差は全ネットで
   0.01〜0.05。効率手の小さな EV 差はネットの平滑性に埋まる（=柱③の「勝敗ノイズ床」の value 版）。
2. **決定は policy prior 質量が支配**: 人間方向への prior が checkpoint ごとに 0.2↔0.6 で漂流し、
   マークの 0↔1 反転と一致。value が無差別なため、探索の訪問配分＝prior＋ノイズとなり、
   **その訪問分布を policy が教師として再学習する自己参照エコー**が prior のランダムウォークを生む。
   LR 半減で乱高下は減衰したがシーソーが残った理由＝学習率でなく**教師信号が無情報**だから。
3. **@82 は value が人間側を支持済み**（不カウンターが +0.05〜0.07 高い）のに prior が誤誘導＝
   柱③の PRIOR_BOUND 診断と完全整合。深探索（flat）が解けることも実証済みなので、
   **再ラベルの露出不足**（5%一様では希少決定点に届かない）が未改善の原因。

## v7 への設計帰結

- **A. 行動特徴の拡張**: 現行 ACTION_DIM は「手種別 + 関与カード特徴 + 3」のみで、
  「この付与が何を可能にするか」（付与後に新たに届く攻撃・リーサル接近等）が policy に見えない
  → 全付与が同一視され prior が文脈で分けられない。盤面から決定的に計算できる汎用特徴で解く。
- **B. 再ラベルの行動種別重み付け**: ATTACH_DON / SELECT_COUNTER 等の決定点で relabel 確率を
  引き上げ（カード固有でなく手種別レベル＝汎用）、無情報エコーの代わりに深探索の答えで prior を
  錨止めする。
