# pre-flight ④c: 表現の“もつれ”アブレーション＝色への過学習が黄盲目の主因（レビュー仮説を実証）

日付: 2026-07-01 / 契機: 外部レビュー「corr≈0 は色次元への entanglement の証拠。色を隠せば転移が回復するはず」
コード: `tests/pre_flight4c_ablation.py` / 先行: `cpu_rl_preflight4b_trace_20260701.md`（0/40=分布シフト確定）

> スナップショット（改変しない）。

## 方法（安い・MCTS不要）
fingerprint の一部次元をゼロマスクして net0 を bootstrap し直し、`corr(net.value, L1評価)` を
**非黄(train分布)→黄(held-out)** のゼロショットで再計測。
- **baseline**: 現状の fingerprint 全次元
- **no-color**: 色6次元[7:13]をマスク
- **behavior-only**: trigger+action[21:50]だけ残す（static/色/type/keyword を0）

## 結果（2seed）
| 表現 | 非黄 corr | **黄 corr** (seed0/seed1) |
|---|---|---|
| baseline | 0.78 / 0.83 | **−0.001 / +0.063** |
| **no-color** | 0.81 / 0.84 | **+0.484 / +0.284** |
| behavior-only | 0.68 / 0.67 | +0.273 / +0.484 |

## 解釈
1. **色次元を隠すだけで黄 corr が ~0 → +0.3〜0.5 に回復。非黄は無傷（0.81〜0.84）**。
   → **レビュー仮説◯: 色への“もつれ(entanglement)”＝spurious shortcut 過学習が黄盲目の主因**。
   色は非黄で「楽に value を説明できる近道」だが、未知色(黄)で近道が壊れると value 全体が崩壊。
   色を外すと振る舞い(trigger/action)次元が黄へ転移する。
2. **behavior-only** も黄を回復するが非黄がやや低下（0.67）＝**振る舞いだけでも転移するが、static スタッツ
   (cost/power等)も in-dist の value に寄与**。→ **no-color が最良**（黄回復かつ非黄維持）。
3. **回復は ~0.4 止まり（非黄 0.8 には未達）**。色を外しても残る差＝**黄固有の振る舞い（ライフ操作等）が
   非黄訓練に乏しく under-trained** な部分。→ **ここは②被覆／特徴拡張の担当**。

## レビューのクローズ質問への回答（データ付き）
「色を隠せば corr≈0 は緩和されるか、それとも相互作用の表現欠落か？」
→ **両方。ただし主因は色もつれ**: 色マスクで corr は 0→0.3〜0.5 に回復（もつれが主因・脱もつれで大きく改善）。
残差（0.5→0.8 の未達分）は**黄固有の振る舞い/相互作用が訓練に乏しい**ため＝被覆/特徴拡張が要る。

## 計画への反映（重要）
- **新レバー①b: 表現の脱もつれ**を最優先の安い根本策として追加。**色次元を除去 or 強い dropout/正則化**で
  近道学習を潰し、value を振る舞い依存にする（no-color は非黄無傷で黄回復＝色は汎化の純負債）。
- **② 被覆は残差担当へ格下げ調整**: 脱もつれ後に残る黄固有振る舞いの under-training を埋める補完。
  「①b 脱もつれ → ② 被覆」の順で、pre-flight⑤ は両方入れて MCTS vs L1 held-out の回復を測る。
- 一般教訓: **色のような強いドメイン固有・低カーディナリティ次元は汎化の近道になりやすい**。
  同様の危険は「特定リーダーID的な特徴」にもある → 追加する特徴(Traits)も**マルチホット＋正則化**で
  もつれを避ける設計にする。
