# 検証①-完: 透視切り分け＝Gen2の勝ちは本物・汎化キャンペーン終了

日付: 2026-07-03 / 位置づけ: `cpu_gen2_dissection_20260703.md`（検証①）§5 の残件
「出荷決定化の透視疑い」の決着。これをもって「誤前提(0/40崩壊)に立つ汎化キャンペーン」を終了する。
コード: `tests/_fairdet_probe.py`（公平決定化はテスト側実装・本番は不変）。
生ログ: `scratchpad/fairdet_match.log`。

## 1. 目的と仮説

出荷Gen2の MCTS決定化 `_determinize_opponent` は「相手手札のみ再サンプル・**自分の山札順と
両者のライフ札中身は実物のまま**」＝探索が自分の将来ドローと両者のライフ・トリガーを透視する。
- H0: Gen2の勝率はこの情報優位に依存（透視を止めると崩れる）
- H1: 依存しない（透視を止めても勝ち越す＝本物の実力）

補足: L1(PIMC)も同じ `_determinize_opponent` を使うため情報優位は**両者に対称**。本検証は
Gen2側だけ公平決定化(透視禁止)に差し替え、相手L1は出荷pimc4のまま測る。

## 2. 実験設定

- Gen2 = `decide_learned`（sims160）だが決定化を `FairGame.determinize`＝両者の隠匿情報
  （自山札順＋両者裏向きライフ＋相手手札・山札）を再サンプルへ差し替え（テスト側・本番不変）。
- 相手 = 本番L1（`cpu_ai.decide`, fair, pimc4）。held-out実デッキ・席替え・各10〜12局。
- 付随: 過去に「公平決定化がハング」と観測したが、単手0.34sで正常＝当時のハングは
  **CPU競合が原因**（決定化は無罪）と確認。

## 3. 結果

| デッキ | 出荷(透視あり) | 公平(透視なし) |
|---|---|---|
| nami_blue_yellow | 0.917 | 0.750 (9/12) |
| blackbeard_black_yellow | 0.833 | 0.917 (11/12) |
| hancock_blue_yellow | 0.667 | 0.800 (8/10) |
| **平均** | **0.806** | **0.822** |

## 4. 判定

**H1採択（透視非依存）。** 透視を止めても平均勝率は 0.806→0.822 とほぼ不変。デッキ別の上下
（nami −0.17／黒ひげ +0.08／hancock +0.13）は N=10〜12・CI±0.25 の統計ノイズ範囲。
Gen2の対L1勝ち越しは情報優位の産物ではなく**素の実力**。

## 5. 検証①の総括（キャンペーン終了）

| 問い | 結論 |
|---|---|
| 実デッキで0/40崩壊するか | ❌ 虚構（fingerprint研究net×黄除外の数字の誤帰属） |
| 実デッキで本番L1に勝てるか | ✅ 0.67〜0.92(透視あり) / 0.75〜0.92(透視なし)＝本物 |
| 前回の変な手#1(リダイレクト無駄撃ち)#2(ライフ追加) | ✅ 両方解消（#153配線修正・ログ状況再現で確認） |
| 勝ちは透視カンニング由来か | ❌ 否定（透視なしでも平均0.82） |

**出荷Gen2 は「実デッキで崩壊せず・本番L1に平均0.82で勝ち越し・既知の退行手も解消済み」の
まともに強い汎化CPU**だった。「Gen2は実デッキで汎化しない」を前提に走った一連の汎化
キャンペーン（脱もつれ／実効状態特徴／パラメトリック訓練／蒸留 vs z）は**存在しない障害**を
対象にしていた。当該コード・レポートは削除済み（本コミット群）。
