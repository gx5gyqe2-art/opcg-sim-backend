# 学習型OPCG CPU「1セット汎化」凍結設計 v3（本走仕様・レビュー全承認版）

日付: 2026-07-01 / **v2 を supersede**。外部敵対レビュー往復（全4巡・最終承認）を反映した本走用の凍結仕様。
根拠実測: genprobe(①) / genprobe2(②訂正) / minitrial(D) / preflight4(1-ply) / preflight4_mcts(0/40) /
preflight4b(機序=分布シフト) / preflight4c(色もつれ主因)。

> スナップショット。gen0 前に凍結する不可逆な選択を確定する文書。

## 1. 確定した実測（設計の根拠）
- **① 効果フィンガープリント表現は必要だが単体では不十分**（in-dist で MCTS が L1 に 0.5〜0.67、
  held-out(黄) vs L1 = 0/40）。識別子(card_id)は汎化価値≈0＝降格。
- **黄崩壊の主因は「色への Spurious Shortcut（もつれ）」**（色マスクで黄 corr 0→+0.3〜0.5・非黄無傷）。
- **残差（色除去後も corr 0.4止まり）は黄固有振る舞いの under-training**＝②被覆/特徴拡張の担当。
- **0/40 はバグでなく真正の分布シフト**（corr 非黄+0.83→黄≈0・NaN無・盤面非感応・交絡なし）。
- **MCTS は必須**（1-ply では value 粗く L1 に大敗）。
- **信頼できる汎化ゲートは vs L1(+MCTS) のみ**（R²・vs-ランダムは実力を過大評価＝偽の安心）。

## 2. 凍結アーキテクチャ
### 表現（encoder ①b）— ID埋め込みなし
- **振る舞い fingerprint（raw色 one-hot 除去）**: trigger種別/action種別/対象ゾーン/up_to/コスト/power/counter/
  keyword。色は汎化の純負債ゆえ raw では入れない。
- **エンジン注入の“実効状態”決定的特徴**（学習でなく `get_target_cards`/`_can_satisfy_node`/
  `negate_onplay_until`/`continuous`/`is_effect_negated`/`_active_restriction` から計算）:
  - **ローカル（合法手ごと）**: この手の効果は現在無効化か 0/1・有効対象数・発動条件への距離
    （あと N ドン/ライフ M 以下で適法、を連続値/ビン）。
  - **グローバル（状態）**: 相手 on-play 無効化 active・restriction・continuous メタ常在フラグ。
- **Trait（特徴）は決定的二値関係マスク**（内積でなく「この効果はこの対象にヒットするか 0/1」を注入）。
- **脱もつれ手法 = raw色除去（実証済）＋上記派生特徴＋②の自然正則化**。**GRL は破棄**（λスケジュールが
  1セットと矛盾）。追加特徴も ID丸暗記回避の関係表現で。

### Policy（ポインタ方策）
- 合法手集合の特徴行列上で softmax（非合法手は入力に存在しない＝マスク不要）。
- **決定的フラグを合法手ごとローカル特徴に Early Fusion（concat）**。per-action 共有 MLP が
  「flag=1→logit を下げる」を1次で即学習＝**無価値手の Prior が最初から低く、40 sims を有効手に全振り**。
- **ハードコード（Prior=0 直挿し）はしない**（無効化下でもスタッツ/ブロッカー目的の置きが勝筋になる縁例）。
  ペナルティ重みはネットに学習させる。

### Value ＆ 報酬
- グローバルなメタ常在を状態ベクトルへ early-fusion。
- **密な補助ヘッド**（ターン先のパワー差/ライフ差予測）を併設し、スパース報酬下での表現崩壊を防ぐアンカーに。
- **L1評価 bootstrap を warm-start＋outcome 移行後も補助信号として継続**（value/policy とも）。
- 本走教師 = **self-play 方策下の outcome（±1）**。ランダム続行 outcome は不可（ノイズ過多）。
  **L1 方策からの KL 制約**で崩壊防止。

### 探索・データ・ゲート・フェイルセーフ
- **探索**: MCTS 必須（低 sims を良 Prior で活かす。“結果で罰する”は従、事前フラグが主）。
- **データ②**: 実アーキタイプを種に**同色・同コスト帯でのミューテーション差し替え**（完全ランダムは
  カーブ/シナジー崩壊で禁）。数%は「極端だが合法なノイズ」を混ぜ OOD 耐性を付与。**①b と並列**で回す
  （直列は新 Shortcut を生む＝被覆自体が自然正則化）。
- **汎化ゲート③**: **vs L1(+MCTS) の held-out 勝率**が唯一の決定的判定。カバレッジ監査は
  **Covering Radius（実デッキ近傍に生成デッキが存在）/ プレイアビリティ=Q値の勾配 /
  駆け引き=Action-Advantage の分散**で測る。
- **フェイルセーフ**: 世代ごと checkpoint 評価＋自動 rollback、フルセット前にマイクロセットで
  ハイパラ感度分析、移行期は Gen2 fallback。

## 3. 実行順
1. **①b encoder（脱もつれ＋エンジン注入特徴）＋②ミューテーション生成器を並列実装**
2. **pre-flight⑤: 信頼ゲート（MCTS vs L1 held-out(黄)）で 0/40 がどれだけ回復するか実測**
3. GREEN なら本走（outcome self-play・KL制約・補助ヘッド・checkpoint+rollback、held-out勝率平坦化で停止）
4. 移行期は Gen2 fallback、新系統が③を通ったら差し替え
