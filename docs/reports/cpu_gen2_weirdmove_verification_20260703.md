# 検証①-補遺: 「前回の変な手」2件がログ状況の再現で治っていることを確定

日付: 2026-07-03 / 位置づけ: `cpu_gen2_dissection_20260703.md`（検証①）で #1=解消・#2=未確定
だった残件の決着。実ゲームログで観測された2つの退行手を、同じ局面を再構成して出荷Gen2の
選択を直接確認した。
コード: `tests/_reproduce_op16080_waste.py`, `tests/_reproduce_op16119_lifeadd.py`。

## 背景（実ログ由来の症状・原因）

原因は共通の**配線バグ**: 学習CPUのアダプタが効果選択に既定1手のみ渡していた
（CONFIRM_OPTIONAL=必ずaccept・SELECT_TARGET min0=必ず0枚）。修正 `merged_search_actions`
（PR #153）は origin/main に載っている。本補遺はその効果を個別局面で実測する。

## #1 OP16-080【相手のアタック時】任意リダイレクトの無駄撃ち

- 再現局面: 相手ターン・相手キャラが p1(OP16-080)の**リーダーを攻撃**・p1手札に【トリガー】1枚・
  p1盤面に《黒ひげ海賊団》無し ⇒ リダイレクト先はリーダーのみ＝発動しても対象不変で
  トリガー1枚を捨てる**純損**。正解=見送り。
- 結果: CONFIRM_OPTIONAL 発火（source マーシャル・D・ティーチ）。選択肢=[decline, accept] 両提示。
  **出荷Gen2 は見送り(decline) 5/5 seed**。実戦マイニング(黒ひげ16ペア)でも同能力の
  CONFIRM を n=202・decline 129(64%)。
- 判定: **✅ 解消**（旧=強制100%発動→修正後=ログ局面で見送り）。

## #2 OP16-119【登場時】「上3枚を見て1枚までライフに加える」を使わない

- 前提の切り分け: 合成盤(make_game・ライフ0)では ON_PLAY の continuation が完遂せず
  ライフが増えなかった（＝先の「net無反応」は退化盤アーティファクト・撤回）。
  富んだ盤面(leader_test_helpers.build・ライフ5)では **resolve_interaction も MCTS探索経路
  (OPCGGame.apply) もライフ追加を正しく完遂**（5→6）＝探索は追加の価値を見られる。
- 結果: SELECT_TARGET(min0/max1・上3候補) 発火。選択肢=追加3/見送り1 両提示。
  **出荷Gen2 は追加 5/5 seed**。接戦盤(低ライフ)では value net も追加を高評価
  （自ライフ1/相手4: 見送り0.960 < 追加0.995／自ライフ2/相手5: 0.990 < 0.999）。
  ※圧勝盤ではvalueが~1.0に飽和し無差別だが、ライフが効く局面では正しく選好。
- 判定: **✅ 解消**（旧=構造的に絶対追加しない→修正後=ログ局面で追加・netも選好）。

## 総括

実ログで観測された退行手2件は、いずれも**配線バグが原因**で、修正(#153・本番main)により
**同じ局面を再現すると正しい手（#1見送り・#2追加）を選ぶ**ことを確認した。検証①(0/40は虚構・
実デッキ勝ち越し)と合わせ、「出荷Gen2の既知の実問題」は解消済みと結論する。

残る未解決（検証①§5より）: 出荷決定化の透視疑い（勝率が情報優位に依存するか・公平決定化の
切り分けは cpu_learned MCTS経路でハングする別バグ）、および要件「vs本番L1 0.80」との差分。
