## 3. なぜ強さが動かなかったか（確定済みの因果・仮説ではない）

エコーは「教師が prior の写し」という**伝搬経路**の問題で、v7 はそこを切った。しかし根本は
**value が効率判断に無差別（±0.01〜0.05）＝ループ内に真実の信号が無い**こと。写し取りを
やめても、写すべき真実がループ内に存在しなければラベルはドリフトのまま——
29 ゲートの 0.42 フラットはその実測形。ラベル整形（D/E/F）では信号は生まれない。

## 4. 結論と処置

- **v7 の役割は「診断の完結」**: v5〜v7 で〈エコー→独立ランダムウォーク→深探索オラクルの
  ドリフト継承→ラベル整形でも不動〉まで因果を測定で埋め、「自己対戦の中に真実源が無い」を
  確定させた。これが v8（真盤面レフェリー＝外部の真実源・教師CPU本線）への転換根拠。
- 本走は停止（learner+w1〜w4 の外部セッションを終了）。p3ckpt・データ枝は git に残置＝
  再開・事後分析はいつでも可能。
- v6/v7 で作った昇格ゲート・アンカーゲート・エコー計器・マーク計測は、将来「レフェリー由来の
  外部ラベル」でネットを更新する日が来たときの品質防壁としてそのまま使う。
