## 読み取り（初回分類からの重要な更新）

1. **@82・@93 は VALUE_BOUND ではなく PRIOR_BOUND だった**。sims 深掘りだけの初回分類では
   「value が差を感じない」に見えたが、prior を一様にした同じ深探索は @93 を 3/3・@82 を 5/6 で
   正着する。つまり **value ネットは差を感じている。policy prior が正着の枝をほぼ0に潰しており、
   PUCT が深部でその枝を読まない**のが真因。誤りの住所は value でなく **policy** にあった。
2. **柱②（深探索再ラベル）が主要マークの担当に昇格**。PRIOR_BOUND は「探索が正着に到達すれば
   解ける」ので、再ラベルの深探索に **prior 平坦化（または温度/Dirichlet 強化）を必ず入れる**ことを
   条件に、@82/@93/g2@21 は柱②の射程に入る。再ラベルされた policy ターゲットが正着に質量を持てば、
   serve 時（通常 prior）にも正着が読まれるようになる——policy の盲点を policy の学習で塞ぐ正攻法。
3. **柱④（特徴設計）の需要は縮小**。残る VALUE_BOUND(hard) 3件（g1@24/@43・g2@38）は
   **L1 も 0.00**＝手作り評価すら人間指摘方向を選ばない。評価困難な局面か、復元盤面の構造的限界
   （実対局の文脈でのみ意味を持つマーク）か、人間の好みに近い判断の可能性があり、優先度を下げる
   （すべて旧 v4 プロファイルの歴史的マークで、現行 v5 ゲートの対象外）。
4. **補助所見**: @82 の L1=0.00 は「実対局の文脈では L1 第二意見が人間側だった」（v5計画 §2.1）と
   食い違う＝復元盤面と実文脈で L1 の判断が変わる局面。@82 の flat=0.83 という結果自体は堅いが、
   このマークの検証は実対局リプレイ文脈での再確認も併用するのが安全。

## v6 計画への反映

- **柱②の仕様が1つ確定**: 再ラベル探索は「深い sims × **prior 平坦化/探索強化**」で行う。
  同じ prior に誘導された深探索では PRIOR_BOUND マークは直らない（s3200 列が実証）。
- **柱④は当面着手不要**（担当マークが実質消えた）。L1 差分分析は VALUE_BOUND(hard) を
  再訪する時まで保留。
- 実装順は **①昇格ゲート → ⑤生成/serve分離 → ②再ラベル（prior平坦化つき）** に単純化。
