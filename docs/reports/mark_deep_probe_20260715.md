## 読み取り

1. **計器の判別力は実証された**: g1@16 が教科書どおりの EXPLORABLE 曲線（0→0.25→1.0）を示した
   ＝「探索の浅さが原因のマーク」は実在し、本計器で検出できる。柱②（深探索再ラベル）は
   この種のマークと一般較正には効く見込みを維持。
2. **しかし v5 の主要残課題 @82・@93 は VALUE_BOUND**: 現ネットの value を葉評価にする限り、
   sims を 20 倍にしても指摘方向を選ばない＝**誤りは探索の浅さでなく value/表現にある**。
   柱②の再ラベル（同じ net で深く読んだ policy ターゲット）では**この2件は直らない**。
   → **柱④（失敗モード逆算の特徴設計）がこの2件の担当**。v5 採用報告 §3-3 の仮説
   （決断文脈を判別する特徴の欠如）と整合。
3. 同じ「無駄ドン付与」でも g1@16 は EXPLORABLE・g3@93 は VALUE_BOUND ＝ 症状が同じでも
   原因層が違う。マークを症状でなく**原因層で分類してから機構を選ぶ**という v6 の設計原則を
   初回計測がそのまま裏書きした。
4. 補助所見: @82 は L1 第二意見が人間側に一致する既知事実（v5計画 §2.1）と合わせると、
   「手作り評価（L1）は差を感じるが学習 value は感じない」＝特徴群の表現力差が原因という
   仮説がさらに濃くなった。柱④の特徴候補設計では L1 評価項との差分分析が有力な出発点。

## v6 計画への反映

- 実装順は不変（①昇格ゲート→⑤分離→②再ラベル→④特徴）だが、**@82/@93 の改善見込みは
  ②でなく④に付け替える**。②の成功基準からマーク改善を外し「一般較正（C3 攻撃ノイズ系）と
  EXPLORABLE 系の改善」に限定する。
- ④の着手時: まず L1 評価項 vs value net の感応差分（@82/@93 局面で L1 が見ている量のうち
  encoder に無い/弱いもの）を特定してから特徴を設計する。
