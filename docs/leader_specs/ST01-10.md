対象ID（12枚）: ST01-001 / ST02-001 / ST03-001 / ST04-001 / ST05-001 / ST06-001 / ST07-001 / ST08-001 / ST09-001 / ST10-001 / ST10-002 / ST10-003

概要: スターターデッキ ST01〜ST10 のリーダー12枚をテキスト精読し、パース結果(AST要約)・実観測(classify)と突き合わせて意味バグを検出する。原文テキストが常に正。`classify` の `WARN: ... 方向不一致` はバフ/KO等が「自分の場の汎用キャラがいない／相手キャラがいない」等で方向検証ができない汎用盤面由来の警告が多く、それ単体ではバグの証拠にしない。

## ST01-001 モンキー・D・ルフィ
- **基本**: 色=赤 / ライフ=5 / パワー=5000 / 属性=打 / 特徴=超新星・麦わらの一味
- **効果テキスト**:
  > 【起動メイン】【ターン1回】このリーダーか自分のキャラ1枚にレストのドン!!1枚までを付与する。
- **能力分解（テキスト解釈）**:
  - 能力0 【ACTIVATE_MAIN / ターン1回】: 効果=ATTACH_DON / 対象=このリーダー or 自分のキャラ1枚(LEADER+CHARACTER, SELF) / 数値=ドン‼1枚まで(is_up_to) / 状態=レスト / 条件=なし / ターン1回=あり
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | アクティブなドン‼1枚以上 | 発動 | リーダーか自分キャラにレストのドン‼1枚付与(+1000) |
  | 2 | 付与先を選ばない | 発動 | 0枚付与可(まで=is_up_to) |
  | 3 | 2回目 | 発動 | 不可(ターン1回) |
  - 境界/異常系: 付与されたドン‼が**レスト状態**であること / 相手キャラは対象外
- **パース照合**: trigger=ACTIVATE_MAIN, TURN_LIMIT=1, ATTACH_DON target=SELF/LEADER+CHARACTER count=1 is_up_to=true, status=RESTED, value=1。対象(自分リーダー/キャラ)・数値・レスト状態・is_up_to・ターン1回すべてテキスト通り。
- **実観測照合**: EXECUTED, p1_don:10→9, stat: 対象 pw1000→2000(+1000)。ドン付与によるパワー上昇が機能。
- **判定**: ✅問題なし — 対象・数値・レスト状態・is_up_to・ターン1回まで整合。

## ST02-001 ユースタス・キッド
- **基本**: 色=緑 / ライフ=5 / パワー=5000 / 属性=特 / 特徴=超新星・キッド海賊団
- **効果テキスト**:
  > 【起動メイン】【ターン1回】③(コストエリアのドン!!を指定の数レストにできる)，自分の手札1枚を捨てることができる：このリーダーをアクティブにする。
- **能力分解（テキスト解釈）**:
  - 能力0 【ACTIVATE_MAIN / ターン1回 / コスト=ドン‼3レスト＋手札1枚捨て】: 効果=ACTIVE(このリーダーをアクティブ)
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | アクティブなドン‼3、手札1枚以上、リーダーがレスト(攻撃済) | 発動 | ドン‼3レスト＋手札1捨て→リーダーをアクティブ |
  | 2 | 手札0枚 | 発動 | コスト払えず不可 |
  | 3 | 2回目 | 発動 | 不可(ターン1回) |
  - 境界/異常系: アクティブなドン‼が3枚未満では発動不可
- **パース照合**: cost=seq[REST_DON value=1, DISCARD HAND count=1]、effect=ACTIVE target=SELF/FIELD count=1 select_mode=SOURCE。捨てコスト・ターン1回・対象(自リーダー)はテキスト通り。⚠️ コスト③＝ドン‼3枚レストだが REST_DON の value=1 と要約され、ドン枚数3との対応が見えない（他ST共通のコスト枚数表現懸念。value がアクション種別なのか枚数なのか要確認）。
- **実観測照合**: EXECUTED, p1_hand:5→4, p1_trash:10→11, ドン co0→1(レスト)。手札捨て・ドンレストは機能。`WARN: ACTIVE 期待だが方向不一致` は汎用盤面でリーダーが既にアクティブ等のため。
- **判定**: ⚠️要確認 — 効果自体は整合だが、コスト③のドン枚数(3)が要約 value=1 に表れず、ドン‼3枚レストが正しく要求されるか個別テスト要。

## ST03-001 クロコダイル
- **基本**: 色=青 / ライフ=5 / パワー=5000 / 属性=特 / 特徴=王下七武海・B・W
- **効果テキスト**:
  > 【起動メイン】【ターン1回】ドン!!-4(自分の場のドン!!を指定の数ドン!!デッキに戻すことができる)：コスト5以下のキャラ1枚までを、持ち主の手札に戻す。
- **能力分解（テキスト解釈）**:
  - 能力0 【ACTIVATE_MAIN / ターン1回 / コスト=ドン‼-4(デッキに戻す)】:
    - 効果 BOUNCE: 「コスト5以下のキャラ1枚まで」を**持ち主の手札**に戻す
    - **対象プレイヤー**: 原文は「コスト5以下のキャラ」＝相手・自分の指定なし＝**両者のキャラが対象**（"相手の"/"自分の" の限定なし）
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 場のドン‼4以上、**相手**にコスト5以下キャラ | 発動 | ドン‼4戻し→相手キャラを相手手札に戻す |
  | 2 | 場のドン‼4以上、**自分**にコスト5以下キャラのみ | 発動 | 自分キャラも対象に取れる（持ち主＝自分の手札へ） |
  | 3 | コスト6以上のキャラしかいない | 発動 | 対象不在で0枚(まで=is_up_to) |
  - 境界/異常系: 「持ち主の手札」＝戻すキャラの所有者の手札へ / 場のドン‼が4枚未満では発動不可
- **パース照合**: cost=RETURN_DON value=4, BOUNCE target=**player=OPPONENT**/CHARACTER cost_max=5 count=1 is_up_to=true。cost_max=5・is_up_to・「持ち主の手札」(BOUNCE=所有者へ)は正しい。🐛 **対象プレイヤーが OPPONENT に限定されている**が、原文は「コスト5以下のキャラ」で相手/自分の限定がなく**両プレイヤーのキャラ**が対象であるべき。自分のキャラを選べない＝対象範囲の欠落。
- **実観測照合**: EXECUTED, p1_don:10→6, p2_hand:3→4, p2_field:3→2。相手キャラのバウンスは機能するが、汎用盤面では相手キャラのみに作用しており、自分キャラを対象にできないバグは観測上も裏付く（target.player=OPPONENT 固定）。
- **判定**: 🐛バグ — BOUNCE の対象プレイヤーが OPPONENT 固定。テキストは「キャラ」で自分・相手両方が対象であり、対象範囲が欠落している。

## ST07-001 シャーロット・リンリン
- **基本**: 色=黄 / ライフ=5 / パワー=5000 / 属性=特 / 特徴=四皇・ビッグ・マム海賊団
- **効果テキスト**:
  > 【ドン‼×2】【アタック時】自分のライフの上か下から1枚を手札に加えることができる：自分のライフが2枚以下の場合、自分の手札1枚までを、ライフの上に加える。
- **能力分解（テキスト解釈）**:
  - 能力0 【ON_ATTACK / 常在(ドン‼×2) / コスト=自分ライフの上か下1枚を手札に加える(任意)】:
    - 条件: 自分のライフが2枚以下の場合
    - 効果 MOVE_CARD: 自分手札1枚までをライフの上(TOP)に加える
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | リーダーにドン‼2、ライフ2枚以下、手札1枚以上 | アタック | ライフ上下1枚を手札へ→手札1枚をライフ上に置く |
  | 2 | ライフ3枚以上 | アタック | 効果側は不発（条件未達） |
  | 3 | ドン‼1枚以下 | アタック | 発動不可 |
  - 境界/異常系: コスト(ライフ上下→手札)は条件と独立で先に発生 / 「上か下」の選択 / ライフ加える先はTOP固定
- **パース照合**: trigger=ON_ATTACK, condition=AND[HAS_DON≥2, LIFE_COUNT≤2], cost=MOVE_CARD(LIFE→HAND), effect=MOVE_CARD(HAND→LIFE TOP) is_up_to=true。⚠️ テキスト構造では「自分のライフが2枚以下の場合」は**効果側の条件**で、コスト(ライフ上下を手札に加える)はライフ枚数条件なしで実行可能。パースは LIFE_COUNT≤2 を発動全体の condition に統合しており、ライフ3枚以上だとコスト(ライフ操作)も含めて何も起きない可能性。原文では先のコスト操作は条件に依存しないため、厳密にはコスト/効果の条件適用範囲が食い違う懸念。dest_position=TOP・is_up_to は正しい。
- **実観測照合**: NO_CHANGE。ON_ATTACK かつライフ2枚以下条件が汎用盤面で未達のため発火せず。未実装の証拠ではない。
- **判定**: ⚠️要確認 — 「ライフ2枚以下」条件をコストにも掛けている疑い（コストは無条件のはず）。ON_ATTACK＋条件のため汎用盤面で検証不能。個別テスト要。

## ST08-001 モンキー・D・ルフィ
- **基本**: 色=黒 / ライフ=5 / パワー=5000 / 属性=打 / 特徴=麦わらの一味
- **効果テキスト**:
  > 【自分のターン中】キャラがKOされた時、このリーダーにレストのドン!!1枚までを、付与する。

  - 注: 「キャラがKOされた時」は自分/相手の限定なし＝**いずれかのキャラがKOされた時**（自軍キャラがKOされた時も含むのが通常裁定）。ターン1回の記載は**なし**。
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自分ターン中、キャラがKOされる | KO発生 | このリーダーにレストのドン‼1枚付与 |
  | 2 | 1ターンに複数回KO | KO発生×N | 都度発動（ターン1回制限なし） |

  - 境界/異常系: 付与は**このリーダー限定**（キャラには付与しない）/ レスト状態 / is_up_to(0枚も可)
- **パース照合**: trigger=YOUR_TURN, ATTACH_DON target=SELF/**LEADER+CHARACTER** count=1 is_up_to=true status=RESTED value=1。⚠️ テキストは付与先が「このリーダー」限定だが、target card_type に CHARACTER も含まれ自分キャラも対象に取れる解釈になっている（対象範囲が広すぎる懸念）。また「キャラがKOされた時」というKOトリガー条件が要約に表れず（condition=null）、KOイベント連動かどうかは要約からは判別不能。レスト状態・is_up_to・YOUR_TURN は正しい。
- **実観測照合**: EXECUTED, p1_don:10→9, stat: リーダー pw1000→2000(+1000)。ドン付与自体は機能。ただし classify はKOイベント無しで EXECUTED しており、「KOされた時」というイベント前提が効いているか不明（汎用発火）。
- **判定**: ⚠️要確認 — (1)付与先が「このリーダー」限定のはずが LEADER+CHARACTER、(2)「キャラがKOされた時」のKOトリガー条件が要約に現れない。意味バグの可能性があり個別テスト要。
