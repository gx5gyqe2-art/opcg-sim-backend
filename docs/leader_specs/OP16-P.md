## OP16-001 ポートガス・D・エース
- **基本**: 色=赤 / ライフ=5 / パワー=5000 / 属性=特 / 特徴=白ひげ海賊団
- **効果テキスト**:
  > 【起動メイン】【ターン1回】自分のパワー8000以上の、「モンキー・Ｄ・ルフィ」か『白ひげ海賊団』を含む特徴を持つキャラ1枚までは、このターン中、【速攻】を得る。
- **能力分解（テキスト解釈）**:
  - 能力0 【ACTIVATE_MAIN】（ターン1回）: 効果=GRANT_KEYWORD（速攻, THIS_TURN） / 対象=自分のキャラ（**パワー8000以上** かつ（名前「モンキー・D・ルフィ」 または 特徴《白ひげ海賊団》））1枚まで
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自分場にパワー8000以上の《白ひげ海賊団》キャラ | 能力発動 | そのキャラが今ターン速攻獲得 |
  | 2 | 自分場にパワー7000の《白ひげ海賊団》キャラのみ | 能力発動 | 対象外（is_up_to で0枚）→付与なし |
  | 3 | パワー8000以上だが名前/特徴いずれも非該当 | 能力発動 | 対象外 |
  - 境界/異常系: パワー8000ちょうど=対象 / 7000=非対象 / ターン1回
- **パース照合**: 🐛 **対象パワー条件が反転**。テキスト「パワー8000以上」だが、パース target は `power_max=8000`（power_min=None）。matcher は `card.get_power() > power_max → 除外`、すなわち **パワー8000以下を対象**としてしまい意味が真逆。原因は `matcher.py` のパワー抽出正規表現が `パワー(\d+)\D?(以下|以上)?` と `\D?` を挟むため「8000以上」の「以」を `\D?` が食べ「上」だけ残り group2 が None → else 分岐で `power_max` に設定される（cost 側 line168 は `\D?` が無く正しく cost_min を取得できる）。名前 names=["モンキー・D・ルフィ"]、traits=["白ひげ海賊団"] は整合。速攻/THIS_TURN/ターン1回も整合。
- **実観測照合**: EXECUTED（汎用盤面に該当キャラがあり付与、見た目変化なし）。ただし上記の通り対象集合が誤り（高パワーキャラを取れず低パワーを誤って取る）ため、実観測 EXECUTED は誤った対象選択の上で成立している可能性が高い。
- **判定**: 🐛 バグ — 「パワー8000以上」が `power_max=8000`（8000以下）として真逆にパースされる。

## OP16-041 バギー

- **効果テキスト**:

- **能力分解（テキスト解釈）**:
  - 能力0 【自動・トリガー】（条件: DON≥1, ターン1回, 自分の《インペルダウン》キャラが**場を離れた時**）: 効果=PLAY_CARD（手札の「インペルダウンの囚人」1枚まで→場）
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | DON1付与・手札に「インペルダウンの囚人」・自分《インペルダウン》キャラがKO/離脱 | 離脱イベント発生 | 「インペルダウンの囚人」を場に登場 |
  | 2 | 手札に「インペルダウンの囚人」なし | 同上 | 登場なし（is_up_to で0枚可） |
  | 3 | DON未付与 | 同上 | 発動不可 |
  - 境界/異常系: トリガー（場を離れた時）の捕捉 / ターン1回 / DON条件
- **パース照合**: 🐛/⚠️ **トリガー違い**。テキストは「自分の《インペルダウン》キャラが場を離れた時」の**自動誘発（LEAVE系トリガー）**だが、パースは `trigger=ACTIVATE_MAIN`（起動メイン）。発動契機が「場を離れた時」→ プレイヤーが任意に起動メインで撃つ能力に化けている。条件側にも「《インペルダウン》キャラが場を離れた」という誘発イベント条件が無く、AND(TURN_LIMIT, HAS_DON≥1) のみ。効果 PLAY_CARD（names=["インペルダウンの囚人"], is_up_to）自体はテキスト整合。
- **実観測照合**: NO_CHANGE。汎用盤面で手札に「インペルダウンの囚人」不在のため未発火と整合。ただしトリガー自体が起動メイン化しているため、本来の「離脱時に発動」契機は検証不能。
- **判定**: 🐛 バグ — 誘発条件「自分の《インペルダウン》キャラが場を離れた時」が欠落し、ACTIVATE_MAIN として起動能力化（トリガー種別・条件節欠落）。
