# リーダーカード効果 テスト仕様書 — OP10 / OP11

対象ID（12枚）:
- OP10: OP10-001 / OP10-002 / OP10-003 / OP10-022 / OP10-042 / OP10-099
- OP11: OP11-001 / OP11-021 / OP11-022 / OP11-040 / OP11-041 / OP11-062

概要: 各リーダーの効果テキスト（原文が正）を精読し、パーサのAST（`summarize_ability` 指紋＋実AST）および
汎用盤面での実観測（`effect_coverage.classify`）と突き合わせて意味バグを検出する。
パーサ指紋に出ない `condition.target` / `effect.target` は実AST（`leader_spec_probe` のオブジェクトダンプ）で裏取りした。

## OP10-001 スモーカー
- **基本**: 色=赤/緑 / ライフ=4 / パワー=5000 / 属性=斬 / 特徴=パンクハザード/海軍
- **効果テキスト**:
  > 【相手のターン中】自分の特徴《海軍》か《パンクハザード》を持つキャラすべてを、パワー+1000。 / 【起動メイン】【ターン1回】自分のパワー7000以上のキャラがいる場合、自分のドン‼2枚までを、アクティブにする。
- **能力分解（テキスト解釈）**:
  - 能力0 【相手のターン中/PASSIVE】: 効果=BUFF / 対象=自分の《海軍》or《パンクハザード》キャラ全部 / 数値=+1000 / 条件=なし
  - 能力1 【起動メイン】: 効果=ACTIVE_DON 2枚まで / 条件=「自分のパワー**7000以上**のキャラがいる」+ ターン1回
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | パワー8000のキャラ1体あり, レストドン2 | 能力1発動 | ドン2枚アクティブ化 |
  | 2 | パワー6000以下のキャラのみ | 能力1発動 | 条件未達で発動不可（NO_CHANGE） |
  - 境界: パワーちょうど7000のキャラのみ → 発動可であるべき
- **パース照合**: 能力0=OK。**能力1の条件が `FIELD_COUNT GE 1` の `target.power_max=7000`**。テキストは「7000**以上**」(GE→power_min=7000) のはず。`power_max=7000` だと「7000**以下**のキャラがいる」という**逆の条件**になり、5000リーダー横の弱小キャラでも成立してしまう。
- **実観測照合**: ACTIVATE_MAIN=EXECUTED（汎用盤面で発火）。条件の逆転は盤面差分では判別不能。
- **判定**: 🐛 — 能力1条件のパワー比較が反転（power_min 7000 のはずが power_max 7000）。
