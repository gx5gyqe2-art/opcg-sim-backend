## OP04-040 クイーン
- **基本**: 色=青/黄 / ライフ=4 / パワー=5000 / 属性=射 / 特徴=百獣海賊団
- **効果テキスト**:
  > 【ドン!!×1】【アタック時】自分のライフと手札の合計枚数が4枚以下の場合、カード1枚を引く。自分のコスト8以上のキャラがいる場合、カード1枚を引く代わりに自分のデッキの上から1枚までを、ライフの上に加えることができる。
- **能力分解**:
  - 能力0 【ドン×1/アタック時】:
    - 条件=DON>=1
    - 基本=（ライフ＋手札の合計≤4 の場合）1ドロー
    - 置換=（コスト8以上の自キャラがいる場合）**ドローの代わりに**デッキ上1枚までをライフに加えることができる(任意)
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | ライフ+手札≤4、コスト8以上キャラ無 | アタック | 1ドロー |
  | 2 | ライフ+手札≤4、コスト8以上キャラ有 | アタック | ドローの代わりに上1枚をライフへ(任意) |
  | 3 | ライフ+手札5枚以上 | アタック | 何も起きない |
- **パース照合**: 🐛複数の食い違い。
  1. 条件が **LIFE_COUNT LE 4**（ライフのみ）。テキストは「ライフ**と手札の合計**4枚以下」＝ライフ＋手札の合算。手札枚数が合算されておらず**条件節欠落**。
  2. 置換側 branch 条件が **FIELD_COUNT GE 1**（キャラがいれば）。テキストは「**コスト8以上**のキャラがいる場合」＝cost_min=8 のフィルタが欠落。
  3. 「カード1枚を引く**代わりに**」の置換関係が、独立した2つのbranch(DRAW と HEAL)に分解され、両条件成立時に**ドローもHEALも両方起きる**構造。本来は択一(代わりに)。
  4. HEAL「上から1枚**までを**」の任意性(is_up_to/「できる」)が HEAL value=1 固定で表現できておらず（強制ヒール化の疑い）。
- **実観測照合**: ON_ATTACK=NO_CHANGE（汎用盤面でライフ条件未達のため未発火）。条件ロジック自体の誤りは上記パースで確認済み。
- **判定**: 🐛バグ — 条件「手札との合計」欠落・「コスト8以上」欠落・「代わりに」の択一が両発動になる分解。
