# リーダーカード効果 テスト仕様書 — EB（エクストラブースター）/ PRB（プレミアムブースター）

対象7枚（テキスト精読・意味検証）:

- EB01: EB01-001, EB01-021, EB01-040
- EB02: EB02-010
- EB03: EB03-001
- EB04: EB04-001
- PRB01: PRB01-001

概要: 各リーダーの効果テキスト（原文が正）を精読し、`leader_spec_probe.py` のパース結果（`summarize_ability` 指紋）および classify 実観測と突き合わせ、意味バグ（対象逆/条件欠落/数値違い/対象プレイヤー違い/トリガー違い/未実装NO_CHANGE 等）を検出する。末尾に `## 検出サマリ`。

注: `summarize_ability` 指紋は条件側 `target` のフィルタ（trait/cost）を省略表示する。FIELD_COUNT 等の条件は parse_target にフィルタを委譲するため、指紋に trait/cost が出なくても実装上効いている可能性がある（断定不可は⚠️）。

## EB01-001 光月おでん
- **基本**: 色=赤/緑 / ライフ=4 / パワー=5000 / 属性=斬 / 特徴=ワノ国・光月家
- **効果テキスト**:
  > ルール上、自分の特徴《ワノ国》を持ちカウンターを持たないキャラカードすべてはカウンター+1000を持つ。 / 【ドン!!×1】【アタック時】自分のコスト5以上の特徴《ワノ国》を持つキャラがいる場合、このリーダーは、次の自分のターン開始時まで、パワー+1000。
- **能力分解（テキスト解釈）**:
  - 能力0 【PASSIVE】: ルール処理（RULE_PROCESSING）。自分の《ワノ国》かつカウンター非所持キャラすべてにカウンター+1000を付与。
  - 能力1 【ON_ATTACK】（条件: DON≥1 AND 自分場に「コスト5以上の《ワノ国》キャラ」が存在）: 効果=BUFF 自リーダー +1000 / 期間=次の自分のターン開始時まで（UNTIL_NEXT_TURN_END相当）
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | DON1付与・自分場にコスト5以上《ワノ国》キャラ有 | アタック宣言 | リーダー +1000（次自ターン開始時まで） |
  | 2 | 自分場に《ワノ国》キャラはいるがコスト5未満のみ | アタック宣言 | 不発（条件未達） |
  | 3 | 自分場の《ワノ国》キャラが《ワノ国》以外特徴のみ | アタック宣言 | 不発 |
  | 4 | 自分場に《ワノ国》カウンター非所持キャラ | 常時 | 当該キャラがカウンター+1000を獲得 |
  - 境界/異常系: コスト5境界（5は条件成立/4は不成立） / DON未付与時の不発 / 持続が「次の自分のターン開始時まで」で正しく失効するか
- **パース照合**: 能力1の条件は AND(HAS_DON≥1, FIELD_COUNT≥1)。テキストの「コスト5以上の特徴《ワノ国》を持つキャラがいる場合」のうち、指紋上 FIELD_COUNT に cost_min=5 / traits=ワノ国 が**明示されていない**。parse_target へ委譲される設計のため実装で効いている可能性はあるが、汎用盤面で確認できず断定不可。BUFF +1000・対象SOURCE・期間は整合。
- **実観測照合**: PASSIVE=EXECUTED（カウンター付与ルール、見た目差分なし）。ON_ATTACK=NO_CHANGE（汎用盤面に「コスト5以上《ワノ国》キャラ」不在＝条件未達と整合、未実装ではない）。
- **判定**: ⚠️ 要確認 — 効果方向・数値は整合。FIELD_COUNT 条件の「コスト5以上 かつ 《ワノ国》」フィルタが実装で適用されるか個別テスト推奨。

## EB03-001 ネフェルタリ・ビビ
- **基本**: 色=赤/青 / ライフ=4 / パワー=5000 / 属性=斬 / 特徴=アラバスタ王国
- **効果テキスト**:
  > 【ターン1回】自分の元々のコスト4以上のキャラがKOされる場合、代わりに自分の手札1枚を捨てることができる。 / 【起動メイン】このリーダーをレストにできる：相手のキャラ1枚までを、このターン中、パワー-2000。その後、自分の【アタック時】効果を持たないキャラ1枚までは、このターン中、【速攻】を得る。
- **能力分解（テキスト解釈）**:
  - 能力0 【PASSIVE】（ターン1回）置換効果: 自分の「元々のコスト4以上」のキャラがKOされる場合、代わりに自分の手札1枚を捨てることができる（KO置換）
  - 能力1 【ACTIVATE_MAIN】（コスト: REST 自リーダー）: seq = 相手キャラ1枚まで THIS_TURN パワー-2000 → その後 自分の**【アタック時】効果を持たない**キャラ1枚まで THIS_TURN【速攻】付与
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自分の元々コスト4以上キャラがKOされる・手札有 | 置換選択 | KOを防ぎ代わりに手札1枚捨て |
  | 2 | リーダーアクティブ・相手キャラ有・自分にアタック時効果なしキャラ有 | 起動メイン発動 | リーダーレスト→相手-2000→アタック時効果なしキャラに速攻付与 |
  | 3 | 自分のキャラが全て【アタック時】効果持ち | 起動メイン発動 | 速攻付与の対象なし（is_up_to 0枚） |
  | 4 | 自分の元々コスト3以下キャラがKO | 置換 | 置換不可（コスト4未満は対象外） |
  - 境界/異常系: 「元々のコスト4以上」=master.cost 参照 / コスト4境界 / 置換のターン1回 / 速攻対象から【アタック時】効果持ちを除外できるか
- **パース照合**: 
  - 能力0: REPLACE_EFFECT, sub=DISCARD(self hand 1), status=LEAVE, 条件 AND(TURN_LIMIT, GENERIC)。「元々のコスト4以上」フィルタが置換対象に効いているかは指紋上不明（GENERIC条件で表現、要裏取り）。⚠️
  - 能力1: REST(self leader) → BUFF(opponent -2000 THIS_TURN is_up_to) → GRANT_KEYWORD(self 速攻 THIS_TURN is_up_to)。**問題**: GRANT_KEYWORD の target に「【アタック時】効果を持たない」フィルタが**ない**（target は zone=FIELD/player=SELF/card_type=CHARACTER のみ）。`TargetQuery` には trigger 能力で除外するフィールドが存在せず（effect_types.py: TargetQuery にトリガー除外フィールドなし、matcher は「【トリガー】を持つ」フィルタのみ実装）、「【アタック時】効果を持たない」制限が**取りこぼし**。本来アタック時効果持ちキャラには速攻を付与できない。
- **実観測照合**: PASSIVE=EXECUTED（置換登録）。ACTIVATE_MAIN=EXECUTED（リーダーrest, 相手キャラ pw1000→-1000(=-2000), 別キャラ kw速攻）。速攻はアタック時効果の有無に関わらず付与され得る点が問題。
- **判定**: 🐛 バグ — 能力1の「自分の【アタック時】効果を持たないキャラ」という対象制限が欠落し、アタック時効果持ちキャラにも速攻が付く。加えて能力0の「元々のコスト4以上」フィルタが置換対象に効くか要裏取り（⚠️）。

## PRB01-001 サンジ
- **基本**: 色=赤 / ライフ=5 / パワー=5000 / 属性=打 / 特徴=麦わらの一味
- **効果テキスト**:
  > 【起動メイン】【ターン1回】自分のコスト8以下の【登場時】効果を持たないキャラ1枚までは、このターン中、【速攻】を得る。 / (このカードは登場したターンにアタックできる)
- **能力分解（テキスト解釈）**:
  - 能力0 【ACTIVATE_MAIN(起動メイン)】（ターン1回）: 効果=GRANT_KEYWORD 自分の「コスト8以下 かつ【登場時】効果を持たない」キャラ1枚まで THIS_TURN【速攻】付与
  - 注: 末尾「(このカードは登場したターンにアタックできる)」はリーダー向けのリマインド文で実効果ではない。
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自分にコスト8以下・登場時効果なしキャラ有 | 起動メイン | 当該キャラに速攻付与（今ターン） |
  | 2 | 自分のキャラが全て【登場時】効果持ち | 起動メイン | 速攻付与の対象なし（is_up_to 0枚） |
  | 3 | コスト9以上キャラのみ | 起動メイン | 対象なし（コスト8以下フィルタ） |
  | 4 | 同一ターン2回目 | 起動メイン | 不可（ターン1回） |
  - 境界/異常系: コスト8境界（8は対象/9は非対象） / 登場時効果持ちキャラの除外 / 速攻が THIS_TURN 失効
- **パース照合**: **2点問題**。
  - (a) **トリガー違い**: テキストは【起動メイン】だが指紋は `trigger=ON_PLAY`。リーダーの起動メイン能力が ON_PLAY（登場時）として解釈されている。リマインド文の「登場した」誤検出が原因の可能性。リーダーに登場時はないため、本来手動発動すべき能力の発火タイミングが誤る恐れ。
  - (b) **対象制限欠落**: GRANT_KEYWORD の target に cost_max=8 は入っているが、「【登場時】効果を持たない」フィルタが**ない**（TargetQuery にトリガー能力除外フィールドが存在しない、matcher も未対応）。登場時効果持ちキャラにも速攻が付き得る。
- **実観測照合**: ON_PLAY=EXECUTED — `p1_field:3→4`（？盤面枚数変化）, `kw速攻`。速攻付与は走るが (a) トリガー区分が ON_PLAY、(b) 登場時効果フィルタ未適用。
- **判定**: 🐛 バグ — (a) 【起動メイン】が ON_PLAY と解釈（トリガー違い）、(b)「【登場時】効果を持たない」対象制限が欠落。いずれも要修正。
