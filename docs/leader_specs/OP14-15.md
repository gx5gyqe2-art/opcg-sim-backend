# リーダーカード効果 テスト仕様書 — OP14 / OP15

対象ID（13枚）:
- OP14: OP14-001, OP14-020, OP14-040, OP14-041, OP14-060, OP14-079, OP14-080
- OP15: OP15-001, OP15-002, OP15-022, OP15-039, OP15-058, OP15-098

概要: 各リーダーをテキスト精読し、パース結果(AST要約)・汎用盤面classify観測と突き合わせ、
意味バグ（対象逆・数値違い・条件節欠落・未実装NO_CHANGE・is_up_to/期間/「元々の」「持ち主の」取りこぼし等）を検出する。
テキストが常に正。NO_CHANGE は未実装(🐛)か汎用盤面で未発火(⚠️)かを切り分ける。

## OP14-001 トラファルガー・ロー
- **基本**: 色=赤 / ライフ=5 / パワー=5000 / 属性=斬 / 特徴=王下七武海・超新星・ハートの海賊団
- **効果テキスト**:
  > 【起動メイン】【ターン1回】自分の特徴《超新星》か《ハートの海賊団》を持つキャラ2枚を選ぶ。選んだキャラそれぞれの元々のパワーを、このターン中、入れ替える。
- **能力分解（テキスト解釈）**:
  - 能力0 【起動メイン/ターン1回】: 効果=《超新星》か《ハートの海賊団》キャラ2枚を選択→2枚の**元々のパワー**をTHIS_TURN入れ替え / ターン1回=○
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 該当キャラがパワー2000と7000の2体 | 能力 | 2体の元々パワーが入れ替わる(THIS_TURN) |
  | 2 | 該当キャラ1体のみ | 能力 | 対象2枚選べず発動不可(count=2固定) |
  | 3 | 同ターン2回目 | 能力 | ターン1回で不可 |
- **パース照合**: SELECT traits=["超新星","ハートの海賊団"]/count=2、SWAP_POWER ref=selected_card/THIS_TURN。テキスト通り。count=2固定（is_up_to=false）も「2枚を選ぶ」と整合。「元々のパワー」もSWAP_POWERで表現される想定。
- **実観測照合**: ACTIVATE_MAIN=INTERACTIVE（2枚選択待ちでsmart-resolve、汎用盤面に該当キャラ不在のため状態変化なし）。選択を要する効果として妥当。
- **判定**: ✅問題なし — パース・観測ともテキストと整合（選択依存で個別テスト要）。

## OP14-020 ジュラキュール・ミホーク
- **基本**: 色=緑 / ライフ=5 / パワー=5000 / 属性=斬 / 特徴=王下七武海
- **効果テキスト**:
  > 相手のリーダーが属性(斬)を持つ場合、このリーダーのパワー+1000。 / 【起動メイン】【ターン1回】自分のカード1枚をレストにできる：コスト5以上のキャラがいる場合、自分のドン‼3枚までを、アクティブにする。その後、自分は、このターン中、キャラカードを登場できない。
- **能力分解**:
  - 能力0 【PASSIVE】: 条件=相手リーダーが属性《斬》を持つ / 効果=このリーダー パワー+1000
  - 能力1 【起動メイン/ターン1回】: コスト=自分のカード1枚をレスト(任意) / 条件=**コスト5以上の**キャラがいる場合 / 効果=自分のドン!!3枚までをアクティブ→その後THIS_TURNキャラ登場不可 / is_up_to=○
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 相手リーダー属性=斬 | 常時 | このリーダー+1000 |
  | 2 | レストドンあり、自場にコスト5以上キャラ | 能力1 | カード1枚レスト、ドン3枚までアクティブ、登場不可付与 |
  | 3 | 自場がコスト4以下のキャラのみ | 能力1 | レストは発生するがドンアクティブ化されない(条件未達) |
- **パース照合**: 🐛 能力1の条件が **FIELD_COUNT GE 1**（キャラが1体以上いれば）。テキストは「**コスト5以上の**キャラがいる場合」＝cost_min=5 のフィルタが欠落（条件節欠落）。コスト4以下のキャラしかいなくてもドンアクティブ化が走る過剰発動の懸念。能力0 LEADER_ATTRIBUTE=斬/OPPONENT、BUFF SOURCE +1000 は正。ACTIVE_DON value=3、RULE_PROCESSING CANNOT_PLAY_CHARACTER/THIS_TURN も正。
- **実観測照合**: PASSIVE=EXECUTED（pw5000→6000、汎用盤面の相手リーダーが斬のため発火）。ACTIVATE_MAIN=EXECUTED（カード1枚rest、ドンは既にアクティブのため可視差分なし）。条件欠落自体はパースで確認。
- **判定**: 🐛バグ — 能力1の条件「コスト5以上のキャラがいる場合」のコスト5フィルタが欠落（FIELD_COUNT>=1化）。

## OP14-041 ボア・ハンコック
- **基本**: 色=青/黄 / ライフ=4 / パワー=5000 / 属性=特 / 特徴=王下七武海・九蛇海賊団
- **効果テキスト**:
  > 【相手のターン中】自分のキャラが登場した時、カード1枚を引く。 / 【ドン‼×1】【ターン1回】自分の元々のパワー5000以上の、特徴《アマゾン・リリー》か《九蛇海賊団》を持つキャラがKOされた時、相手のライフの上から1枚までを、持ち主の手札に加える。
- **能力分解**:
  - 能力0 【相手のターン中】: 発火=自分のキャラが登場した時 / 効果=1ドロー
  - 能力1 【ドン×1/ターン1回】: 発火=自分の**元々のパワー5000以上**の《アマゾン・リリー》か《九蛇海賊団》キャラがKOされた時 / 条件=DON>=1 / 効果=相手のライフの上から1枚までを**持ち主の(=相手の)**手札に加える / is_up_to=○
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 相手ターン、自キャラ登場 | 登場時 | 1ドロー |
  | 2 | 付与ドン1、元々パワー5000以上の九蛇キャラがKO | KO時 | 相手ライフ上1枚までを**相手の手札**へ |
  | 3 | KOされたのが元々パワー4000のキャラ | — | 発火せず |
- **パース照合**: 🐛複数の食い違い。
  1. 能力1 が **trigger=ACTIVATE_MAIN（起動メイン）** になっている。テキストは「キャラがKOされた時」＝KO誘発(PASSIVE/トリガー)。発火タイミングが起動メインに化けている。
  2. 効果 MOVE_CARD が **zone=LIFE/player=OPPONENT → destination=HAND**。テキストは「相手のライフの上から1枚までを、**持ち主の**手札に加える」。ライフの持ち主は相手なので手札も**相手(OPPONENT)の手札**であるべき。MOVE_CARDのdestinationが移動先プレイヤーを区別していないなら、相手ライフを**自分の**手札に加える＝対象プレイヤー違いの疑い。
  3. MOVE_CARD target に **card_type=["CHARACTER"], traits=["アマゾン・リリー","九蛇海賊団"], power_max=5000** が付いている。これは KO される**キャラ側の発火条件**（元々パワー5000以上）であるべきものが、**移動対象（相手ライフ）のフィルタ**に誤って付与されている。さらに power_max=5000 はテキスト「パワー5000**以上**」(min)と逆で**power_min** であるべき（max/min 逆＋付与先誤り）。
  4. 発火条件「元々のパワー5000以上の…キャラがKOされた時」がトリガー条件として表現されず、上記の通り移動対象フィルタに紛れている。
- **実観測照合**: OPPONENT_TURN=EXECUTED（自キャラ登場ドロー相当、p1_hand 5→6/p1_deck 20→19）。ACTIVATE_MAIN=NO_CHANGE（相手ライフフィルタにキャラ特徴/power_maxが付き、汎用盤面の相手ライフが該当せず未発火）。パース上の対象・トリガー誤りが NO_CHANGE の主因と見られる。
- **判定**: 🐛バグ — 能力1がKO誘発でなくACTIVATE_MAIN化、KO条件フィルタが移動対象(相手ライフ)に誤付与＋power_max(≤5000)で5000以上(min)と逆、「持ち主の手札」が区別されず対象プレイヤー違いの疑い。
