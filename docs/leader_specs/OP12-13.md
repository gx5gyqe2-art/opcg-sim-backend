## OP12-001 シルバーズ・レイリー
- **基本**: 色=赤 / ライフ=5 / パワー=5000 / 属性=斬 / 特徴=元ロジャー海賊団
- **効果テキスト**:
  > ルール上、自分はコスト5以上のカードをデッキに入れることができない。 / 【起動メイン】【ターン1回】自分の手札からイベント2枚を公開することができる：自分の元々のパワー4000以下のキャラ1枚までを、このターン中、パワー+2000。
- **能力分解（テキスト解釈）**:
  - 能力0 【常時/ルール】: コスト5以上のカードをデッキに入れられない(RULE_PROCESSING)
  - 能力1 【起動メイン】【ターン1回】: コスト=手札イベント**2枚ちょうど**を公開 / 効果=自分の元々のパワー4000以下のキャラ1枚までを、このターン中、パワー+2000 / 対象=自フィールド(CHARACTER,元々power≤4000) / 数値=+2000 / 期間=THIS_TURN / 条件=ターン1回
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 手札にイベント2枚・元々パワー4000のキャラ | 発動 | イベント2枚公開、対象キャラ+2000(このターン) |
  | 2 | 手札にイベント1枚のみ | 発動 | コスト(2枚公開)払えず発動不可 |
  | 3 | 元々パワー5000のキャラのみ | 発動 | 対象不在(1枚まで=0可) |
  - 境界/異常系: 公開は2枚ちょうど(任意枚数不可) / 「元々の」パワーで判定(バフ後5000でも元々4000なら対象)
- **パース照合**: cost=REVEAL(HAND,EVENT,count=2)だが **`is_up_to:true`** になっている。テキストは「イベント2枚を公開」＝2枚ちょうど。is_up_toにより1枚や0枚公開で発動できてしまう疑い。effect=BUFF(SELF,CHARACTER,power_max=4000,+2000,THIS_TURN)は整合。「元々の」パワー判定はpower_maxで表現され妥当。
- **実観測照合**: ACTIVATE_MAIN `EXECUTED` pw1000→3000(+2000)。バフ自体は正しく適用。
- **判定**: ⚠️ 要確認 — コスト公開枚数が`is_up_to:true`で「2枚ちょうど」が緩和されている疑い、1枚/0枚公開で発動できないか要個別テスト。

## OP12-020 ロロノア・ゾロ
- **基本**: 色=緑 / ライフ=5 / パワー=5000 / 属性=斬 / 特徴=超新星・麦わらの一味
- **効果テキスト**:
  > 【ドン‼×3】【起動メイン】【ターン1回】このターン中、このリーダーが相手のキャラとバトルしている場合、このリーダーをアクティブにする。その後、このリーダーは、このターン中、相手の元々のコスト7以下のキャラへアタックできない。
- **能力分解（テキスト解釈）**:
  - 能力0 【ドン‼×3】【起動メイン】【ターン1回】: 条件=ドン!!3付与＋ターン1回 / 効果1=このターン中このリーダーが相手キャラとバトルしている場合、このリーダーをアクティブにする / 効果2=その後このターン中、このリーダーは相手の元々のコスト7以下のキャラへアタックできない(アタック制限) / 期間=THIS_TURN
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | ドン3付与・このリーダーが相手キャラとバトル中 | 発動 | リーダーactive、以後コスト7以下キャラへアタック不可 |
  | 2 | ドン3付与・バトルしていない | 発動 | アクティブ化されず、アタック制限のみ付与 |
  | 3 | ドン2のみ | 発動 | 条件未達で不発 |
  - 境界/異常系: 「相手の元々のコスト7以下」=コスト8以上にはアタック可 / アタック制限は自リーダーへの制約(対象選択ではない)
- **パース照合**: condition=AND[TURN_LIMIT EQ1, HAS_DON GE3]は整合。branch条件が`FIELD_COUNT OPPONENT GE 1`＝「相手がキャラを1枚以上持つ」になっており、テキストの **「このリーダーが相手のキャラとバトルしている場合」** という"バトル中"の限定が落ち、単に相手がキャラを持っていればアクティブ化する緩和の疑い。ATTACK_DISABLEのtargetが`OPPONENT,cost_max=7,count=1,select=CHOOSE`＝相手キャラ1枚を選んで無効化、の形だが、テキストは **「このリーダーがコスト7以下キャラへアタックできない」**＝自リーダーへのアタック制限(対象選択ではなく全コスト7以下が対象外)。対象/意味の取り違えの疑い。「元々の」コスト判定も要確認。
- **実観測照合**: ACTIVATE_MAIN `NO_CHANGE`。ドン3・バトル中条件が汎用盤面で未達のため未発火と推定。
- **判定**: 🐛 バグ — branch条件が「バトル中」でなく「相手キャラ存在」に緩和、かつATTACK_DISABLEが自リーダー制限でなく相手キャラ1枚選択になっている疑い。

## OP12-040 クザン
- **基本**: 色=青 / ライフ=5 / パワー=5000 / 属性=特 / 特徴=海軍
- **効果テキスト**:
  > 自分の特徴《海軍》を持つカードの効果で自分の手札からカードが捨てられた時、捨てた枚数分カードを引く。
- **能力分解（テキスト解釈）**:
  - 能力0 【誘発/常時】: トリガー=自分の特徴《海軍》カードの効果で自分の手札からカードが捨てられた時 / 効果=捨てた枚数分カードを引く / 数値=捨て枚数分(動的) / 条件=《海軍》カード効果起因の捨て
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 《海軍》カードの効果で手札3枚捨て | 捨て発生 | 3枚引く |
  | 2 | 《海軍》以外のカード効果で手札捨て | 捨て発生 | 引かない(対象外) |
  | 3 | コスト支払い等の自発捨て(効果以外) | — | 引かない |
  - 境界/異常系: 捨て枚数=引く枚数の一致 / 《海軍》起因限定 / 「効果で」起因限定
- **パース照合**: trigger=PASSIVE, effect=DRAW(value=1固定)。**(a) 誘発トリガー「《海軍》カードの効果で手札が捨てられた時」が指紋に表現されず**、PASSIVE常時扱い。**(b) 引く枚数が固定1**で「捨てた枚数分」(動的)が落ちている。**(c) 《海軍》特徴・「効果で」起因の条件が欠落**。
- **実観測照合**: PASSIVE `EXECUTED` p1_hand:5→6, p1_deck:20→19(1枚ドロー)。汎用盤面で無条件に1枚引いている＝誘発条件が機能していない兆候。
- **判定**: 🐛 バグ — 誘発条件(《海軍》効果起因の捨て)欠落＋固定1枚ドローで「捨てた枚数分」が未実装、PASSIVE常時ドロー化の疑い。

## OP12-081 コアラ
- **基本**: 色=黒/黄 / ライフ=4 / パワー=5000 / 属性=打 / 特徴=ドレスローザ・革命軍
- **効果テキスト**:
  > このリーダーが相手のリーダーにアタックした時、自分のコスト8以上のキャラが2枚以上いる場合、カード1枚を引く。 / 【ターン1回】相手が、元々のコスト8以上のキャラを登場させた時かキャラの効果でキャラを登場させた時、発動できる。相手は自身のライフの上から1枚を、手札に加える。
- **能力分解（テキスト解釈）**:
  - 能力0 【アタック時(相手リーダーへ)】: 条件=自分のコスト8以上キャラが2枚以上 / 効果=カード1枚を引く / 対象=自分(SELF) / 数値=1
  - 能力1 【ターン1回】【誘発】: トリガー=相手が元々コスト8以上のキャラを登場させた時、または相手がキャラ効果でキャラを登場させた時 / 効果=相手は自身のライフの上1枚を手札に加える / 対象=相手のライフ→相手の手札 / ターン1回
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自分にコスト8以上キャラ2枚・相手リーダーへアタック | アタック時 | 1枚引く |
  | 2 | 自分にコスト8以上キャラ1枚 | アタック時 | 条件未達で引かない |
  | 3 | 相手が元々コスト8以上キャラを登場 | 誘発 | 相手が自ライフ上1枚を手札へ(ターン1回) |
  - 境界/異常系: 能力0は「自分の」コスト8以上が2枚以上(相手ではない) / 能力1は相手のライフ→相手の手札 / 「相手リーダーへの」アタック限定
- **パース照合**: 能力0 condition=`FIELD_COUNT OPPONENT GE 2`＝**プレイヤーが相手(OPPONENT)になっている**。テキストは「**自分の**コスト8以上のキャラが2枚以上」＝SELF。**対象プレイヤー逆**。さらに**「コスト8以上」の絞り込みが欠落**(単なるキャラ2枚以上)。「相手のリーダーにアタックした時」のアタック先限定も指紋に無い。能力1 **trigger=ACTIVATE_MAIN**＝**起動メイン化**しており、テキストの「相手がコスト8以上キャラを登場させた時/効果でキャラ登場時」という相手起因の誘発トリガーが失われている疑い。effectのMOVE_CARD(LIFE OPPONENT→HAND)は「相手のライフ→相手手札」で対象は整合。
- **実観測照合**: 能力0 `ON_ATTACK: NO_CHANGE`(相手キャラ2枚未満で未発火)。能力1はACTIVATE_MAIN扱いだが観測出力なし(誘発経路未走行)。
- **判定**: 🐛 バグ — 能力0は条件プレイヤーが逆(OPPONENT→本来SELF)かつ「コスト8以上」絞り込み欠落、能力1は誘発トリガーがACTIVATE_MAIN化している疑い。
