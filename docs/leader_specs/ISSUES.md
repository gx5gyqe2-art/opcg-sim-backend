# リーダー効果 検出バグ一覧（根本原因別）

全137リーダーの仕様書（各セットファイル）から 🐛 を集約し、根本原因パターン別に
整理したもの。修正は **パターン単位** で行うと効率がよい（1修正で複数カードが直る）。
各カードの詳細・テストケースはセットファイルの該当節を参照。

## A. パワー「以上」が常に「以下」として解釈される（不等号反転）✅ 修正済み

**根本原因（裏取り済み）**: `opcg_sim/src/core/effects/matcher.py:209` の
パワー抽出正規表現 `パワー(\d+)\D?(以下|以上)?` で `\D?` が「以上」の「以」を
先に食ってしまい group2 が None → else 分岐で `power_min` でなく `power_max` に
設定される。cost 側(line 168)は `\D?` が無く正常。

**修正（2026-06）**: matcher.py:209 から `\D?` を除去し cost 側と同形に揃えた。
下表の単純ケース（OP10-001/003・OP16-001・P-086・ST13-001）と、複合だが
パワー条件のみが原因だった ST30-001 の power 条件は解消（xfail マーカー除去済み・
通常テスト化）。全カード挙動ベースラインで「パワーN以上」フィルタを持つ 18 能力が
正方向（≥N）へ是正された（`full_card_baseline.json` 再生成済み）。
**残り**: OP14-041 は群 B（KO 誘発が ACTIVATE_MAIN 化）が主因のため未解消、
ST30-001 の対象名欠落は群 E のため未解消（いずれも xfail 継続）。

| ID | 内容 |
|----|------|
| OP10-001 | 条件「パワー7000以上」が power_max=7000 |
| OP10-003 | 条件「パワー6000以上」が power_max=6000 |
| OP16-001 | 「パワー8000以上」が power_max=8000 |
| P-086 | コスト対象「パワー3000以上」が power_max=3000 |
| ST13-001 | コスト対象「パワー7000以上」が power_max=7000 |

## B. 誘発トリガーが ACTIVATE_MAIN / YOUR_TURN / ON_PLAY に化ける

**症状**: 「〜した時」系の自動誘発（KOされた時 / 場を離れた時 / ライフにダメージを
与えた時 / イベント発動時 等）がパーサで認識されず、起動メインや自ターン常時起動に
フォールバック。発火条件が消えるため **無条件・過剰発動** になる。
`ON_DAMAGE_DEALT_TO_LIFE` 等の enum は定義のみで parser 未割当（grep で確認済み）。

| ID | 化けた誘発 |
|----|-----------|
| OP01-061 | 相手キャラがKOされた時 → YOUR_TURN |
| OP01-062 | イベント発動時 → (条件欠落、1ターン複数ドロー可) |
| OP03-040 | 相手ライフにダメージを与えた時 → ACTIVATE_MAIN |
| OP03-076 | 相手のキャラがKOされた時 → YOUR_TURN |
| OP06-042 | ドンがドンデッキに戻された時 → 自ターン無条件ドロー |
