# リーダー効果 既知の挙動差異（テキスト準拠との差分）

テキスト準拠の期待挙動（各セット仕様ファイル参照）と現挙動が一致しない既知の項目を、
事実ベースで記載する。各項目は対応するテスト（`tests/test_leader_*.py`）で固定されており、
差異が解消されればテストのマーカーを外して通常テスト化する。

## 現在の既知差異

現在、テキスト準拠の期待と一致しない既知差異はない（下記「解消済み」を参照）。

### 解消済み

- **EB01-001「《ワノ国》カウンター非所持キャラにカウンター+1000」**: カウンターは手札から
  使うため、**手札ベース**で実装（カウンターを持たない《ワノ国》キャラを手札にある間
  +1000 カウンター札にする）。パーサ `counter_buff` が「カウンター+Nを持つ」を拾い、
  「カウンターを持たない」を `NO_COUNTER` フラグ化（`matcher` が既存カウンター保持カードを除外）、
  PASSIVE BUFF(COUNTER, zone=HAND) として `passive_counter` に載せる。戦闘のカウンター候補列挙
  （`gamestate.py` BATTLE_COUNTER）を `current_counter > 0` 基準に変更し、+1000 化したキャラが
  候補に出るようにした（既存「手札の…はカウンター+Nになる」系も同時に正しく候補化）。テストは
  `test_eb01_001_passive_grants_counter_to_wano`（xfail 解除・通常テスト化）。

- **ST03-001 ほか「持ち主の〜」系の側無指定**: 「（コストN以下の）キャラ…を持ち主の
  手札に戻す／デッキの下に置く／ライフに加える」で側の明示がない場合の対象を、テキスト
  準拠で自分・相手の両方（`Player.ALL`）に統一（`rules/atoms.py` の `bounce` /
  `deck_bottom_general` / 新規 `field_char_to_life`）。波及は `get_target_cards` で `ALL`
  候補を「相手→自分」順に並べて吸収し、除去の既定選択（CPU・自己対戦・監査）は従来
  どおり相手キャラを選ぶ。UI 上は両側から選択可能。「自分の/相手の」明示と「この…」
  自己参照は対象外。
- **ライフ送りの誤パース**: 「キャラ…持ち主のライフの上か下に表向きで加える」が `life_face`
  で `FACE_UP_LIFE`（自ライフ反転）に誤パースされていた問題を、`field_char_to_life`
  （場のキャラ→持ち主のライフへ `MOVE_CARD`）で修正（OP03-123 / OP11-116 ほか）。
- **OP11-001 / OP11-101（置換 LEAVE）**: 「（相手効果で）場を離れる代わりに〜」の
  `REPLACE_EFFECT` は実装済みで、検証テストを追加（`test_leader_op10_11.py`）。置換
  `sub_effect` の source は「離れるカード」であり、OP11-101「代わりに自分のライフの上に
  裏向きで加える」は離れる《超新星》キャラ自身をライフへ移す。

## 補足

- 上記以外のリーダー効果は、対応するセット仕様ファイルの期待挙動が通常テストで固定されている。
- パーサ・対象解決・条件評価・継続効果の仕組みは `docs/SPEC.md`（§3 カード効果システム／
  §6 実装上の不変条件）を参照。
- 挙動を変更した場合の検証手順（挙動ベースライン再生成・パーサ比較・構造監査・品質ゲート）は
  `docs/TEST_SPEC.md`（§4 ルール追加・検証フロー／§5 品質ゲート）を参照。
