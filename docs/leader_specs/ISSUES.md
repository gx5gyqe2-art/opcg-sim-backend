# リーダー効果 既知の挙動差異（テキスト準拠との差分）

テキスト準拠の期待挙動（各セット仕様ファイル参照）と現挙動が一致しない既知の項目を、
事実ベースで記載する。各項目は対応するテスト（`tests/test_leader_*.py`）で固定されており、
差異が解消されればテストのマーカーを外して通常テスト化する。

## 現在の既知差異

| ID | テキスト準拠の期待 | 現挙動 | テスト | 備考 |
|----|----------------------|--------|--------|------|
| EB01-001 | 自分の特徴《ワノ国》を持ちカウンターを持たないキャラはカウンター+1000を得る | 当該ルール（`RULE_PROCESSING`）は盤面に作用しない | `test_eb01_001_passive_grants_counter_to_wano`（xfail strict=False） | カウンターは手札で消費され、`passive_counter` の再計算は手札のみ対象。どのゾーンにどう反映するかの仕様が未確定 |

### 解消済み

- **ST03-001**（側無指定 BOUNCE の対象範囲）: 「（コストN以下の）キャラ…持ち主の手札に戻す」で
  側の明示がない場合の対象を、テキスト準拠で自分・相手の両方（`Player.ALL`）に修正
  （`rules/atoms.py` の bounce ルール）。約47枚への影響は、`get_target_cards` で `ALL`
  候補を「相手→自分」順に並べることで吸収し、除去の既定選択（CPU・自己対戦・監査）は
  従来どおり相手キャラを選ぶ。UI 上は両側から選択可能。

## 補足

- 上記以外のリーダー効果は、対応するセット仕様ファイルの期待挙動が通常テストで固定されている。
- パーサ・対象解決・条件評価・継続効果の仕組みは `docs/SPEC.md`（§3 カード効果システム／
  §6 実装上の不変条件）を参照。
- 挙動を変更した場合の検証手順（挙動ベースライン再生成・パーサ比較・構造監査・品質ゲート）は
  `docs/TEST_SPEC.md`（§4 ルール追加・検証フロー／§5 品質ゲート）を参照。
