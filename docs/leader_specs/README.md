# リーダーカード効果 テスト仕様書（全137枚）

全リーダーカードを1枚ずつテキスト精読し、パース結果(AST)・実行観測(classify)と
突き合わせて「効果の意味的な正しさ」を検証したテスト仕様書。既存の自動ツール
（構造監査=クラッシュ/カード消失、方向監査）では拾えない **意味バグ** の発見が目的。

- 作成ガイド・テンプレート: [_GUIDE.md](_GUIDE.md)
- プローブ: `OPCG_LOG_SILENT=1 python tests/leader_spec_probe.py <ID>|--set <SET>|--all`
- **検出バグの全体一覧（根本原因別）: [ISSUES.md](ISSUES.md)**

## 使い方

1. 修正対象を決める際は [ISSUES.md](ISSUES.md) の根本原因グループから着手する
   （1修正で複数カードが直る）。
2. 個別カードの期待挙動・テストケースは各セットファイルの該当カード節を参照。
3. 修正後は該当カードの「テストケース」を実装してリグレッションを固定し、
   仕様書の判定ラベルを更新する。
