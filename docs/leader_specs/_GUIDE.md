# リーダーカード効果 仕様 — 作成ガイド

## 目的
全137リーダーカードについて、テキストと **テキスト準拠の期待挙動・テストケース** を
セット別にまとめた仕様を作る。原文テキストが仕様の基準。

## 入力（プローブ）
各セットの全リーダーについて、テキスト＋パース結果(AST要約)＋実行観測を出力:

```
OPCG_LOG_SILENT=1 python tests/leader_spec_probe.py --set OP01 2>&1 | grep -v "GCS Client"
# 単体: ... leader_spec_probe.py OP01-001
```

出力の見方:
- **効果テキスト**: 原文（仕様の基準）
- **パース結果**: `summarize_ability` 指紋（エンジンの解釈）
- **実行観測 (classify)**: 汎用盤面で能力を発動した結果の status と盤面差分
  - `EXECUTED` 盤面が変化 / `NO_CHANGE` 変化なし / `INTERACTIVE` 選択待ちで停止 / `ERROR` 例外

## 各リーダーの記述テンプレート

```markdown
## <ID> <名前>
- **基本**: 色=… / ライフ=… / パワー=… / 属性=… / 特徴=…
- **効果テキスト**:
  > <原文をそのまま>
- **期待挙動（テキスト準拠）**: 能力ごとに箇条書き
  - 能力0 【トリガー】: コスト=… / 効果=… / 対象=… / 数値=… / 期間=… / 条件=… / ターン1回=…
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | … | 能力発動 | … |
  - 境界/異常系: 対象不在 / 条件未達 / コスト払えない 等
```

記述は事実ベースで、テキストが要求する対象・数値・条件・期間・「までを」(is_up_to)・
ターン1回・「持ち主の」「元々のコスト」等の細部を期待挙動として明示する。

## 出力先
- セットごと: `docs/leader_specs/<SET>.md`（例 `OP01.md`）。
- ファイル冒頭に対象ID一覧と概要を書く。
- テストへの落とし込みは [_TEST_GUIDE.md](_TEST_GUIDE.md) を参照。
