# OP02 リーダーカード効果 テスト仕様書

対象ID一覧（8枚）: OP02-001, OP02-002, OP02-025, OP02-026, OP02-049, OP02-071, OP02-072, OP02-093

概要: OP02のリーダー8枚をテキスト精読し、パース結果(AST)・実観測(classify)と突き合わせて意味バグを検証する。
特に「トリガー誤認」「効果種別の取り違え（コスト-1→ドン付与）」「条件節の対象ゾーン違い（キャラ枚数→手札枚数）」「条件節の欠落（コスト0判定の欠落）」「次回登場割引の即時バフ化」を重点確認した。

## OP02-001 エドワード・ニューゲート
- **基本**: 色=赤 / ライフ=6 / パワー=6000 / 属性=特 / 特徴=四皇・白ひげ海賊団
- **効果テキスト**:
  > 【自分のターン終了時】自分のライフの上から1枚を手札に加える。
- **能力分解（テキスト解釈）**:
  - 能力0 【自分のターン終了時】: コスト=なし / 効果=自分のライフ上1枚をHANDへ / 対象=自分のLIFE上1枚 / 数値=1 / 期間=INSTANT / 条件=なし / ターン1回=なし（強制・毎ターン）
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | ライフ5枚・手札5枚 | 自ターン終了 | ライフ4枚・手札6枚（上から1枚移動） |
  | 2 | ライフ0枚 | 自ターン終了 | 移動なし（対象不在） |
  - 境界/異常系: ライフ0で対象不在 / 強制効果のため任意選択ではない点
- **パース照合**: trigger=TURN_END、MOVE_CARD（LIFE→HAND, SELF, count=1）でテキスト通り。is_up_false（強制）も整合。
- **実観測照合**: `p1_hand:5→6, p1_life:5→4` EXECUTED。ライフ減・手札増で期待通り。
- **判定**: ✅ 問題なし — トリガー・対象・移動方向ともテキストと整合。

## OP02-002 モンキー・D・ガープ
- **基本**: 色=赤/黒 / ライフ=4 / パワー=5000 / 属性=打 / 特徴=海軍
- **効果テキスト**:
  > 【自分のターン中】このリーダーか自分のキャラにドン‼が付与された時、相手のコスト7以下のキャラ1枚までを、このターン中、コスト-1。
- **能力分解（テキスト解釈）**:
  - 能力0 【自分のターン中／ドン付与された時】: トリガー=自リーダーor自キャラにドン付与された時 / 効果=相手のコスト7以下キャラ1枚「までを」コスト-1 / 対象=相手キャラ(cost≤7) / 数値=コスト-1(COST_REDUCTION) / 期間=THIS_TURN / is_up_to=true
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 相手にコスト5キャラ、自リーダーにドン付与 | ドン付与イベント発火 | 相手キャラのコストが-1（このターン中） |
  | 2 | 相手キャラがコスト8のみ | ドン付与 | 対象不在で効果なし |
  | 3 | 相手キャラ0枚 | ドン付与 | 0枚選択可（までを）、NO_CHANGE許容 |
  - 境界/異常系: コスト7境界の包含 / コスト8の除外 / 0枚選択
- **パース照合**: 🐛 **効果種別が誤り**。テキストは「相手キャラのコスト-1」だが、パースは `type=ATTACH_DON, value=1, target.player=OPPONENT` となっており「相手キャラにドン‼を付与（=パワー+1000）」と解釈している。本来は `BUFF / COST_REDUCTION / value=-1` であるべき。トリガーも「ドンが付与された時」ではなく単なる YOUR_TURN として解釈されている疑い。
- **実観測照合**: 🐛 `stat: 74969702:pw1000→2000; __don__P:pw10→9`。相手キャラにドンが付与されパワーが上昇している。テキストの「コスト-1」とは正反対の効果（相手を強化してしまう）。
- **判定**: 🐛 バグ — 「コスト-1」が「相手キャラへのドン付与（パワー+1000）」に誤実装。効果種別が取り違えられ、相手を弱体ではなく強化している。

## OP02-025 錦えもん
- **基本**: 色=緑 / ライフ=5 / パワー=5000 / 属性=斬 / 特徴=ワノ国・赤鞘九人男
- **効果テキスト**:
  > 【起動メイン】【ターン1回】自分のキャラが1枚以下の場合、このターン中、次に自分が手札から登場させるコスト3以上の特徴《ワノ国》を持つキャラカードの支払うコストは1少なくなる。
- **能力分解（テキスト解釈）**:
  - 能力0 【起動メイン】【ターン1回】: 条件=自分のキャラ≤1枚 / 効果=このターン中「次に」登場させるコスト3以上《ワノ国》キャラの支払いコスト-1 / 対象=次回登場するキャラへの遅延割引 / 数値=-1 / 期間=THIS_TURN / ターン1回=あり
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自キャラ0枚、手札にコスト4《ワノ国》 | 起動→そのキャラ登場 | 登場時コストが1少ない（実支払3） |
  | 2 | 自キャラ2枚 | 起動 | 条件未達で発動不可 |
  | 3 | 起動後、次に出すのがコスト2《ワノ国》 | 登場 | 割引適用外（コスト3以上のみ） |
  | 4 | 既にターン1回使用済 | 再起動 | 不可 |
  - 境界/異常系: コスト3境界 / 《ワノ国》以外 / 2回目使用
- **パース照合**: ⚠️ 条件 FIELD_COUNT LE 1 SELF・TURN_LIMIT EQ 1 はテキスト通り。ただし効果が `BUFF / COST_REDUCTION` を **HANDのカード(traits=ワノ国, cost_min=3)** に即時付与する形になっており、「次に登場させる1枚への遅延割引（is_up_to無し・select=CHOOSE）」という"次回登場時にコストが少なくなる"継続効果として正しく機能するか要確認。手札の特定カードに今コスト-1を付ける即時バフ解釈だと、登場フロー上で割引が適用されない懸念がある。
- **実観測照合**: ⚠️ ACTIVATE_MAIN: NO_CHANGE。汎用盤面では自キャラ枚数・手札条件・《ワノ国》対象不在で発火しない（条件未達の可能性が高い）。未実装か単なる条件未達かは汎用盤面では切り分け不能。
- **判定**: ⚠️ 要確認 — 「次に登場させるキャラの支払いコスト減」という遅延割引が手札カードへの即時バフとして表現されている。実際に登場時に割引が乗るか個別テストが必要。

## OP02-026 サンジ
- **基本**: 色=緑/青 / ライフ=4 / パワー=5000 / 属性=打 / 特徴=麦わらの一味
- **効果テキスト**:
  > 【ターン1回】自分が元々の効果のないキャラを手札から登場させた時、自分のキャラが3枚以下の場合、自分のドン‼2枚までを、アクティブにする。
- **能力分解（テキスト解釈）**:
  - 能力0 【ターン1回／登場時】: トリガー=自分が「元々の効果のないキャラ」を手札から登場させた時 / 条件=自分のキャラ≤3枚 / 効果=自分のドン2枚「までを」アクティブ化 / 数値=2 / 期間=INSTANT / ターン1回=あり
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | 自キャラ2枚、レストドン3枚 | 効果なしキャラを登場 | ドン2枚がアクティブに |
  | 2 | 自キャラ4枚 | 効果なしキャラ登場 | 条件未達で発動なし |
  | 3 | 効果を持つキャラを登場 | 登場 | トリガー対象外（元々効果のないキャラ限定） |
  | 4 | レストドン0枚 | 登場 | アクティブ化対象不在 |
  - 境界/異常系: キャラ3枚境界 / 効果持ちキャラの除外 / ターン2回目
- **パース照合**: 🐛 **条件のゾーン取り違え**。テキストは「自分のキャラが3枚以下」（場のキャラ枚数）だが、パースは `HAND_COUNT LE 3 SELF`（手札枚数3以下）になっている。`FIELD_COUNT` であるべき。また 🐛 **トリガー誤認**: テキストは「キャラを手札から登場させた時」だが trigger=ACTIVATE_MAIN（起動メイン）で解釈されている。「元々の効果のないキャラ」という対象条件も AST 上に見当たらず欠落の疑い。
- **実観測照合**: ⚠️ ACTIVATE_MAIN: NO_CHANGE。トリガー自体が登場時ではなく起動メインに化けているため、汎用盤面のメインフェイズ起動では条件(手札3以下)未達等で発火しない。実観測単独では切り分け困難だが、パース上のトリガー/条件誤認が確定的。
- **判定**: 🐛 バグ — 条件が「場のキャラ枚数」ではなく「手札枚数」(HAND_COUNT) で実装、かつトリガーが登場時ではなくACTIVATE_MAIN。「元々の効果のないキャラ」条件も欠落の疑い。

## OP02-093 スモーカー
- **基本**: 色=黒 / ライフ=5 / パワー=5000 / 属性=特 / 特徴=海軍
- **効果テキスト**:
  > 【ドン‼×1】【起動メイン】【ターン1回】相手のキャラ1枚までを、このターン中、コスト-1。その後、コスト0のキャラがいる場合、このリーダーは、このターン中、パワー+1000。
- **能力分解（テキスト解釈）**:
  - 能力0 【ドン‼×1】【起動メイン】【ターン1回】: 起動条件=ドン1枚以上・ターン1回 / 効果1=相手キャラ1枚「までを」コスト-1(THIS_TURN) / 効果2=その後「コスト0のキャラがいる場合」このリーダー、このターン中パワー+1000 / 対象=相手キャラ→自リーダー / 数値=-1 / +1000 / 期間=THIS_TURN
- **テストケース**:
  | # | 前提盤面 | 操作 | 期待結果 |
  |---|----------|------|----------|
  | 1 | ドン1枚、相手にコスト1キャラ | 起動 | 相手キャラがコスト0に→コスト0キャラ存在→リーダー+1000 |
  | 2 | ドン1枚、相手にコスト3キャラのみ | 起動 | コスト-1で2に。コスト0キャラ不在→+1000なし |
  | 3 | 相手キャラ0枚 | 起動 | コスト-1対象0枚、コスト0キャラ不在→+1000なし |
  | 4 | ドン0枚 | 起動 | 【ドン×1】支払えず発動不可 |
  - 境界/異常系: コスト1→0で条件成立 / コスト0キャラ不在で+1000なし / ターン2回目
- **パース照合**: 🐛 **条件節の欠落**。後続バフの分岐条件がテキストの「コスト0のキャラがいる場合」ではなく、パースでは `FIELD_COUNT GE 1 SELF`（自分の場にキャラが1枚以上いる場合）になっている。本来は「コスト0のキャラが存在する（＝コスト-1で0になったキャラ等）」という**コスト0判定**であるべきで、対象プレイヤーや判定内容が一致しない。前半の BUFF(OPPONENT, COST_REDUCTION, -1, THIS_TURN, is_up_to=true) はテキスト通り。
- **実観測照合**: ⚠️ ACTIVATE_MAIN: NO_CHANGE。汎用盤面では【ドン×1】条件・相手キャラ対象等で発火しない可能性。ただしパース上、後続バフ条件が「コスト0キャラの存在」ではなく「自分の場のキャラ数≥1」に化けている点が確定的なバグ。これだと自分の場にキャラがいれば（コスト0キャラが無くても）リーダーが+1000され得る一方、相手のコスト0キャラが居ても自場が空なら+1000しない、という誤動作になる。
- **判定**: 🐛 バグ — 後続パワー+1000の条件「コスト0のキャラがいる場合」が `自分の場のキャラ数≥1` に誤実装。コスト0判定が欠落しており、発動可否が逆転し得る。
