## 0. 背景（1段落）

実アプリ対局への人間マーク（2局×16件）の分析で、learned CPU の課題を探索側と価値/policyネット側に
分離した。探索側の実害（読み出しの訪問貼り付き・終局距離の無差別・世界線引き直しの計画非一貫・
読み出しの楽観乗り換え）は 2026-07-11〜12 の2 PR（#180 / #181）で解消済み。**残る実害はすべて
ネット起因**であり、その解消を Gen4 学習 run として1回にまとめて実行する。

## 2. Gen4 run の構成（この順で1回にまとめる）

### 2-1. 前提整備（探索側の小改修・学習の前に）

- **(a) sticky 世界線の自己対戦移植**: `tests/harness/p3_loop.py::selfplay_game`（および
  `pd_gen` 系）の per-move determinize を (game, turn, player) 単位の固定 seed に変更
  （`cpu_learned._world_rng` と同じ規約）。学習データから「付与だけして攻撃しない」非一貫
  プレイを除く＝#1 の汚染源対策。
- **(b) 楽観ランプ計器**: trace（および自己対戦データ）に「前 decide の chosen Q → 今 decide の
  root value」の差分を記録。用途＝(i) 読み出しゲートの再較正データ、(ii) 混合ラベルの品質監視。

### 2-2. 学習本体: 混合ラベル

gen3 の実証（`gen3_adoption_20260711.md` §3-4: 勝敗ラベル単独の追い学習は round60 以降
自己対局均衡へ回帰＝忘却）を踏まえ、ラベルを変える:

- **value**: `y = α · z(勝敗 ±1) + (1−α) · q_root(探索後 root Q・終局減衰込み)` の混合。
  q_root は飽和せず「何手で負けるか／粘れるか」の距離を含む＝#3 の直接対策。α は 0.5 起点で
  ホールドアウト較正。
- **policy**: 従来どおり訪問分布（探索改善が自動で教師に反映される）。
- 開始点: Gen3（round60 凍結）からの温スタート。アーキ変更なし（LC+EffFeat v3・SCALARS_V3）。

### 2-3. 判定ゲート（3層・gen3 と同じ流儀＋マーク回帰を昇格）

1. **人間マーク回帰（16件）**: `replay_reeval.py`（8/8 復元対応済み）で g1/g2 全マークを
   before/after 比較。**必達**: #1〜4 の代表（g1@16/@43/@63・g2@21/@32/@38）のうち
   過半が人間指摘方向へ改善し、既存の正着（g1@12/@24 等）が退行しない。
2. **対 L1 多様97**（`p3_vs_l1.py`）: Gen3 実測 0.854 を下回らない（CI 込みで判断）。
3. **net-vs-net CRN**（arena・対 Gen3 凍結）: 直接対戦で非劣。単一対戦相手の勝率だけで
   採否を決めない（gen3 §4 の非推移性の教訓）。

### 2-4. 完了時の後始末

- 本計画書を削除し、結果を `docs/reports/gen4_adoption_YYYYMMDD.md`（凍結スナップショット）と
  SPEC/TEST_SPEC の正本更新へ吸収する。
- 採用時のロールバック手段は gen3 と同じ（`cpu_learned._DEFAULT_*` の2行・旧 npz は同梱維持）。

## 3. やらないこと（明示）

- カード個別のパッチ・ハードコード（ユーザ方針: 汎用実装のみ）。
- #7（subtree 再利用/多世界平均）: Gen4 の対象外。ネット改善後もレイテンシ予算内で
  伸びしろが必要になった時に別途計測付きで検討。
- 勝敗ラベル単独での追い学習の再開（gen3 §3 で忘却を実証済み）。
