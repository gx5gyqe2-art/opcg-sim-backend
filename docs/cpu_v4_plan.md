## 0. 呼称の整理（ユーザ指定・2026-07-12）

学習 CPU（ネット）の世代は **v番号** で呼ぶ:

| 呼称 | 実体 | 過去文書での旧称 |
|---|---|---|
| **v1** | 旧出荷ネット（単色自己対戦・`gen2_*.npz`） | 「Gen2」 |
| **v3** | **現本番**＝v1 を蒸留→実効10,112局の追い学習（LC+EffFeat・`gen3_*.npz`） | 「Gen3」 |
| **v4** | 本計画の成果物（v3 からの温スタート） | （起草時の仮称「Gen4」） |

> 注意: **符号化世代（encoder version v1/v2/v3・`SCALARS_V3` 等）はネット世代とは別軸**。
> v3 ネットは符号化 v3 を使うため番号が一致しているが、v4 ネットも符号化 v3 のままで良い

## 1. 背景（1段落）

実アプリ対局への人間マーク（2局×16件）の分析で、learned CPU の課題を探索側と価値/policyネット側に
分離した。探索側の実害（読み出しの訪問貼り付き・終局距離の無差別・世界線引き直しの計画非一貫・
読み出しの楽観乗り換え）は 2026-07-11〜12 の2 PR（#180 / #181）で解消済み。**残る実害はすべて
ネット起因**であり、その解消を v4 学習 run として1回にまとめて実行する。

## 3. ゲーム特性から見た根因仮説: 「2つの時計」の欠落

#1〜4 は個別の症状に見えるが、ゲーム特性の観点では**同じ1つの量の欠落**で説明できる:
このゲームは「相手を削り切るまでの自分の残りターン数」と「削り切られるまでの自分の残りターン数」
の**レース**であり、ほぼ全てのリソースの交換レートがこの時計に依存する。

- **ライフ**: 残りゲームが長いほど守る価値が高い。短いレースなら受けてカードに変えてよい。
- **防御系の永続効果**（ナミ【ドン!!×1】+2000 等）: 「残りターン数 × 毎ターンの節約」の**年金**。
  ゲーム長が分からなければ値付け不能（#2）。
- **今すぐの1点**: 自分のリーサルが相手の展開曲線より先に来るなら価値大。来ないなら
  **相手に手札を1枚与えるだけ**（ライフ→手札）で、後半に強いデッキ（センゴク等）への
  早期チップはほぼ利敵（#1・g2@20 の「このターンに割りたい理由がない」）。
- **受けるか守るか**: 「相手の盤面圧が高い（時計が加速する）なら守る／相手のライフが少ない
  （ゲームが短い）なら受けてよい」＝**両者の時計の比較**そのもの（#4・g2@32 のマーク文言）。
- **敗勢の値**: 「負け」を点でなく**距離**（あと何ターン粘れるか）で表現できていない（#3）。
  現行の value 教師 ±1 は「3ターンで潰れた負け」と「10ターン粘った負け」を同じ値にする＝
  **ゲーム長は現行ラベルから原理的に学習不能**。

傍証: 手書き評価 L1 は時計を**明示的に**持っており（`_clock_of`／`γ_surv`＝あと何ターン展開に
回せるか／`amp`＝圧力によるカウンター価値増幅）、防御系マーク（g1@63・g2@32）では L1 の第二意見が
人間指摘側だった。v3 は L1 に勝率で勝ったが、L1 が手書きで持つ**時間構造を獲得しないまま**
材料勘定の精度で勝っている。v4 の学習仕様（§4）はこの診断に基づく。

## 4. v4 run の構成（この順で1回にまとめる）

### 4-1. 前提整備（探索側の小改修・学習の前に）

- **(a) sticky 世界線の自己対戦移植**: `tests/harness/p3_loop.py::selfplay_game`（および
  `pd_gen` 系）の per-move determinize を (game, turn, player) 単位の固定 seed に変更
  （`cpu_learned._world_rng` と同じ規約）。学習データから「付与だけして攻撃しない」非一貫
  プレイを除く＝#1 の汚染源対策。
- **(b) 楽観ランプ計器**: trace（および自己対戦データ）に「前 decide の chosen Q → 今 decide の
  root value」の差分を記録。用途＝(i) 読み出しゲートの再較正データ、(ii) 混合ラベルの品質監視。

### 4-2. 学習本体: 混合ラベル＋残りターン補助ターゲット

v3 採用時の実証（`gen3_adoption_20260711.md` §3-4: 勝敗ラベル単独の追い学習は round60 以降
自己対局均衡へ回帰＝忘却）を踏まえ、ラベルを変える:

- **value（混合ラベル）**: `y = α · z(勝敗 ±1) + (1−α) · q_root(探索後 root Q・終局減衰込み)`。
  q_root は飽和せず「何手で負けるか／粘れるか」の距離を含む＝#3 の直接対策。α は 0.5 起点で
  ホールドアウト較正。
- **残りターン数の補助ターゲット（§3 の診断に基づく追加）**: 各局面に「終局までの残りターン数」を
  ラベル付けし、補助ヘッド（または補助損失）で予測させる。**時計の概念を明示的に教える**、
  汎用で安価な手当て。自己対戦記録から機械的にラベル生成できる（追加コスト≈0）。
  主損失への重みはホールドアウトで較正（補助はあくまで表現学習の誘導・推論時は不使用でも可）。
- **policy**: 従来どおり訪問分布（探索改善が自動で教師に反映される）。
- 開始点: v3（round60 凍結・`gen3_*.npz`）からの温スタート。アーキ変更なし
  （LC+EffFeat・SCALARS_V3。補助ヘッドのみ追加＝恒等温スタートを崩さない形で足す）。

### 4-3. 判定ゲート（3層・v3 採用時と同じ流儀＋マーク回帰を昇格）

1. **人間マーク回帰（16件）**: `replay_reeval.py`（8/8 復元対応済み）で g1/g2 全マークを
   before/after 比較。**必達**: #1〜4 の代表（g1@16/@43/@63・g2@21/@32/@38）のうち
   過半が人間指摘方向へ改善し、既存の正着（g1@12/@24 等）が退行しない。
2. **対 L1 多様97**（`p3_vs_l1.py`）: v3 実測 0.854 を下回らない（CI 込みで判断）。
3. **net-vs-net CRN**（arena・対 v3 凍結）: 直接対戦で非劣。単一対戦相手の勝率だけで
   採否を決めない（v3 採用報告 §4 の非推移性の教訓）。

補助指標（§3 の診断の検証）: (i) 残りターン補助ヘッドの予測誤差、(ii) 自己対戦の
**平均ゲーム長の分布**（防御が報われる長期戦がデータに現れているか）を run 中監視する。

### 4-4. 完了時の後始末

- 本計画書を削除し、結果を `docs/reports/v4_adoption_YYYYMMDD.md`（凍結スナップショット）と
  SPEC/TEST_SPEC の正本更新へ吸収する。
- 採用時のロールバック手段は v3 と同じ（`cpu_learned._DEFAULT_*` の2行・旧 npz は同梱維持）。
