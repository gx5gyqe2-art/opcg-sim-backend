# CPU v8 計画: 教師CPU本線への転換（2026-07-16 確定）

## 0. 転換の判断（なぜ自己対戦の改良を止めるか）

- v7 本走の実測: **8,352局・ゲート15回全棄却**（勝率 0.33〜0.50 帯・昇格ゼロ）。
  v7 の狙い（教師エコー低減）は KL +31%/top1不一致 +6.5pt で達成したが、勝率は動かなかった。
- 原因は仮説でなく確定済み（`docs/reports/seesaw_probe_20260716.md`）:
  効率判断に value が無差別（±0.01〜0.05）→ 訪問は prior 追従 → 教師=prior のエコー →
  **ループ内に真実の信号が無い**。ラベル平滑化や再ラベルでは信号は生まれない。
- 一方、反実仮想レフェリー（真盤面＋CRN対照ロールアウト）は外部の真実源として機能実証済み
  （`docs/reports/referee_true_board_20260716.md`: @64 逆転検出・@93 実利化・@68 根拠つき反論）。
- ユーザ方針: 「何万回も繰り返した勝率ではなく、数回全ての選択肢を試してこれが正しい、
  という人間の教師になれるCPU」。**勝率レースから離脱し、レフェリーを教師CPUとして製品化する。**

v7 の自走は停止（learner/w1〜w4 の外部セッションを停止。状態は git に永続化済み・再開可能）。
昇格ゲート／アンカーゲート等の v6/v7 機構は将来ネットを更新する時のための資産としてそのまま残す。

## 1. 柱A: ターンプラン自動列挙（手指定の排除）

現状のプランモードは手順列を人間が文字列指定する。これを自動化する:

- **列挙**: 決定点から自ターン内の手順を DFS で展開。**手番が自分から離れた瞬間**
  （攻撃宣言=防御応答へ・TURN_END=相手ターンへ）をプラン終端とする——「以降はロールアウトの
  領分」という境界を action_type の列挙でなく手番遷移で判定（汎用・カード非依存）。
- **等価除去**: 同一手集合の並べ替え（sorted equivキー多重集合）は最初に見つかった順序のみ残す。
  資源消費で順序非可換な場合は合法な順序だけが自然に列挙される。
- **ビーム**: 各深さで中間盤面を value net（固定 gen5）で評価し上位 B 本のみ展開
  （既定 B=12・深さ上限 L=4・プラン上限 P=16）。**切り捨ては件数を必ずログ**（無言の縮約禁止）。
- 実行はプランごとに CRN 世界線を共有（従来のプランモードと同じ）。

## 2. 柱B: 同価値バンドの既定化

第1報の教訓（±0.3 のライフ差はノイズ）を判定既定に昇格:

- 勝ち数が同じ かつ |残ライフ差の差| < 0.5（`--band` 可変）なら**同価値**と申告し、
  バンド外の差だけを序列として断定する。
- root 全数モードにも適用（最善と同帯の手に ≈ マーク）。

## 3. 柱C: コーチングスイープ（1局まるごと採点）

`tests/scripts/coach_sweep.py`（新規・1トピック=1ファイル）:

- 録画1局の指定範囲の決定点（既定=CPU側全決定）ごとに真盤面を再生し、柱Aの自動列挙＋
  レフェリーで**実際の手（プラン）と最良プランの差**を測る。
- 出力: 損失があった決定だけを「実際の手 / 最良プラン / 勝ち差・ライフ差」で列挙する採点表。
  同価値バンド内は不問（作法差を報告しない）。
- コスト制御: 既定 worlds=4・sims=32・決定点範囲指定。**教師の答え合わせは少数局面を
  深く**が原則（全対局×全決定の常時実行は想定しない）。
